## 2. 격리 원칙 — 왜 완전 분리인가

### 오염 시나리오 (절대 금지)
```
❌ A팀이 B팀 결과를 보는 경우    → 실험 오염
❌ B팀이 A팀 방식을 참고하는 경우 → 독립성 훼손
❌ 두 팀이 같은 프로젝트에 공존   → 문화적 교차 감염
❌ 같은 컨텍스트 파일 공유        → 사전 지식 오염
```

### 격리 구현 방법 (MyCrew 기준)
```
Group A: 별도 워크스페이스 또는 격리된 프로젝트 계정
Group B: 현재 MyCrew 워크스페이스
동일 과제: 아리가 양쪽에 동시 투입 (결과 공유 차단)
```

## 3. Group A — 적대적 프레임워크 설계

### 운영 방식
```
Round 1: 에이전트 A1이 과제 수행 → 결과 출력
Round 2: 에이전트 A2가 A1 결과를 '반박'하는 역할로 수행
Round 3: A1이 반박에 대응하며 방어·수정
Round 4: 심판(Judge Agent)이 최종 승자 결정
```

### 특징
- 에이전트 간 명시적 경쟁 구조
- 결과물은 '승자'의 안으로 확정
- 협력 없음, 학습 전달 없음

## 4. Group B — CKS 프레임워크 설계

### 운영 방식 (4단계 루프)

```
STEP 1: 독립 작업 (Independent Work)
  에이전트 B1, B2가 동일 과제를 독립적으로 수행
  서로의 결과를 보지 않음
  ↓

STEP 2: 교차 평가 (Cross-Evaluation)
  B1이 B2의 결과를 읽고 평가 (비판이 아닌 인식)
  B2가 B1의 결과를 읽고 평가
  평가 기준: "내가 못한 것, 상대가 한 것은 무엇인가?"
  ↓

STEP 3: 회고 루프 (Retrospective Loop)
  각 에이전트가 교차 평가를 바탕으로 자신의 출력 보완
  흡수할 것: 상대의 고유 강점
  유지할 것: 자신의 고유 관점
  ↓

STEP 4: 통합 (Synthesis)
  조정자(Coordinator)가 두 보완안을 통합
  창발 아이디어 식별 및 기록
  최종 결과물 확정
```

### 핵심 원칙
- **비적대적**: 누가 맞는지가 아니라 무엇이 더 좋은지
- **비파괴적**: 상대 결과를 부정하지 않고 흡수
- **창발 지향**: 단일 에이전트로는 불가능한 아이디어 출현 목표

## 5. 실험 과제 설계

### 과제 선정 기준
- 단일 정답이 없는 창의적·전략적 과제
- 실제 비즈니스 컨텍스트 (소시안 마케팅)
- 정량 측정이 가능한 산출물 (KPI 수치, 아이디어 수 등)
- 동일 과제를 반복 실험 가능한 구조

### 과제 유형별 분류
| 유형 | 예시 | 측정 초점 |
|------|------|---------|
| **전략 기획** | 캠페인 기획안 작성 | EII, KSI |
| **콘텐츠 생성** | 릴스 대본 3종 | HER, OP |
| **문제 해결** | 전환율 개선 방안 | PSC, IRC |
| **데이터 분석** | VOC 기반 인사이트 | TEI, HER |

## 6. 실험 라운드 설계

### 파일럿 라운드 (이미 완료: 2026-04-16)
- 과제: 소시안 마케팅팀 온보딩 설계
- Group B 방식으로 진행 (루카 + 소넷 독립 → Prime 교차평가)
- Group A 없음 (대조군 미설정) → 향후 재실험 필요

### Round 1 (계획)
- 동일 과제 설정
- Group A, B 동시 투입
- 완전 격리 환경 구성
- 결과 수집 후 벤치마크 측정

### 누적 실험 구조
```
Round 1 → 데이터 수집 → KSI 계산 → 피드백 반영
Round 2 → 데이터 수집 → KSI 계산 → 피드백 반영
...
Round N → 통계적 유의성 확인 → 논문 데이터 확정
```

## 7. MyCrew 플랫폼과의 연결

```
MyCrew 플랫폼 = 리빙 랩 (Living Lab)
    │
    ├── [격리 프로젝트 A] 적대적 팀
    │     - 에이전트: 토론형 프롬프트 설정
    │     - 결과: 칸반 A열에 별도 기록
    │
    ├── [격리 프로젝트 B] CKS 팀
    │     - 에이전트: 교차평가·회고 루프 프롬프트
    │     - 결과: 칸반 B열에 별도 기록
    │
    └── [벤치마크 대시보드]
          - TEI / HER / KSI / EII / OP 실시간 집계
          - 논문 데이터 자동 추출
```
