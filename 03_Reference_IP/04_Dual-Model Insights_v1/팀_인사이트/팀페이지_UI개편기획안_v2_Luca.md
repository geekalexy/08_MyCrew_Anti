## 🏗 핵심 설계 원칙 (v2.1 신규 추가)

> **"UI는 SaaS, 실험은 뒷단"**

| 레이어 | 역할 | 주요 구성 |
|--------|------|---------|
| **UI Layer** (프론트엔드) | 일반 사용자가 보는 상용 대시보드 | 팀 이름, 역할, 프로젝트 배지, 멤버 관리 |
| **Orchestration Layer** (ARI 엔진) | 실제 에이전트 라우팅 | CKS 위상 결정, 실험군/대조군 분리 |
| **Research Layer** (백엔드 전용) | CKS 실험, 지표 수집 | KSI/TEI 측정, experiment_log.db, 위상 분석 |

**UI에서 영구 금지 항목** (상용화 저해 요소):
- ~~"실험군 / 대조군"~~ 같은 연구 용어
- ~~"CKS 프레임워크", "적대적 대조군"~~ 같은 내부 아키텍처 명칭
- ~~실험 역할 배지~~ (experiment_role 표시)
- ~~팀 컨트롤 바~~ (Play/Pause/Stop 개별 카드 버튼) ← 팀 단위로 이동

## 1. 탭 아키텍처 (대분류)

팀 페이지 상세 화면은 크게 두 탭으로 나뉩니다.

1. **[조직도] 탭**: 에이전트 명단, 채용, 그룹핑 조회
2. **[팀 관리] 탭**: 팀명 수정, 로고 변경, 워크스페이스 설정

> **⚠ 분석 지표 탭 제거**: 상용화 1차 버전에서는 내부 CKS 메트릭(토큰 소모, KSI 등)을 UI에서 제거. 내부 모니터링은 백엔드 로그로만 기록.

## 2. 세부 UI 컴포넌트 배치

### A. 조직도 탭 (Roster Tab) — Y-Shape Org

```
              ┌──────────┐
              │   ARI    │  ← 중앙 오케스트레이터 (단독 배치)
              └──────────┘
           /               \
  ┌──────────────┐    ┌──────────────┐
  │  🟢 Team A   │    │  🔵 Team B   │
  │ (크리에이티브)│    │  (검수팀)    │
  │  PICO LUMI  │    │  NOVA OLLIE  │
  └──────────────┘    └──────────────┘
```

- **ARI (비서)**: 최상단 정중앙, 양쪽 팀으로 업무 조율하는 구심점
- **Team A (크리에이티브)**: 좌측 브랜치 — PICO(기획자), LUMI(디자이너)
- **Team B (검수팀)**: 우측 브랜치 — NOVA(마케터), OLLIE(디렉터)
- 기존 오리지널 조직도 UI 룩(트리/선 연결감) 계승

**에이전트 카드 표시 항목** (v2.1 기준):
- 이름 (수정 가능)
- 역할 (수정 가능)
- 모델명
- 온라인 상태 dot
- ~~CKS 실험 역할 배지~~ → **삭제됨**
- ~~개별 Play/Pause/Stop 컨트롤~~ → **팀 헤더로 이동됨**

**팀 그룹 헤더**:
- 팀명 클릭 → 인라인 편집 (상용 서비스 UX)
- 팀 컨트롤 버튼 (▶ ⏸ ⏹) 헤더 우측에 배치
- 프로젝트 배지 🏷️ 표시

**헤더 채용 버튼 제거**:
- 상단 `[+ 새 팀 만들기]` 버튼 → OrgView 헤더에서 제거 검토
- 대신 각 그룹 트리 마지막에 `[+ 임시 크루 영입]` 빈 슬롯 카드 유지

### B. 다중 팀 및 프로젝트 매핑

- `[+ 새 팀 만들기]` 클릭 → 모달
  - "기존 프로젝트에 합류" vs "새 프로젝트 생성" 선택
  - 컨텍스트 오염 방지를 위한 프로젝트 분리 UX

### C. [팀 관리] 탭 (v2.1 축소)

- **워크스페이스 설정**: 회사 로고, 회사명 편집 (기존 헤더에서 이전)
- ~~CKS 분석 메트릭 카드~~ → **v2.1에서 제거** (백엔드 전용 처리)
- Team Guidelines Editor

## 3. 백엔드 레이어 (UI 비노출 유지 항목)

아래 항목은 백엔드와 DB에는 존재하지만 **절대 프론트엔드 UI에 노출하지 않는다.**

| 항목 | 저장 위치 | 노출 여부 |
|------|---------|---------|
| teamGroup (A/B/independent) | agentStore.js | ❌ 내부 라우팅용만 사용 |
| experimentRole | agentStore.js | ❌ 완전 숨김 |
| KSI / TEI 지표 | experiment_log.db | ❌ 개발자 콘솔/로그만 |
| CKS 위상 구조 | Comment DTO (내부) | ❌ 백엔드 로직 전용 |
| GET /api/metrics/cks | server.js Mock | ❌ 내부 API만 (UI 연결 X) |
