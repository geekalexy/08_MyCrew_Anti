## 3. 정리 계획 (TO-BE)

### Phase A — 즉시 정리 가능 (잡파일)

```
[ ] notebook_output_*.txt 45개 → 01_아리_엔진/outputs/ 로 이동 또는 삭제
[ ] test_*.js 14개 → 01_아리_엔진/tests/ 폴더 신설 후 이동
[ ] .env.legacy → 01_아리_엔진/_archive/ 이동
[ ] check_models.js, list-models.* → tests/ 또는 scripts/ 이동
```

### Phase B — 세션로그 통합

```
현재 (7개 파일, 2곳에 분산):

  세션로그_20260424_Luca.md  ← 한글명
  SESSION_LOG_20260425_Luca.md  ← 날짜형식 다름

목표:
  02_System_Development/
  └── _SESSION_LOGS/          ← 전용 폴더 신설
      ├── 2026-04-22_Luca.md
      ├── 2026-04-22_Sonnet.md
      ├── 2026-04-23_Luca.md
      ├── 2026-04-23_Prime.md
      ├── 2026-04-24_Luca.md
      ├── 2026-04-25_Luca.md

규칙: YYYY-MM-DD_[작성자].md 통일
```

## 4. 파일명 통일 규칙 (앞으로)

| 종류 | 규칙 | 예시 |
|------|------|------|
| 세션로그 | `YYYY-MM-DD_[작성자].md` | `2026-04-25_Sonnet.md` |
| PRD | `Phase[번호]_[주제]_PRD.md` | `Phase26_AI팩토리_PRD.md` |
| 구현보고서 | `Phase[번호]_Sprint[번호]_보고서.md` | `Phase26_Sprint2_보고서.md` |
| 운영가이드 | `[주제]_가이드.md` | `Anti-Bridge_가이드.md` |
| 테스트 | `test_[기능].js` → `tests/` 폴더 내 | `tests/test_tts.js` |

## 5. 즉시 실행 vs 나중에

| 작업 | 우선순위 | 이유 |
|------|---------|------|
| notebook_output 삭제 | 🔴 즉시 | 아무 기능 없는 잡파일 45개 |
| 세션로그 폴더 통합 | 🟡 이번 주 | 검색/참조 편의 |
| test 파일 이동 | 🟡 이번 주 | 루트 가독성 |
| 아키텍처 문서 분류 | 🟢 다음 주 | 현재 작동에 영향 없음 |
| 스킬 명칭 정리 | 🟢 나중에 | 우선순위 낮음 |
