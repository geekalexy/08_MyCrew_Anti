# Cmux + Paperclip 멀티프로젝트 확장 전략

## 1. 워크스페이스 복제 방법

cmux에는 워크스페이스 포크/복제 기능이 없으므로 **부트스트랩 스크립트**로 자동화합니다.

### 새 프로젝트 생성 흐름
1. `cmux new-workspace --name "새프로젝트"`
2. 패널 분할 (C1, C2, Bridge, Paperclip 터미널/브라우저)
3. 각 패널에서 자동 실행 (claude, paperclip onboard, bridge.py)
4. C2 학습 데이터 자동 주입

### 복제 가능 여부

| 항목 | 복제 가능 | 방법 |
|---|---|---|
| 패널 레이아웃 | ✅ | 스크립트로 new-split 자동화 |
| Bridge 스크립트 | ✅ | 파일 복사 + surface ID만 변경 |
| C2 학습 데이터 | ✅ | memory 폴더 복사 또는 초기 프롬프트 |
| Paperclip 회사 | ❌ | 새로 `npx paperclipai onboard` |
| 텔레그램 봇 | ⚠️ | 봇 1개로 멀티 프로젝트 or 봇 추가 생성 |
| Tailscale | ✅ | 이미 설정됨, 포트만 다르게 |

## 2. 멀티 오너 (텔레그램 수신자 분리)

### 방법 비교

| 방법 | 구조 | 장단점 |
|---|---|---|
| **A. 봇 1개 + Chat ID 분리** | 봇은 하나, 프로젝트별 수신자 Chat ID만 다르게 | 가장 간단. 봇 관리 1개. 단, 모든 프로젝트가 같은 봇 토큰 공유 |
| **B. 프로젝트별 봇 생성** | 각 프로젝트마다 @BotFather로 새 봇 생성 | 완전 분리. 오너가 자기 봇만 봄. 관리 다소 복잡 |
| **C. 텔레그램 그룹 활용** | 프로젝트별 그룹 채널 생성, 봇을 그룹에 초대 | 오너 + 팀원도 함께 볼 수 있음. 협업에 유리 |

### 추천: A안 (봇 1개 + Chat ID 분리)

```python
# 현재
chat_id = bot_data["chat_id"]  # 1명

# 변경
PROJECT_OWNERS = {
    "왕짱플젝": 123456789,    # 본인 Chat ID
    "뉴프로젝트": 987654321,  # 새 오너 Chat ID
}
```

새 오너가 할 일:
1. 텔레그램에서 봇에게 `/start` → Chat ID 자동 등록
2. 브릿지가 해당 프로젝트 알림을 그 Chat ID로만 전송

### 보안 고려

| 항목 | 대응 |
|---|---|
| 다른 오너가 내 프로젝트 접근 | Chat ID 화이트리스트로 차단 |
| cmux 접근 분리 | 프로젝트별 별도 워크스페이스, 오너는 자기 surface만 제어 |
| 봇 토큰 노출 | 오너별 봇 분리(B안)가 더 안전 |

> 상용화/판매 시 B안(프로젝트별 봇), 내부 운영 시 A안이 빠름

## 3. A안 규모 한계

| 규모 | 상태 | 병목 |
|---|---|---|
| 1~5개 | 문제없음 | - |
| 5~15개 | 가능하지만 주의 | PC 메모리/CPU |
| 15개+ | 한계 | 텔레그램 폴링 지연, cmux 패널 과다, 메모리 부족 |

**프로젝트당 리소스:**
- Claude Code x2 (C1, C2) → ~500MB
- Paperclip → ~300MB
- Bridge → ~50MB
- **합계: 약 1GB/프로젝트**

MacBook Pro 기준 **5~8개**가 현실적.

## 4. 고객 PC 이식 방법

### A안: 원커맨드 설치 스크립트 (가장 현실적)

```bash
curl -sL https://your-domain.com/install.sh | bash
```

스크립트가 하는 일:
1. Homebrew 확인/설치
2. cmux 설치
3. Node.js, Python3 확인
4. `npx paperclipai onboard --yes`
5. `pip install python-telegram-bot`
6. Bridge 스크립트 복사 + 고객 봇 토큰/Chat ID 설정
7. Claude Code 2개 패널 자동 생성
8. C2 학습 데이터 자동 주입
9. Tailscale 설치 + 로그인 안내

**고객이 제공해야 할 것:**
- 텔레그램 봇 토큰 (BotFather에서 생성)
- Anthropic API 키 (Claude Code용)
- Tailscale 계정

### B안: Docker (확장성 고려 시)

```bash
docker run -d \
  -e TELEGRAM_TOKEN=xxx \
  -e ANTHROPIC_KEY=xxx \
  your-image/socian-agent
```

cmux가 GUI 앱이라 Docker에 넣기 어려움. Paperclip + Bridge만 컨테이너화하고 cmux는 호스트 설치하는 하이브리드 방식이 현실적.

### C안: 클라우드 SaaS (궁극적 방향)

```
고객 → 웹 대시보드 가입 → 회사 생성 → 텔레그램 연결
(서버에서 Paperclip + Bridge가 고객별로 실행)
```

인프라 비용 발생, 개발 규모 큼.
