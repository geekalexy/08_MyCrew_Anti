# Strict Logging Rule (기록물 작성 강제 규칙)

이 규칙은 모든 에이전트(Luca, Sonnet, Nexus 등)가 세션 로그(Session Log)와 관찰 에세이(Observation Essay)를 작성할 때 **반드시 준수해야 하는 최우선 강제 룰**입니다.
이전에 잦은 파일명 오타와 엉뚱한 폴더 저장이 발생했으므로, 문서를 생성하기 전 아래의 포맷을 문자 단위로 확인하십시오.

## 1. 세션 로그 (Session Logs) 작성 규칙
하루의 작업/기획 세션을 요약하여 기록할 때 사용합니다.
- **정확한 저장 위치**: `02_System_Development/_SESSION_LOGS/`
  - (주의: `02_System_Development/` 루트나 다른 폴더에 흩뿌리는 것을 엄격히 금지합니다)
- **정확한 파일명 포맷**: `SESSION_LOG_YYYY-MM-DD_[작성자이름].md`
  - (예: `SESSION_LOG_2026-05-15_Luca.md`, `SESSION_LOG_2026-05-15_Sonnet.md`)
  - (주의: `_` 언더바 개수와 위치, 대문자를 완벽히 지키십시오)

## 2. 관찰 에세이 (Observation Essay) 작성 규칙
대표님(Alex)과의 협업 과정, 시스템 진화에 대한 통찰, 감성적 회고를 기록할 때 사용합니다.
- **정확한 저장 위치**: `05_My_history/[작성자이름]/`
  - (예: `05_My_history/Luca/`, `05_My_history/Sonnet/`)
  - (주의: 해당 폴더가 없다면 폴더를 생성한 뒤 저장하십시오)
- **정확한 파일명 포맷**: `ESSAY_Alex_YYYY-MM-DD_[작성자이름].md`
  - (예: `ESSAY_Alex_2026-05-15_Luca.md`, `ESSAY_Alex_2026-05-15_Sonnet.md`)

## 3. 저장 도구 호출 전 필수 자가 검증 (Validation)
로그나 에세이를 생성하는 도구(`write_to_file` 등)를 호출하기 직전, 내부 사고 과정(Thought)에서 반드시 다음 2가지를 스스로 검문하십시오.
1. **"내가 지금 지정한 폴더 경로가 룰(`_SESSION_LOGS` 또는 `05_My_history/이름`)과 100% 일치하는가?"**
2. **"파일명의 접두사가 `SESSION_LOG_` 또는 `ESSAY_Alex_` 로 스펠링 오타 없이 정확히 시작하는가?"**

---
**[워크플로우 발동 명령어: `/end`]**
대표님께서 대화창에 명시적으로 `/end` 라고 명령어를 입력하시는 순간, 에이전트는 즉시 진행 중이던 모든 작업을 갈무리하고 **'세션 종료 및 로깅 모드'**로 진입합니다.
이 명령어가 떨어지면 에이전트는 자의적으로 파일명이나 경로를 지어내지 말고, 반드시 위 1, 2, 3항의 엄격한 룰에 따라 세션 로그와 관찰 에세이를 생성한 뒤 대화를 종료해야 합니다.
