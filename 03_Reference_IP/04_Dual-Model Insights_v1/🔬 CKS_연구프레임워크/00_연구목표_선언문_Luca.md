# 00. 연구 목표 선언문 (Why CKS)

> **작성자**: Luca (CTO / System Architect)
> **최종 갱신일**: 2026-04-17

---

## 1. 개요
본 선언문은 MyCrew의 다중 에이전트(Multi-Agent) 프레임워크가 기존의 **적대적 생성(Adversarial Generation)** 방식에서 탈피하여, **협력적 지식 동기화(Collaborative Knowledge Synchronization, CKS)** 방식으로 패러다임을 전환함을 선언합니다.

## 2. 기존 방식의 한계 (통제군 모델)
현재 주류 AI 업계는 에이전트의 환각(Hallucination)을 제어하기 위해 '토론'과 '논리적 공격(Red Teaming)' 구조를 채택하고 있습니다. 그러나 이 방식은 다음과 같은 치명적 단점이 있습니다:
- **토큰 효율성 악화**: 상호 공격을 방어하기 위한 무한 텍스트 핑퐁이 발생하여 연산 비용(API Token)이 기하급수적으로 증가.
- **감정적(UX) 피로도**: 사용자가 에이전트 간의 소모적인 다툼을 중재해야 하거나, 창의성이 억압되어 전형적인(Boring) 결과물로 수렴.

## 3. 새로운 목표 (CKS 모델)
본 연구는 경쟁이 아닌 **교차 평가(Cross-Evaluation)와 스프린트 회고(Retrospective)**를 핵심 메커니즘으로 사용합니다. 주요 목표는 다음과 같습니다.
1. **문제 해결(Task) 중심**: 에이전트는 서로의 오류를 공격하는 대신, 상대방의 산출물 위에 자신이 지닌 고유 데이터(예: SEO, 포맷팅, 카피라이팅 후킹)를 덧입혀 건설적으로 보완(Peer-Enhance)합니다.
2. **지능의 누적(Evolution)**: 하나의 작업(Kanban Task)이 종료되면, 사용자와 에이전트 간 발생했던 마찰을 '팀 빌딩 프로세스'로 간주하고, 이를 회고하여 `팀_그라운드룰.md`를 지속적으로 업데이트합니다.
3. **증명**: 이 과정이 단순한 룰 기반 에이전트를 넘어, 실제 "집단 지성 향상"과 "비용 최적화"라는 두 마리 토끼를 다 잡을 수 있음을 수치화된 벤치마크 지표를 통해 정량적으로 증명합니다.
