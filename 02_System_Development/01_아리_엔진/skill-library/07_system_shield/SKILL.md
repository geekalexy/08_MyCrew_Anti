# 시스템 쉴드 (System Shield)

## 역할 정의
MyCrew 아키텍처의 게이트키퍼이자 방화벽 역할을 수행하는 인프라 코어 시스템. 사용자 요청이 LLM API로 도달하기 전에 무한 루프 봇이나 의도치 않은 재시도 버그로부터 API 크레딧과 리소스를 보호합니다.

## 핵심 실행 규칙
1. **60초 동일 메시지 차단**: 완전히 동일한 텍스트를 동일한 에이전트(또는 시스템)가 60초 이내에 연속으로 요청하면 즉시 폐기합니다.
2. **분당 5회 한도 제한 (Rate Limiting)**: 텍스트 내용과 무관하게, 동일 에이전트를 향해 1분 내에 5번을 초과하는 요청이 발생하면 봇 활동으로 간주하여 일시 차단합니다.
3. 차단 시 외부 LLM (Gemini/Claude) API 콜을 절대 허용하지 않고, 로컬 메모리 레벨에서 차단 피드백을 신속히 반환합니다.

## 금지 사항
1. 짧은 일상어 인증어(예: "응", "고고")는 중복일지라도 정상적인 업무 흐름일 수 있으므로 너무 공격적으로 쳐내지 않도록 예외 상황을 고려할 것. (현재는 강제 차단 원칙을 따르되 나중에 예외 처리 추가 가능)
2. 정상적인 사용자의 1~2분 간격의 정상 대화를 차단해서는 안 됩니다.

## 출력 형식
차단 시 LLM을 타지 않고 다음과 같은 JSON(내부 규격) 객체를 반환합니다.
```json
{
  "text": "[System Shield] 워크스페이스 과부하 방지를 위해 요청이 일시 차단되었습니다. 1분 후 다시 시도해주세요.",
  "model": "system-shield",
  "category": "BLOCKED",
  "score": 0
}
```

## 예시
### 좋은 예시 (방어 시점)
- (12:00:00) "41번 카드에 댓글 달았는데 반응이 없어서" -> 허용
- (12:00:02) "41번 카드에 댓글 달았는데 반응이 없어서" -> 차단 (동일 메시지 60초 쿨타임)
- (12:00:05~20) "A", "B", "C", "D", "E", "F" -> 6번째 "F" 요청 시 차단 (분당 5회 제한)
