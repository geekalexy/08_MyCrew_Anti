# [04] Hermes: 외부 채널 커뮤니케이션 통합 전략

**작성:** Luca | **Phase:** Hermes Integration | **Prime 참조 리뷰:** 4차

---

## 목표
텔레그램 봇을 시작으로 다양한 외부 메신저 채널과 아리 엔진을 연결하는 **통합 메시지 브릿지(Hermes)**를 구현한다.

---

## 핵심 설계

### 1. 어댑터 패턴으로 채널 추상화
```
외부 채널 (텔레그램, 슬랙, ...)
       ↓
[채널 어댑터] (telegram.js, slack.js ...)
       ↓
[Hermes 메시지 큐]
       ↓
[아리 엔진 / executor.js]
```
- 채널별 어댑터는 메시지를 표준 포맷 `{ from, text, chatId, platform }`으로 변환.
- 채널을 새로 추가해도 코어 엔진 변경 없이 어댑터만 추가.

### 2. 텔레그램 봇 토큰 관리
- 봇 토큰은 `KeyProvider`를 통해 관리 (`TELEGRAM_BOT_TOKEN`).
- 온보딩 이후, 설정 탭의 '워크스페이스' 섹션에서 토큰 등록.
- 등록 즉시 봇 초기화 (서버 재시작 불필요).

### 3. 아웃바운드 메시지
- 아리가 태스크를 완료하면 작업 요청자의 채팅방으로 결과를 Push.
- `io.emit('task:done')` → 서버에서 텔레그램 sendMessage 자동 트리거.
