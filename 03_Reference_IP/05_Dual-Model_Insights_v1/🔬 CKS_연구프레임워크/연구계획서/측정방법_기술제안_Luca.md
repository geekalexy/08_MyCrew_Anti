## 1. 정량적 지표: 요구사항 충족도 (Requirement Fulfillment Score)

**실행 방법 (LLM-as-a-Judge):**
실험에 참여한 모델(Gemini Pro, Claude Sonnet) 외의 독립적이고 성능이 검증된 모델
(예: GPT-4o 또는 설정이 격리된 별도의 Claude Opus 인스턴스)을 '절대 심사관'으로 배치합니다.
평가 기준(Rubric)을 프롬프트로 주입하고, Phase 1(초기 산출물)과 Phase 4(회고 후 최종 산출물)를
블라인드로 주어 1~10점 척도로 평가하게 합니다.

**평가 루브릭:** 컨텍스트 명확성, 템플릿 실용성, 워크플로우 완결성, 필요 스킬 정의의 정확성

**필요 프로그램/도구:**
- **TruLens / Ragas:** LLM 애플리케이션의 결과물을 정량적으로 평가하기 위한 오픈소스 프레임워크.
- **LangSmith:** 각 에이전트의 실행 체인(Chain)을 추적하고 결과물에 라벨링 및 채점을 수행하는 플랫폼.

---

## 2. 정량적 지표: 토큰 및 턴 효율성 (Token & Turn Efficiency)

**실행 방법:**
단일 에이전트가 동일한 품질의 기획안(컨텍스트, 템플릿, 워크플로우, 스킬)을 도출하기 위해
사용자와 주고받아야 하는 프롬프트 턴(Turn) 횟수와 전체 누적 토큰(Prompt + Completion Token)
사용량을 측정하여 다중 에이전트 교차 평가 방식과 비교합니다.

**필요 프로그램/도구:**
- **Helicone / Langfuse:** LLM API 호출에 대한 토큰 사용량, 지연 시간(Latency), 비용을 실시간으로 추적하고 대시보드화하는 오픈소스 옵저버빌리티(Observability) 도구.
- **API Meta-data Logging:** 복잡한 도구 없이 API 응답 객체의 `usage` 데이터를 터미널 로컬 환경에서 SQLite나 JSON 파일로 직접 적재하는 Python 스크립트.

---

## 3. 정성적 지표: 지식 동기화 지수 (Knowledge Synchronization Index)

**실행 방법:**
에이전트 간 텍스트 교환 과정에서 상대방의 아이디어를 얼마나 수용했는지
의미론적 유사도(Semantic Similarity)로 측정합니다.
예를 들어 Claude Sonnet이 제안한 특정 워크플로우 개념이 Gemini Pro의 최종 산출물에
얼마나 임베딩되었는지 벡터 거리를 계산합니다.

**필요 프로그램/도구:**
- **Sentence Transformers (Hugging Face):** 텍스트를 벡터로 변환하여 코사인 유사도(Cosine Similarity)를 계산하는 파이썬 라이브러리. 로컬 환경에서 가볍게 구동 가능.
- **ChromaDB / FAISS:** 생성된 임베딩 벡터를 저장하고 비교 검색하는 로컬 벡터 데이터베이스.

---

## 4. 정성적 지표: 창의성 및 아키텍처 확장성 발현 (Emergent Scalability)

**실행 방법 — 델타(Delta) 추출 분석:**
초기 프롬프트(Phase 1)에 명시되지 않았던 새로운 '스킬 정의'나 '워크플로우 단계'가
교차 평가 및 회고(Phase 4) 과정에서 자발적으로 도출된 건수를 카운트합니다.

**필요 프로그램/도구:**
- **DSPy:** 프롬프트 튜닝 및 LLM의 논리적 파이프라인 컴파일 프레임워크.
  구조화된 데이터(JSON) 출력을 강제하여 전/후 구조 비교를 용이하게 함.
- **JSON Diff 분석기:** 각 Phase의 산출물을 JSON 형태로 강제 출력(`response_format: {"type": "json_object"}`)하게 한 뒤,
  추가된 Key-Value 페어를 스크립트로 자동 추출 및 카운트.

---

## 5. 실험 자동화 및 로컬 구축 아키텍처

**오케스트레이션:** LangChain 또는 Microsoft AutoGen을 사용하여
터미널 환경에서 3개 모델(Opus, Sonnet, Pro)의 API를 묶고 역할을 부여.

**로깅 파이프라인:**
에이전트 간의 모든 메시지 기록, 타임스탬프, 소모 토큰량을 로컬 DB에 자동 기록하도록 설계하여,
실험 종료 후 평가지표 계산 스크립트가 해당 DB를 읽어 즉시 수치화된 논문용 테이블 데이터를
출력하도록 구성.
