## 1. 비즈니스 맥락 (비즈니스_맥락.md → 3.md) 비교

### Luca 특성
- **간결·핵심 지향**: 각 버전이 17~19줄 내로 압축. 헤드라인/서브섹션만으로 빠른 훑기 가능
- **v1→v3 피벗 정확**: "소셜링(인플루언서 매칭 플랫폼)" → "소시안(인스타DM 자동화 SaaS)"로 사업 정의 전환을 정확히 반영
- **구체적 기능 언급**: v3에서 다중 키워드, 리타겟팅 DM, 릴스 자동 업로드, 이벤트 추첨 등 핵심 기능 4종 명시
- **수익 모델 명시**: "Freemium 모델" 명기, 공식 채널 4종 나열

### Sonnet 특성
- **압도적인 정보 밀도**: v3 기준 130줄, 표(Table) 중심으로 가격 4플랜(Free/A/B/C)의 정확한 금액·할인율·기능 비교표를 완벽 수록
- **출처 명기 습관**: 매 버전마다 `> 출처: 웹사이트 + 가격페이지(브라우저 파싱) + 가이드북 + 블로그 종합`처럼 데이터 소스를 명시
- **v1 오분류 발생**: v1에서 "소셜 리스닝 & 인텔리전스"(B2B)로 사업 카테고리 자체를 오판 → v2에서 수정
- **크루 업무 가이드 내재**: 각 버전 마지막에 "AI 크루 업무 활용 가이드" 섹션을 별도로 두어 에이전트가 즉시 참조할 행동 지침 제공

### 🔑 핵심 차이
| 항목 | Luca | Sonnet |
|------|------|--------|
| 가격 상세 | "무료 체험 → 프로 플랜" (대략) | **4플랜 × 프로모/정상 가격 × 할인율 완비** |
| 기능 목록 수 | 핵심 4종 텍스트 나열 | **8종 + 가이드북 링크 테이블** |
| 타겟 페르소나 | "공구 셀러, 인플루언서" | **4가지 페르소나 × 각별 필요 기능 명시** |
| 문서 길이 | 17~19줄 | 91~130줄 |
| v1 정확도 | ✅ (약간 구형이나 오분류 없음) | ❌ v1에서 사업 카테고리 오판 |

## 2. 브랜드 가이드라인 비교

### Luca 특성
- **톤앤매너 3원칙** 일관 유지: Professional / Clear / Energetic
- **핵심 브랜드 키워드** 5~6개로 깔끔 정리
- v3에서 "무료 체험 & 프로 플랜", "매니챗보다 쉬운" 등 실무 소구 포인트 반영
- **짧고 빠른 합의용 문서**로 적합

### Sonnet 특성
- **공식 슬로건 정확 인용**: `"매니챗보다 쉬운 인스타DM 자동화, 소시안"` — 웹사이트 OG 태그에서 직접 확인
- **해시태그/SEO 키워드 체계**: 공식 해시태그 9종 + SEO 타겟 키워드 별도 나열
- **금지 사항 구체화**: "서비스명 오표기(소시언❌→소시안✅)", "가격 정보는 반드시 현행 프로모션 여부 병기" 등 실수 방지 규칙
- v1에서 "데이터로 사람을 이해하고, 커뮤니티를 이해한다"라는 잘못된 미션 도출 → v3에서 완벽 수정

### 🔑 핵심 차이
| 항목 | Luca | Sonnet |
|------|------|--------|
| 슬로건 | 고유 슬로건 미포함 (핵심 가치 서술) | **웹사이트 OG에서 공식 슬로건 직접 인용** |
| 해시태그 | 별도 섹션 없음 | **공식 9종 + SEO 키워드 세트** |
| 금지 사항 | 없음 | **4가지 구체적 금지 규칙** |
| 보이스 톤 정의 | 추상적 3원칙 | **실제 블로그 톤 분석 기반 (이모지 패턴, 말투 등)** |

## 3. 디자인 스타일 가이드 비교

### Luca 특성
- 컬러를 **이름(Vibrant Blue, Pure White)으로 서술** — 직관적이지만 HEX 코드 부재
- 타이포그래피를 **무드 중심으로 묘사**: "모던하고 가독성 뛰어난 네오그로테스크"
- v3에서 SUIT, Noto Sans KR 폰트 명시, 아이콘 시스템(기능별 픽토그램 매핑) 추가
- **커스토머 저니 시각화** 관점 포함 (실제 화면 캡처 블록 등)

### Sonnet 특성
- **HEX 코드 완비**: `#3B7BF5` (소시안 블루), `#0F1F4B` (다크 네이비), `#F5F7FA` (연회색) 등 7종 정확 매핑
- **텍스트 스케일 테이블**: Hero(40-48px/700) → Body(15-16px/400) 5단계 완비
- **레이아웃 수치 체계**: 1200px 컨테이너, 12 Column Grid, 거터 24px, 8px Base 단위
- v3에서 **CSS 직접 확인값** 반영: `line-height: 170%`, `letter-spacing: 0.32px`, 모바일 분기점 `900px`
- **컴포넌트 코드 블록** 제공: 버튼(Primary/Secondary), 카드의 구체적 CSS 속성값

### 🔑 핵심 차이
| 항목 | Luca | Sonnet |
|------|------|--------|
| 컬러 코드 | ❌ 이름 + 무드 서술 | ✅ **HEX 7종 완비** |
| 폰트 확인 | v3에서 SUIT 명시 | v3에서 **SUIT (CSS 직접 확인)** |
| 레이아웃 수치 | ❌ 없음 | ✅ **Grid/거터/분기점 전부** |
| 컴포넌트 코드 | ❌ 없음 | ✅ **버튼/카드 CSS 코드 블록** |
| SNS 콘텐츠 스타일 | ❌ 없음 | ✅ **블로그/인스타 포스팅 패턴 분석** |

## 🏆 최종 비교 결론 — 컨텍스트 UX

### Sonnet이 우세한 영역 (정보 깊이 · 구조화 · 정밀도)
1. **데이터 파싱 정밀도**: 가격/기능/CSS 등 하드 팩트를 브라우저·문서에서 직접 추출하여 기계적으로 정확
2. **테이블 중심 구조화**: 모든 정보를 표(Table)로 정리하여 AI 크루가 즉시 참조 가능
3. **버전 진화 폭**: v1 오분류 → v3 최종본까지의 개선 폭이 극적이며, 각 버전의 개선점이 명확히 기록됨
4. **크루 업무 가이드 내재**: 에이전트가 바로 따라쓸 "행동 지침"이 문서 안에 내장

### Luca가 우세한 영역 (간결성 · 실무 즉시성 · 안정성)
1. **간결함**: 17~19줄로 압축된 핵심 요약 — 인간 의사결정자(CEO)가 빠르게 검토하기에 최적
2. **초반 정확도 안정**: v1부터 사업 카테고리 오분류 없이 안정적으로 시작
3. **직관적 서술**: 기술적 수치보다 "무드(mood)" 중심 서술로 비전공자도 쉽게 이해
4. **기획 아이디어 내포**: 디자인 가이드에 "커스토머 저니 시각화" 같은 UX 인사이트 포함

### Prime 권장: 최적 병합 전략
> **Sonnet의 v3 데이터 + Luca의 간결한 요약 구조 = 최종 SSOT**

1. 비즈니스_맥락 → Sonnet v3 기반 + Luca의 "핵심 가치/타겟/해결책" 3줄 요약 헤더 추가
2. 브랜드_가이드라인 → Sonnet v3 기반 (슬로건·해시태그·금지사항 이미 완비)
3. 디자인_스타일_가이드 → Sonnet v3 기반 (HEX·CSS 확인값 이미 완비) + Luca의 아이콘 시스템 섹션 병합
