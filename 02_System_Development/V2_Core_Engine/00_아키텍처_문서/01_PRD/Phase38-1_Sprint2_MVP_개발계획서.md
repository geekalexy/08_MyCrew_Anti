## 📅 2. 타임라인 및 작업 명세 (Timeline)

### 🚀 [Step 1] 익스텐션 스켈레톤 구축 (오늘 야간 / Sprint 1)
가장 빠르게 구동되는 React 기반 크롬 익스텐션 개발 환경을 세팅합니다.

*   **작업 1-1. Vite + React 프로젝트 초기화**
    *   `02_System_Development/03_크롬_익스텐션` 폴더 신규 생성.
    *   Vite 기반의 React 프로젝트 셋업 (`npm create vite@latest`).
    *   `vite-plugin-crx` 라이브러리 연동으로 크롬 익스텐션 핫 리로드(Hot Reload) 환경 구축.
*   **작업 1-2. Manifest V3 및 Side Panel API 설정**
    *   `manifest.json` 파일 생성 및 `sidePanel` 퍼미션(권한) 추가.
    *   보안 정책 우회를 위해 `host_permissions`에 `http://localhost:3000/*` 및 `ws://localhost:3000/*` 명시적 허용.
*   **산출물**: 크롬 브라우저 우측에 빈 깡통 React 패널 띄우기 성공.

### 🔌 [Step 2] 통신 채널 뚫기 및 챗 UI 연동 (내일 오전 / Sprint 2-A)
사용자가 텍스트를 입력할 수 있는 기본적인 UI와 백엔드로 데이터를 쏘는 채널을 구축합니다.

*   **작업 2-1. Basic Chat UI 컴포넌트 개발**
    *   메시지 리스트 영역과 입력창(Input Box), 전송 버튼만 있는 깔끔하고 직관적인 UI 구현.
*   **작업 2-2. 확장 프로그램 -> 백엔드 통신 모듈 작성**
    *   로컬 마이크루 백엔드와의 통신을 위해 `socket.io-client` 설치 및 연동.
    *   (웹소켓 연결이 불안정할 경우를 대비하여, REST API 형태의 HTTP `fetch` 통신 채널도 함께 준비).

### 🧠 [Step 3] LLM 백엔드 릴레이 및 테스트 (내일 오후 / Sprint 2-B)
마이크루의 `server.js`(Ari 엔진) 쪽에 익스텐션의 질문을 받아주는 릴레이 엔드포인트를 붙입니다.

*   **작업 3-1. 백엔드(server.js) 엔드포인트 개방**
    *   익스텐션에서 날아온 텍스트를 수신할 API 라우트(예: `POST /api/extension/chat`) 또는 소켓 이벤트(`extension_chat_req`) 생성.
*   **작업 3-2. Ari 엔진 LLM 호출 연동**
    *   수신된 텍스트를 기존에 세팅된 LLM 어댑터(Gemini 등)에 넘겨주고, 받은 답변을 다시 익스텐션으로 반환.
*   **산출물**: 
    *   대표님이 크롬 확장 패널에서 "테스트" 입력.
    *   에이전트가 "네, 정상 수신했습니다!" 라고 답변하는 **엔드-투-엔드(End-to-End) 테스트 성공**.

## 🛠️ 3. 기술적 위험(Risk) 및 회피 전략
1.  **CORS 차단 에러**: 크롬 익스텐션은 태생적으로 타 도메인 접속을 차단하려 합니다. 
    *   *전략*: 백엔드 `server.js`의 CORS 설정에서 `chrome-extension://*` 오리진을 명시적으로 Allow 하도록 내일 바로 수정합니다.
2.  **React 빌드 충돌**: Vite 빌드 시 싱글 페이지 애플리케이션(SPA) 속성 때문에 익스텐션 구조와 충돌할 수 있습니다.
    *   *전략*: `@crxjs/vite-plugin` 플러그인을 사용하여 개발 및 빌드 환경의 복잡성을 단번에 우회합니다.

## 🏁 4. 완료 기준 (DoD, Definition of Done)
*   [ ] 브라우저 우측 상단의 익스텐션 아이콘을 누르면 사이드 패널이 열린다.
*   [ ] 사이드 패널의 입력창에 텍스트를 치고 전송하면, 백엔드 서버 로그에 해당 텍스트가 찍힌다.
*   [ ] 백엔드의 AI가 생성한 답변이 즉시 사이드 패널 화면에 말풍선으로 나타난다.
