# 터미널 복구 가이드 (비개발자용)

## 언제 필요한가요?
- cmux 패널(터미널 창)이 실수로 닫혔을 때
- AI 비서(Ari 등)가 응답하지 않을 때
- "터미널이 사라졌어요" 상황

---

## 1. AI 비서(Ari) 터미널이 닫혔을 때

### 복구 방법
1. cmux 화면에서 빈 공간에 **우클릭** 또는 상단 메뉴에서 **New Split** 선택
2. 새 터미널이 열리면 `claude` 입력 후 **Enter**
3. "Trust this folder?" 질문이 나오면 **Enter** (Yes)
4. AI 비서가 다시 시작됩니다

### 복구 후 상태
- AI 비서의 기억(역할, 규칙, 스킬): **그대로 유지됨**
- 방금 전 대화 내용: **사라짐** (새 대화로 시작)
- 텔레그램 연결: **자동 복구됨**

### 주의사항
- 텔레그램에서 메시지를 보내도 응답이 없으면 이 복구를 시도하세요
- 복구 후 텔레그램에서 `/start`를 한번 보내면 연결이 확인됩니다

---

## 2. 텔레그램 브릿지가 멈췄을 때

### 증상
- 텔레그램에서 메시지를 보내도 아무 반응이 없음
- AI 비서 터미널은 살아있는데 텔레그램만 안 됨

### 복구 방법
1. cmux에서 Bridge 터미널을 찾습니다 (python3 ... cmux_telegram_bridge.py 가 실행 중인 창)
2. **Ctrl + C**를 눌러 중지
3. **위 방향키 ↑**를 눌러 이전 명령어 불러오기
4. **Enter**를 눌러 다시 실행
5. 텔레그램에서 아무 메시지 보내서 확인

---

## 3. Paperclip 서버가 멈췄을 때

### 증상
- AI 회사 명령어(/iss, /inbox 등)가 작동하지 않음
- "에이전트 상태를 확인할 수 없습니다" 메시지

### 복구 방법
1. cmux에서 Paperclip 터미널을 찾습니다 (npx paperclipai 가 실행 중인 창)
2. **Ctrl + C**를 눌러 중지
3. `npx paperclipai onboard --yes` 입력 후 **Enter**
4. 서버가 다시 시작되면 정상

---

## 4. 전체가 안 될 때 (cmux 앱 자체를 껐을 때)

### 복구 방법
1. **cmux 앱**을 다시 실행합니다 (Dock 또는 Applications에서 클릭)
2. 기존 워크스페이스가 자동으로 복원됩니다
3. 각 터미널에서 프로그램이 중지되어 있을 수 있습니다:
   - AI 비서 터미널: `claude` 입력 → Enter
   - Bridge 터미널: **↑** → Enter (이전 명령 재실행)
   - Paperclip 터미널: `npx paperclipai onboard --yes` → Enter
4. 텔레그램에서 `/start` 보내서 연결 확인

---

## 5. 컴퓨터를 재시작했을 때

### 복구 순서
1. **cmux 앱** 실행
2. **Paperclip 터미널**에서: `npx paperclipai onboard --yes` → Enter
3. **Bridge 터미널**에서: 이전 python3 명령 재실행 (↑ → Enter)
4. **AI 비서 터미널**에서: `claude` → Enter
5. 텔레그램에서 `/start`로 확인

> 순서가 중요합니다: Paperclip 먼저 → Bridge → AI 비서

---

## 요약

| 상황 | 해결 |
|---|---|
| AI 비서 안 됨 | 새 터미널에서 `claude` |
| 텔레그램 안 됨 | Bridge 터미널에서 Ctrl+C → ↑ → Enter |
| 이슈/회사 안 됨 | Paperclip 터미널에서 재시작 |
| 전부 안 됨 | cmux 앱 재실행 → 복구 스크립트 실행 |
