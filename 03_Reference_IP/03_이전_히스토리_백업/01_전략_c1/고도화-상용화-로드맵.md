## Phase 3: 유료 전환 + 클라우드 이전 (3 ~ 6개월)

### 3-1. 유료 전환 검증
- [ ] 분양 고객 "돈 내고 계속 쓰겠다" 의사 확인
- [ ] Pro 티어 월 80~90만원 가격 검증
- [ ] 미확인 시 → 피봇 or 가격 조정

### 3-2. AWS Linux 클라우드 이전
- [ ] cmux → tmux 리팩토링 (1~2주)
- [ ] Playwright 헤드리스 브라우저 세팅
- [ ] Bridge 스크립트 tmux 호환 수정
- [ ] Tailscale 메쉬 네트워크 구성

### 3-3. 자체 웹 대시보드 MVP
- [ ] FastAPI + React 기반 대시보드
- [ ] 핵심 기능: 에이전트 현황, 이슈 목록, 댓글/지시, 승인 처리, 서류 업로드
- [ ] Paperclip은 백엔드 엔진으로만 동작 (고객 비노출)

### 3-4. DB 2단계: 클라우드 DB 전환
- [ ] Supabase(PostgreSQL) 구축
- [ ] User, Crew, Agent Profile, Instruction Ledger 동기화
- [ ] Session, Feedback Loop 테이블 추가
- [ ] 멀티 디바이스 맥락 접근 지원

### 3-5. 추상화 레이어 구축
- [ ] 사용자 → Luca(허브) → API 레이어 → Paperclip 구조 확립
- [ ] 엔진 교체 가능한 인터페이스 설계 (향후 CrewAI 등 대체 대비)
- [ ] 데이터 흐름: Luca가 해석 + DB 기록 동시 → Paperclip 실행 → 결과 동기화

### 3-6. 보안 레이어
- [ ] 고객간 데이터 격리 (DB 레벨)
- [ ] 민감 정보 익명화 토큰 처리 (교사/법률 등 도메인)
- [ ] 세션별 고유 ID 격리

## Phase 4: 에이전트 OS 완성 + SaaS 확장 (6개월 ~)

### 4-1. 에이전트 OS 인터페이스
- [ ] 하이브리드 워크스페이스: 좌측 채팅(Command Center) + 우측 칸반(Execution Board)
- [ ] Agent Identity 패널: 실시간 상태, 사용 도구, 토큰 비용 표시
- [ ] System Log 트레이: 서버 상태 + 네트워크 현황

### 4-2. 상태 기반 에이전트 제어
- [ ] Thought Streaming: 에이전트 추론 과정 실시간 노출
- [ ] Human-in-the-loop 게이트웨이 고도화 (현 텔레그램 승인 버튼 → 웹 통합)
- [ ] Context Sync: 채팅 지시 → 이슈 인스트럭션 실시간 보정

### 4-3. DB 3단계: 지능 레이어
- [ ] Vector DB (Pinecone/Milvus) 도입
- [ ] Experience DB: 에이전트 업무 지식 벡터화 (RAG)
- [ ] 자가 학습 루프: 피드백 → 에이전트 자동 개선 사이클 완성
- [ ] Global Goal Table: CEO 에이전트 최종 목표/제약 조건 관리

### 4-4. SaaS 확장
- [ ] 서버 스케일링 (고객 15~50명+)
- [ ] Free(설치형) + Pro(대행) 병행 운영
- [ ] Company Import/Export: 최적화된 팀 구성 패키징 배포
- [ ] 분양 고객 → 3개월차 수준 에이전트로 즉시 시작

### 4-5. AI 모델 의존성 헤지
- [ ] 멀티모델 어댑터 완성 (Claude, Gemini, 기타)
- [ ] 모델 가격/성능 변동 시 자동 전환 체계
- [ ] Anthropic API 정책 변경 대비 대체 경로 확보
