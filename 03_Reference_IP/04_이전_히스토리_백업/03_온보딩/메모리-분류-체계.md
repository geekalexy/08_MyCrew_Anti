# mycrew 메모리 분류 체계

## 목적
C2 분양 시 이관해야 할 메모리와 이관하면 안 되는 메모리를 명확히 분리.
운영 경험에서 쌓이는 노하우를 자산으로 축적.

---

## 1. 시스템 메모리 (분양 시 이관 O)

C2의 기본 동작에 필요한 설정 및 정책.

| 파일 | 내용 |
|---|---|
| 자비스 역할 | 능동적 에이전트 관리, 판단 기준 |
| cmux/tmux 사용법 | 패널 조작, 브라우저 명령 |
| 비노출 정책 | C1 존재 비노출, 시스템 구조 비공개 |
| 텔레그램 명령어 | /iss, /inbox, /newiss 등 |
| 응답 톤 | 보고체, 한국어 |
| 댓글 규칙 | 표식 없이, 즉시 수행, @멘션 활용 |

---

## 2. 스킬 메모리 (분양 시 이관 O)

에이전트별 노하우, 업무 패턴, 판단 사례. 운영하면서 축적되는 핵심 자산.

### 파일 구조
```
memory/skills/
  agent_ceo.md        — CEO 스킬
  agent_cmo.md        — CMO 스킬
  agent_cto.md        — CTO 스킬
  agent_pm.md         — PM 스킬
  agent_analyst.md    — 분석가 스킬
  jarvis_playbook.md  — 자비스 판단 노하우
```

### 기록 규칙
- 에이전트가 잘한 것: 상황 → 판단/행동 → 결과
- 에이전트가 못한 것: 실수 → 원인 → 개선
- 자비스 판단 사례: 보드 전달 vs 직접 처리 판단
- 협업 패턴: 어떤 업무에서 누가 누구와 잘 협업하는지

### 가치
- 분양 고객의 C2가 처음부터 숙련된 자비스로 동작
- 에이전트 운영 경험이 모든 고객에게 공유됨
- mycrew의 핵심 차별화 자산

---

## 3. 프로젝트 메모리 (분양 시 이관 X)

특정 회사의 업무 내용, 고객 정보, 이슈 데이터.

| 파일 | 내용 | 이관 |
|---|---|---|
| project_socian_* | 소시안 이슈, 팀 구성 | ❌ |
| project_dm_consent_* | 소시안 DM 수신동의 | ❌ |
| project_issue_cache | 소시안 이슈 캐시 | ❌ |
| Notion 연동 데이터 | 고객 업무 문서 | ❌ |

---

## 분양 시 이관 프로세스

```
1. 시스템 메모리 → 그대로 복사
2. 스킬 메모리 (skills/) → 그대로 복사
3. 프로젝트 메모리 → 복사하지 않음
4. 새 프로젝트 메모리 → bootstrap에서 자동 생성 (user_role 등)
```
