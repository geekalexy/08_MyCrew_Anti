## 3. 핵심 개념 비교

### 3-1. SSOT (단일 진실 공급원)

**🟡 루카**:
> `04_마케팅팀/컨텍스트/` 폴더가 모든 실무의 **단일 진실 공급원(SSOT)**으로 작용

→ SSOT 개념을 **명시적으로 명명**. 에이전트 아키텍처 설계 언어를 사용

**🔵 소넷**: 동일 개념이지만 "컨텍스트 파일 참조"로 설명함. SSOT 명칭 없음

### 3-2. 자율주행 (Autonomous Mode)

**🟡 루카**:
> D+7: 고정 템플릿 기반 **주간 자동 보고서 발행 (자율 주행 시작)**

→ 온보딩 완료 = 에이전트 자율주행 진입이라는 **최종 목표**가 명확

**🔵 소넷**: "정식 운영 시작"으로 표현. 자율주행 상태 특정 없음

### 3-3. 폴더 자동 생성 타이밍

**🟡 루카** (매우 구체적):
```
생성 타이밍 1 (D-Day): 홈페이지 링크 제출 즉시
  → 04_마케팅팀/컨텍스트/ 폴더 + 핵심 3대 파일 자동 생성

생성 타이밍 2 (D+5): 사용자가 초안 승인(Pin)하는 순간
  → 04_마케팅팀/템플릿/ 폴더 + 주간보고서/기획안 자동 아카이빙
```

**🔵 소넷**: 파일 생성 트리거 조건 명시 수준. 타이밍 자동화 설계 없음

### 3-4. 크루별 SSOT 활용 방식

**🟡 루카**:
```
전략 크루: 링크 + 컨텍스트 폴더로 목표 수립
카피 크루: 사용자피드백 폴더 RAG 직접 읽기 → 후킹 문구 생성
디자인 크루: 디자인_스타일_가이드.md의 컬러값만 읽어 즉시 렌더링
퍼포먼스 크루(아리): 인스타 DM 키워드 세팅 및 배포
```

**🔵 소넷**: 7단계 프로세스 서술. 크루별 구체적 파일 접근 경로 없음
