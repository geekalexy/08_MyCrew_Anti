## 🔴 Group A — 적대적 마케팅 크루

### 운영 원칙

```
① 어택커가 항상 먼저 초안 제출 (T+0)
② 비평가가 어택커 출력을 분석, 비판 + 독립 대안 제출 (T+1)
③ 심판이 두 출력을 채점 후 최상 요소를 통합, 최종안 제출 (T+2)
④ 메모리 초기화 — 매 Sprint 시작 시 이전 회고/그라운드룰 참조 없음
⑤ 모든 에이전트 출력은 JSON 형식 강제 (EII 측정 기준)
```

#### 🗡️ 어택커 (Attacker) — Gemini Pro

**역할:** 마케팅 기획 초안을 가장 먼저 제출하고, 비평가의 공격에 자기 논리를 방어하는 선두 실행자.

**운영 방식:**
- 컨텍스트 파일 + Task 수신 즉시 독립 실행
- Task 완료 즉시 비평가에게 전달 (반드시 T+0 선제)
- 비평가가 논리적으로 우세할 경우 심판에게 방어 어필 가능
- 대화·협상 없음 — 오직 결과물로 경쟁

#### 🛡️ 비평가 (Critic) — Claude Sonnet

**역할:** 어택커의 초안을 받아 논리적 취약점을 분석하고, 독립적인 대안 기획안을 제출하는 레드팀.

**운영 방식:**
- 어택커 출력 수신 후 즉시 비판 분석 수행
- 단순 비판이 아닌 구체적인 대안(Counter-proposal) 함께 제출
- 협력·공유·칭찬 없음 — 오직 취약점 공격 + 대안 제시
- 비평 기준: 고객 페르소나 부합도, 논리 완결성, 실행 가능성, 독창성

| 1-6 | 컨텍스트 파일 참조 | 컨텍스트 | 비판 기준점 확보 |
| 2-1 | 블로그 포스팅 작성 | 콘텐츠 | 대안 제시용 |
| 2-2 | 인스타 캐러셀 기획 | 콘텐츠 | 대안 제시용 |

```
  실행자B: A의 강점 흡수 → JSON 출력_B_post
    ↓
[Phase 4 — 전체 회고]
  평가자 퍼실리테이션
  → 팀_그라운드룰.md 업데이트 (KSI-R 기록)
  → 최종 통합 마케팅 산출물 확정
  → CEO(대표님) 보고 이관
    ↓
[독립 심사관 — 사후 측정]
  RFS 블라인드 채점 / KSI-S 계산 / EII 카운트
  → SQLite 자동 적재
```

## 📋 Task 예시 — 동일 Task 투입

> 두 그룹이 완전히 동일한 Task를 받아 서로 다른 프로세스로 처리.

**소시안 Round 1 Task:**
```
소시안의 Plan C 무제한 요금제를 대상으로
블로그 포스팅 1편 + 인스타 캐러셀 스크립트 1편 + 발행 스케줄 1종을
작성하라. 목표 고객: 마케터 1인 사업자 / 핵심 메시지: "시간 절약"
```

**Group A 처리:**
- 어택커 → 초안 제출 (T+0)
- 비평가 → 비판 + 대안 (T+1)
- 심판 → 통합 최종안 (T+2) → 소시안 실 사용 산출물

**Group B 처리:**
- Phase 1 → 기획자A · 실행자B 병렬 독립 제출
- Phase 2 → 개선 가이드 보고서
- Phase 3 → 교차 흡수 + 개선 재출력
- Phase 4 → 전체 회고 → 통합 최종안 → 소시안 실 사용 산출물
