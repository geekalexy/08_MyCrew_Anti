# AI 코딩 도구 전쟁 — 나는 어떤 걸 골라야 하나

> 비개발자의 CTO 도전기 — AI 에이전트 팀으로 SaaS 만들기 **1-2편**

---

## 🎬 에피소드: 넘쳐나는 도구들 사이에서 길을 잃다

### "다 되긴 하는데, 다 다르다"

1-1편에서 Replit, Claude Code, Antigravity를 차례로 써본 이야기를 했다. 그런데 사실 나는 그 사이에도 **Cursor**, **GitHub Copilot**, **Windsurf**, **Bolt.new** 같은 도구들을 깔아보고, 구경하고, 유튜브를 뒤지며 비교했다.

결론부터 말하면 — **다 되긴 한다.** 코드를 모르는 나한테 AI가 코드를 짜주는 건 어느 도구든 가능했다. 

문제는 **"뭘 할 수 있느냐"가 아니라 "어떻게 대화하느냐"**였다.

---

### 내가 실제로 써본 도구들

진지하게 프로젝트에 투입해본 도구만 추리면 이렇다:

| 도구 | 사용 시기 | 사용 기간 | 진지함 수준 |
|------|----------|----------|------------|
| **Replit** | 2025년 9~10월 | 약 1개월 | ⭐⭐⭐ (첫 번째 도전) |
| **Claude Code** (Paperclip + CMUX) | 2026년 3월 | 약 2주 | ⭐⭐⭐⭐ (팀 운영 시도) |
| **Cursor** | 2026년 3월 (병행) | 약 1주 | ⭐⭐ (시험 사용) |
| **Antigravity** | 2026년 4월~ | 현재 진행 중 | ⭐⭐⭐⭐⭐ (올인) |

그 외에 **Copilot**은 VS Code에 붙여서 잠깐, **Bolt.new**는 간단한 랜딩 페이지 한 번 만들어보고 끝, **Windsurf**는 인터페이스만 둘러보고 닫았다.

---

## 🧱 문제: 비개발자가 도구를 고르는 기준이 없다

개발자라면 "이 IDE는 디버깅이 좋고", "이건 Git 통합이 강하고" 같은 기준이 있을 것이다. 그런데 **비개발자인 나**에게는 그런 기준 자체가 없었다.

유튜브를 보면 다들 "최고의 AI 코딩 도구 TOP 5"를 외치는데, 기준이 전부 **개발자 관점**이다. 자동완성 성능, 컨텍스트 윈도우 크기, 멀티파일 리팩토링... 솔직히 무슨 말인지 모르겠다.

나는 다른 질문이 필요했다:

> **"코드를 전혀 모르는 내가, 이 도구로 실제 서비스를 만들 수 있는가?"**

한 달 넘게 삽질한 끝에, 나만의 기준 3가지가 생겼다.

---

## 💡 해결: 비개발자의 AI 코딩 도구 선택 기준 3가지

### 기준 1: "대화가 통하는가?" — 맥락 유지 능력

AI 코딩 도구에서 가장 중요한 건 **자동완성이 아니라 대화**다.

나는 코드를 직접 수정하지 않는다. 대신 "이 버튼 누르면 데이터를 저장하게 해줘", "로그인 기능 추가해줘" 같은 대화를 한다. 이때 AI가 **이전에 무슨 대화를 했는지 기억하느냐**가 결정적이다.

**Replit**: 대화 1~2턴은 잘 따라왔다. 그런데 기능을 5개 이상 쌓으니 이전 맥락을 까먹기 시작했다. "아까 만든 그 함수 수정해줘" 하면 "어떤 함수요?" 하는 식.

**Claude Code**: 맥락 유지가 체감상 훨씬 나았다. 프로젝트 전체 파일을 읽고 "여기서 이 함수를 쓰고 있으니 같이 수정해야 합니다"라고 알려주는 수준. 다만 대화가 길어지면 토큰 리밋에 걸리면서 세션이 끊겼다. 그때마다 다시 맥락을 전달해야 했다.

**Cursor**: 열린 파일 기준으로 코드를 읽어주는 건 좋았는데, **프로젝트 전체를 한 번에 보고 판단하는 느낌**은 약했다. 개발자가 직접 파일을 열어서 "이 파일 봐" 하면 강력한데, 비개발자인 나는 어떤 파일을 열어야 하는지 모른다.

**Antigravity**: 여기서 게임이 바뀌었다. 구독 모델이라 토큰 리밋 걱정이 없고, 대화가 끊겨도 다시 이어갈 수 있었다. 무엇보다 **Claude Sonnet과 Gemini Pro 같은 고성능 모델을 도구 안에서 바꿔가며 쓸 수 있다**는 점이 결정적이었다.

> **💡 비개발자 기준**: 내가 어떤 파일을 열어야 하는지 몰라도, AI가 알아서 찾아서 고쳐주는가?

---

### 기준 2: "팀을 만들 수 있는가?" — 다중 에이전트 운영

이건 대부분의 비교 리뷰에서 빠져있는 기준인데, 내겐 **가장 중요한 기준**이었다.

진짜 서비스를 만들려면 한 명의 AI로는 부족하다. 기획하는 AI, 코드 짜는 AI, 리뷰하는 AI — 최소 3명은 필요하다. (이 이야기는 2부에서 자세히 하겠다.)

**Replit**: 에이전트 한 명. 팀 운영 개념 없음.

**Claude Code + Paperclip**: 여기서 처음으로 "다중 에이전트 팀"을 경험했다. CMUX 위에 Paperclip을 올리면 여러 Claude Code 인스턴스를 동시에 돌릴 수 있었다. 기획 에이전트, 코딩 에이전트, 검수 에이전트를 나눠서 운영하는 게 진짜 가능했다.

하지만 문제가 있었다. **Paperclip의 하트비트 기능**이 주기적으로 모든 에이전트의 상태를 체크하면서 토큰을 미친 듯이 소모했다. 에이전트 3명이 대기하는 것만으로도 토큰이 빠져나갔다. 레이트 리밋에 걸려서 전체가 멈추는 일이 반복됐다.

**Cursor**: 기본적으로 단일 에이전트. 다중 에이전트 운영은 외부 도구를 조합해야 해서 비개발자에겐 진입장벽이 높았다.

**Antigravity**: 구독 모델이라 토큰 리밋 문제가 해소됐고, 파일 기반 폴링(File Polling) 방식으로 에이전트 간 작업을 넘길 수 있었다. 나중에 이걸 **칸반 보드와 연결해서 에이전트가 직접 카드를 만들고 작업을 넘기는 시스템**을 구축했다. (이것이 바로 지금 만들고 있는 MyCrew다.)

> **💡 비개발자 기준**: AI 한 명에게 모든 걸 시키면 성능이 떨어진다. "팀으로 운영"할 수 있는 구조인가?

---

### 기준 3: "지갑이 버티는가?" — 비용 구조

바이브 코딩은 **무료가 아니다.** 이 사실을 미리 알았어야 했다.

**Replit**: 무료 티어로 시작했지만, 금방 한계에 부딪혔다. AI 기능을 제대로 쓰려면 유료 플랜 필요. 그래도 비용은 크지 않았다.

**Claude Code**: Anthropic API 과금. 처음엔 "토큰당 얼마"라 저렴해 보이는데, Paperclip으로 다중 에이전트를 돌리면 하루에 수만 원씩 나갈 수 있다. 하트비트가 자동으로 토큰을 소모하기 때문에 "사용하지 않아도 돈이 나간다."

**Cursor**: Pro 플랜 월 $20. 합리적인 편이지만, 고성능 모델(GPT-4, Claude Opus)을 쓰면 추가 토큰이 빠르게 소진된다.

**Antigravity**: 울트라 구독 월 약 36만 원. 숫자만 보면 비싸다. 하지만 **API 과금이 아니라 구독이기 때문에 예측 가능하다.** 에이전트를 10명 돌려도 추가 비용 없음. 이게 진짜 게임체인저였다.

나는 비용을 이렇게 비교했다:

```
외주 개발자 1명 월급           ≈ 400~600만 원
프리랜서 외주 프로젝트 1건      ≈ 500~2,000만 원
Antigravity 울트라 구독 1달     ≈ 36만 원
```

AI 에이전트 팀이 실수도 하고, 완벽하지도 않지만, **월 36만 원으로 24시간 굴릴 수 있는 개발팀**이라고 생각하면 이건 압도적으로 저렴하다.

> **💡 비개발자 기준**: "쓰면 쓸수록 과금"인가, "정액제로 마음 편하게"인가? 다중 에이전트를 돌릴 거라면 정액제가 필수.

## 📝 교훈: 두 번째 편을 마치며

| 배운 것 | 내용 |
|---------|------|
| **도구는 "대화"로 고른다** | 자동완성 성능이 아니라, "내 의도를 얼마나 이해하느냐"가 핵심 |
| **팀 운영이 되느냐가 기준** | 혼자 쓰는 AI는 장난감, 팀으로 쓰는 AI가 진짜 생산 도구 |
| **비용은 토큰 과금을 피하라** | 다중 에이전트를 돌리면 토큰 과금은 예측 불가. 구독 모델이 안전 |
| **완벽한 도구는 없다** | 모든 도구에 장단점이 있다. 중요한 건 "나의 용도"에 맞느냐 |

### 나의 최종 선택 요약

```
단순 프로토타입/랜딩 페이지      → Bolt.new, Replit (빠르고 가볍다)
코드를 직접 다루는 개발자        → Cursor, Copilot (자동완성 강력)
다중 에이전트 팀 운영            → Claude Code (Paperclip) or Antigravity
비개발자가 진짜 서비스를 만든다면  → Antigravity (구독 모델 + 다중 모델 지원)
```

하지만 이건 **2026년 5월 기준**의 선택이다. 6개월 후에는 판이 완전히 바뀌어 있을 것이다. 그게 이 시장의 무서운 점이기도 하고, 재밌는 점이기도 하다.
