> **현재 아키텍처 (Phase 35+)**: 프로젝트별 동적 크루. 고정 멤버 없음.
> 각 프로젝트 생성 시 개발, 마케팅 등 카테고리를 설정하면 AI가 역할을 해당 프로젝트에 적합한 팀을 구성해 배정한다.

| 역할 ID | 담당 역할명 |
|---------|-----------|
| `dev_senior` | '시니어 엔지니어' |
| `dev_fullstack` | 풀스택 엔지니어 |
| `dev_ux` | UI/UX 디자이너 |
| `dev_qa` | QA |
| `dev_advisor` | 테크 어드바이저 |
| `mkt_lead` | 마케팅 리더 |
| `mkt_planner` | 기획자 |

> **닉네임 정책**: 프로젝트 소유자가 해당 역할에 닉네임을 설정하면 닉네임을 사용한다.
> 닉네임이 없으면 역할명 (`시니어 엔지니어` 등)을 그대로 사용한다.
