# 🚀 ［B2B 상용화 대비］ MyCrew 워크스페이스 UI/UX 용어 및 레이아웃 전환 기획안

**문서: 상용화_대비_UIUX_개편안_Luca.md**
**목적**: 연구실(Lab) 느낌이 강한 CKS 실험용 컴포넌트들을 실제 고객(B2B/B2C SaaS 사용자)에게 익숙하고 세련된 직관적 UI로 변환합니다.

---

## 1. 네이밍(Terminology) 패치: 연구실에서 오피스로
고객은 '실험'을 하러 오는 것이 아니라, '내 일을 대신해 줄 똑똑한 팀'을 꾸리러 옵니다.

| 현재 (실험용 명칭) | 상용화 전환 (SaaS 프로덕트 명칭) | UX 의도 및 설명 |
| :--- | :--- | :--- |
| **Team A (협력적 CKS)** | **크리에이티브 스쿼드 (Synergy Mode)** | 고객이 "창의적인 아이디어 기획(마케팅, 디자인, 브레인스토밍)"에 투입하는 모드 |
| **Team B (적대적 대조군)** | **QA/검증 특화 스쿼드 (Cross-Check Mode)** | 고객이 "보수적이고 꼼꼼한 검수가 필요한 작업"에 투입하는 모드. (부정적 어감 제거) |
| **독립 심사관 (ARI)** | **비서 (Secretary)** | 메인 화면 최상단에서 고객과의 주 채팅을 담당하며, 각 팀으로 오더를 배분하는 오케스트레이터 |
| **CKS 분석 메트릭** | **팀 생산성 지표 (Team ROI)** | 이 팀을 굴려서 실제 사람 대비 얼마나 시간/돈(토큰)을 아꼈는지 보여주는 지표 |
| **팀 그라운드 룰** | **팀 컬처 핏 (Team Culture)** | AI가 지켜야 할 회사만의 톤앤매너와 업무 규칙 가이드북 |

## 2. OrgView (팀 관리/조직도) UI 개편 방향

### 탭 1: [Squads (내 스쿼드)] - 기존 조직도 탭
*   **에이전트 표기법**: 복잡한 괄호를 제거하고 간결한 **`이름-직함`** 포맷으로 통일합니다. 
    *   **ARI-비서** (메인 라우터)
    *   **LUMI-디자이너**
    *   **PICO-기획자**
    *   **NOVA-마케터**
    *   **OLLIE-디렉터**

### 탭 2: [Analytics & Culture (성과 및 문화)] - 기존 관리/분석 탭
*   **Token ROI (비용 절감액 환산)**: 총 토큰 소모량을 보여주되, 이를 *"이번 달 아낀 인간의 업무 시간: 32시간"* 처럼 실제 고객이 가치를 체감할 수 있는 ROI(투자 대비 수익) 지표 시각화로 바꿉니다.
*   **Culture Fit (성장하는 팀)**: CKS 그라운드 룰이 몇 건 등록되었냐가 아니라, *"우리 팀이 최근 학습한 업무 지침: ['MZ세대 톤앤매너 적용법']"* 형태로 노출하여, AI 팀이 유저의 입맛에 맞게 커스텀 진화하고 있음을 시각화합니다.

## 3. 태스크 라우팅 기획 (사용자 오더 할당 방식)

두 팀(A팀, B팀)의 공존과 선택적 실행에 대한 상용화 아키텍처입니다. **고객에게 2가지 오더 방식(Mode)을 전부 제공**하는 것이 핵심입니다.

*   **Mode 1: "A/B 옵션 채택" 모드 (대표님 제안 구조)**
    *   **동작 로직**: 사용자가 메인 칸반의 **'새 작업 만들기(카드 생성)'** 창에 "신제품 인스타 기획해 줘"라고 오더를 내립니다. 
    *   이때 **비서(ARI / 라우터)**가 채팅이나 프롬프트로 사용자에게 *"더 향상된 결과물을 도출하기 위해 2팀이 같은 지시를 수행하고 보고합니다"* 라고 안내하며, 내부적으로 오더를 복제하여 `Team A`와 `Team B` 양쪽으로 동시에 쏩니다.
    *   **사용자 경험(UX)**: 5분 뒤 유저에게 "A팀 산출물" vs "B팀 산출물" 2개의 시안이 도착합니다. 유저가 마음에 드는 **하나를 채택(Pick)**하면, 선택받은 팀의 `팀 컬처 핏(Team Culture)`만 긍정적으로 업데이트되며 진화합니다. (가장 고급스러운 SaaS 형태)
*   **Mode 2: "지정 스쿼드 다이렉트 할당" 모드**
    *   **동작 로직**: 사용자가 이미 두 팀의 성향을 파악한 상태입니다. "이번 건은 꼼꼼하고 보수적인 검수 위주의 작업이야"라고 판단하면, 칸반 보드에서 **Team B 보드에 직접 카드를 생성(Drag & Drop)**합니다.
    *   **사용자 경험(UX)**: Team B만 단독으로 핑(Ping)이 돌아가고 결과물을 냅니다. (서버 토큰 비용 절감을 원하는 사용자를 위한 선택적 단독 구동)

## 3. 사이드바 (LNB) 문서 트리 구조 최적화
워크스페이스 좌측 네비게이션에서 '회사 내규 및 가이드'를 열람할 수 있는 구조를 확립합니다.

*   **최상단 메뉴**: `팀.md` 
    *   (기존) `페이퍼클립.md`는 폐기. 대신 실제 AI 코어 메뉴얼인 **`클로드.md` (혹은 AI 코어 연동 백서)**로 대체합니다.
    *   (신규) 하위 1레이어에 **`MYCREW.md`** 섹션을 정식으로 추가합니다.
*   **UX 동작 시나리오**: 좌측 트리에서 `MYCREW.md`를 클릭 시, 우측 메인 뷰(Viewer)에서 "MyCrew 팀의 비전, 컨텐츠 바디, 그리고 플랫폼 사용 가이드" 전체 내용을 읽을 수 있도록 풍부한 마크다운 문서 형태로 노출합니다.

---

## 4. 요약: 우리가 얻게 되는 것
이렇게 껍데기(UI 용어와 LNB 구조)만 살짝 포장해 두어도, 
백엔드에서는 완전히 동일하게 **"적대적 모델 VS 협력적 CKS 모델"의 성능 벤치마크 데이터**가 그대로 쌓이면서도,
사용자에게는 완벽하게 **"똑똑하고 스스로 진화하는 가상 직원(SaaS) 관리 플랫폼"**으로 셀링(Selling)할 수 있게 됩니다.
