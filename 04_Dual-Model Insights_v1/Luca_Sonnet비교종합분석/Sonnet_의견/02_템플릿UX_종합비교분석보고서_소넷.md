## 4. 카드 UX 비교

### 4-1. 카드 헤더

| 항목 | 🟡 루카 | 🔵 소넷 |
|------|--------|--------|
| 타이틀 | `🔥 기획 퀄리티 90%를 결정하는 [10초 셋업]` | `📦 크루 자료 세팅 ⚡ 결과물 만족도의 90%` |
| 서브타이틀 | `"타자는 안 치셔도 됩니다"` | `"필수 항목만 채워도 시작 가능"` |
| 임팩트 | ✅✅ 매우 강렬 | ✅ 설명적 |

### 4-2. 입력 블록 구성

| 블록 | 🟡 루카 | 🔵 소넷 |
|------|--------|--------|
| URL 입력 | URL 붙여넣기 폼 1개 | 채널별 URL 3개 분리 |
| 폴더 연동 | ✅ **체크박스 기본 ON** (`사용자_피드백` 포함) | 파일 업로드 섹션 A~D |
| 사이드바 핀 | ✅ **"즐겨찾기에 핀 꽂아주세요" CTA** | ❌ 없음 |
| 충족도 표시 | ❌ | ✅ ████ 85% 게이지 |
| 업데이트 주기 | ❌ | ✅ 2주/1달/이벤트시 선택 |

### 4-3. 루카만의 독창적 UX 아이디어

1. **폴더 체크박스 기본 ON** — 사용자가 "체크된 걸 해제"할 에너지 없이 그냥 제출
2. **"사이드바 즐겨찾기 핀"** 유도 — 폴더 재접근 습관 형성
3. **"10초 셋업" 네이밍** — 심리적 부담 극도로 낮춤

## 5. 최종안 권고

```
채팅 시나리오:
  → 루카의 핵심 1케이스 (강렬한 첫 멘트 + 타자 제로 안심)
  → 소넷의 3케이스 보완 (거부·재방문 대응)
  → 루카의 완료 후 메시지 (재이용 동기) + 소넷의 분석 결과 공유 합산

카드 UX:
  → 루카의 헤더 카피 ("10초 셋업") 채택
  → 루카의 폴더 체크박스 기본 ON + 사이드바 핀 유도 채택
  → 소넷의 충족도 게이지 + 업데이트 주기 설정 추가
```

**결론: 카드 UX는 루카 60% + 소넷 40% 병합 권장**
