## 1. 💼 비즈니스 모델 연계: 듀얼 티어(Dual-Tier) 배포 전략

마이크루의 아키텍처는 백엔드(Node), 프론트엔드(Vite)가 분리된 MSA(Microservices Architecture) 형태입니다. 공식 NotebookLM API를 연동하여 비용 방어와 유료 전환을 극대화하는 **Two-Track 전략**을 채택합니다.

### 🟢 Tier 1: 100% 로컬 프리미엄 체험판 (Free/Freemium)
- **배포 형태**: 사용자가 데스크톱 앱(또는 통합 패키지 스크립트)을 ダウン로드 받아 자신의 PC에 직접 구동.
- **비용 구조 (서버비 $0)**: 
  - 사용자의 PC 자원(CPU/RAM)을 중앙 서버 대신 100% 활용.
  - **BYOK (Bring Your Own Key)**: 구동 시 앱 내에 사용자 본인의 Gemini/Claude API 키를 등록하게 하여 AI 호출 토큰 요금을 마이크루 본사에 부담시키지 않음.
- **타겟 효과**: 인프라 적자 없이 전 세계 수만 명의 유저가 강력한 7인 AI 요원을 무료로 체험하며 '락인(Lock-in)' 효과를 체감.

### 💎 Tier 2: 클라우드 싱크 (Premium Cloud Sync)
- **배포 형태**: 로컬에서 체험 후, "모바일 활용" 및 "팀 단위 협업"이 필요한 고객군을 대상으로 구독료(예: 월 $30~$50)를 받고 중앙 클라우드 서버 접근 권한(OAuth) 부여.
- **제공 가치**:
  1. **멀티디바이스 동기화**: 카페에서 스마트폰으로 접속해도 칸반 보드 및 대본 상태 실시간 확인.
  2. **무제한 토큰 풀(Pool)**: 유저 개별 API 키 체제에서 벗어나, 엔터프라이즈 환경의 중앙 AI 토큰 자유 활용 기능 제공 (마진 확보).
  3. **팀 워크스페이스**: 직원들과 에이전트 작업 현황 공유 및 멀티플레이 부여.

## 2. ⚡ 공식 NotebookLM API 전환 전략

과거 봇 탐지 회피를 위해 로컬 우회 터널(BYOMCP)을 사용했으나, 공식 구글 클라우드 NotebookLM Enterprise API 출시로 인해 클라우드 네이티브 아키텍처로 전면 개편되었습니다.

### [기존 방식의 문제점 (폐기)]
- Puppeteer(헤드리스 브라우저) 구동에 따른 막대한 자원 낭비 및 관리 복잡성.
- 구글의 Bot Detection으로 인한 밴(Ban) 위험.

### [새로운 방식: 공식 API 직접 호출]
1. 유저는 대시보드를 통해 Google OAuth 2.0으로 로그인합니다.
2. 획득한 OAuth Bearer 토큰을 활용하여 아리 엔진(Ari Engine)이 Google Cloud의 NotebookLM API 엔드포인트(`notebooks.create`, `content.sources` 등)를 직접 호출합니다.
3. 클라이언트 브라우저 자동화가 완전히 제거되어, 서버 자원 낭비율이 0%로 수렴합니다.
4. **결과:** 안정성 100%, 처리 속도 대폭 향상, 멀티 플랫폼(모바일 포함) 완벽 대응 가능.

## 3. 💿 통합 설치 시퀀스 (Integration Sequence)

현재 구현된 `start_mycrew.command` 스크립트를 통해 로컬 유저도 1-Click 설치 및 백그라운드 구동 경험을 제공합니다.

```bash
# 구동 2단계 (Script Architecture)
1. 아리 엔진 부팅 (Port 4000) - AI 에이전트 두뇌 및 메인 API
2. 웹 대시보드 부팅 (Port 5173) - UI 프론트엔드 연결
```

- 사용자는 여러 개의 터미널을 띄우거나 설정을 만질 필요 없이, 실행 파일 클릭 한 번으로 위 2가지 마이크로 서비스가 백그라운드에서 상호 연결되어 즉각적인 AI 활용이 가능하도록 모듈화 완료.
