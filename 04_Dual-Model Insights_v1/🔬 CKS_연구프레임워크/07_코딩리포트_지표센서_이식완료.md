# [적용 완료 리포트] CKS 메트릭 센서(Sensor) 이식 내역 

> **분류**: 개발 현황 보고
> **단계**: Phase 4 - 실데이터 연동 (Data Pipeline Active)
> **대상**: MyCrew 팀원 및 시스템 아키텍트 (코드 리뷰어용)

이 문서는 CKS 연구 지표의 100% 자동화를 위해, 프론트엔드 대시보드뿐만 아니라 **엔진 내부 깊숙한 곳에 메트릭 센서(Metrics Sensor)를 심은 내역**을 개발팀에게 공유하기 위해 작성되었습니다.

---

## 🚀 1. IRC (반복 수정 횟수) 추적기 이식
* **의미**: 사용자가 태스크를 승인(Done)하지 않고, 다시 "In-Progress"로 돌려보낼 때마다 카운트.
* **타겟 파일**:
  * `database.js` (`incrementCksIrc` 메소드 추가)
  * `server.js` (`PATCH /api/tasks/:id/rework` 메소드)
* **어떻게 동작하는가?**:
  사용자(대표님)가 대시보드에서 `재작업 지시(Rework)` 버튼을 누르면 서버의 `/api/tasks/:id/rework`가 호출됩니다. 상태값이 Review에서 In_Progress로 롤백될 때, 새롭게 추가된 `dbManager.incrementCksIrc(sid)` 함수가 발동하여 해당 태스크의 `irc_count`를 DB에서 즉시 누적시킵니다.

## 🚀 2. TEI (토큰 사용량 누적) 센서 이식
* **의미**: 각 에이전트가 통신할 때 구글/앤스로픽 API 단에서 발생한 단어 토큰 사용량을 누적 로깅.
* **타겟 파일**:
  * `ai-engine/adapters/geminiAdapter.js`
  * `ai-engine/adapters/anthropicAdapter.js`
  * `ai-engine/executor.js`
* **어떻게 동작하는가?**:
  기존에는 어댑터가 `{ text: "응답" }`만 주었습니다. 이번 업데이트로 `usageMetadata.totalTokenCount`를 가로채어 `{ text: "응답", tokenUsage: 1042 }`를 리턴하도록 패치했습니다.
  최종적으로 `executor.js` 실행 완료 직후, `dbManager.accumulateCksTokens()`를 호출하여 태스크의 `tei_tokens` 값을 높입니다. 다중 에이전트가 여러 번 생각해도 계속 토큰이 누적 계상됩니다.

## 🚀 3. 정성/정량 메타 지표 프롬프팅 (KSI-R, KSI-S, HER, EII)
* **의미**: 판관(Opus-as-a-Judge) 및 Brain인 프라임/넥서스 에이전트가 리포트를 제출할 때, 스스로 실험 지표를 계산하여 뱉어냅니다.
* **타겟 파일**:
  * `ai-engine/adapters/antigravityAdapter.js`
  * `ai-engine/executor.js`
* **어떻게 동작하는가?**:
  기존의 Anti-Bridge "JSON 형태로 결과를 출력해라"는 지시어에 아래와 같은 강력한 예시 프롬프트를 덧붙였습니다.
  ```json
  "instructions": "리포트를 출력할 때 반드시 CKS 프레임워크 연구를 위한 _meta 지표를 평가하여 포함할 것.
  포맷 예시:
  {
    \"text\": \"...최종 리포트 결과...\",
    \"_meta\": {
      \"ksi_r\": 92,     // 룰 생존율
      \"ksi_s\": 0.85,   // 문맥 동기화 수준
      \"her\": 0,        // 차단된 오류 건수
      \"eii\": 4.5       // 창의적 파생 점수
    }
  }"
  ```
  에이전트가 위 JSON 포맷에 맞게 뿜어내면, `executor.js`가 정규식으로 `_meta` 블록을 파싱하고 `dbManager.updateCksEvalMetrics()`를 호출하여 DB의 해당 필드를 안전하게 덮어씁니다(UPSERT).
