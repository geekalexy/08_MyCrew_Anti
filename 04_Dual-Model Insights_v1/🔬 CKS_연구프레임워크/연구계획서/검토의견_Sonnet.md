## 🔴 수정 필수 — 치명적 문제 (3개)

---

### ① 평가자 순환 편향 (Circular Bias)

Section 5에서 "LLM-as-a-Judge" 기법으로 요구사항 충족도를 측정하겠다고 했는데:

```
문제:
  Claude Opus = 실험 내 평가 에이전트 (Section 3)
  Claude Opus = 메트릭 측정 Judge (Section 5)?

→ 자신의 결과물을 자신이 채점하는 순환 편향
→ 심사위원이 가장 먼저 지적할 치명적 문제
```

**수정안:** 메트릭 측정용 Judge는 실험에 참여하지 않은 **별도 모델 또는 인간 평가자**로 분리 명시

---

### ② Phase 4 메타 프롬프팅의 오염 가능성 미설명

> "실험의 목적과 구조를 전체 에이전트에게 공개(Meta-prompting)"

```
예상 심사위원 질문:
"실험 구조를 에이전트에게 알려주면
 에이전트가 '좋은 연구 결과를 만들어야 한다'는
 편향된 행동을 하는 것 아닌가?"
```

**수정안:** Phase 4 메타 프롬프팅의 이론적 근거 추가하거나,
메타 프롬프팅 유/무를 별도 비교 조건으로 설정

---

### ③ KSI 공식 미정의

"빈도 분석"이라고만 서술 — 수치화 불가능한 정성 지표로 분류될 위험

**수정안 (공식 추가):**
```
KSI = ( f(B→A) + f(A→B) ) ÷ 2

f(B→A): Phase 3 이후 B 출력에서 A의 Phase 1 고유 개념 등장 비율
f(A→B): Phase 3 이후 A 출력에서 B의 Phase 1 고유 개념 등장 비율

KSI > 0.5: 유의미한 지식 동기화
KSI → 0:  교차 평가가 실질적 영향 없음
```
