# Phase 37 자율 파이프라인 버그 분석 및 해결 방안
**작성일:** 2026-05-05
**상태:** 🚨 버그 분석 완료 / 해결 방안 대기 중 (명시적 승인 전 코드 수정 금지)

## 1. 문제 정의 (CEO 지적 사항)
1. **결과물 누락 ("작업이 완료되었습니다." 빈 깡통 응답)**: 풀스택 엔지니어 등 실무자가 산출물을 전혀 만들지 않고 "작업이 완료되었습니다"라는 코멘트만 남김.
2. **어드바이저의 역할 위반 및 환각 코드 리뷰**: 마지막에 한 번만 리뷰해야 할 어드바이저가 초기 PRD 작성(실무)을 수행함. 또한 디스크에 전달되지도 않은 코드를 뇌피셜(환각)로 리뷰하는 중대한 오류가 발생함.
3. **잘못된 에이전트 할당**: 백엔드 업무(미니앱 백엔드 초기 환경 구축)가 QA 담당(`dev_qa`: 3 Flash)에게 엉뚱하게 할당됨.
4. **파일 전달 및 카드 링크 실패**: 실무자가 디스크에 저장한 파일 내용이 다음 작업자에게 주입되지 않음.

---

## 2. 근본 원인 분석 (Root Cause Analysis)

### A. 결과물 증발 원인 (`executor.js` XML 파서 논리 오류)
- 에이전트는 `<review_request>`, `<next_sprint>`, `<file_operations>` 태그 안에만 결과를 출력함.
- `executor.js`의 파서가 태그를 추출한 뒤 원문 텍스트에서 정규식으로 삭제해버리면서 남은 텍스트가 `""`(빈 문자열)이 됨.
- 엔진은 빈 텍스트를 감지하면 디폴트 텍스트인 `"작업이 완료되었습니다."`를 강제로 덮어씌움.

### B. 어드바이저 환각 리뷰 및 파일 전달 실패 (Card Link 구조적 한계)
- 현재 시스템의 카드 링크(`#1`, `#4`)는 DB에 저장된 **댓글(C)과 첨부파일(F) 데이터만** 가져올 수 있음.
- 실무자(`dev_fullstack`)가 `<file_operations>` 도구를 써서 디스크(예: `04_IO/outputs/`)에 코드를 쓰더라도, DB 첨부파일로 등록되지 않기 때문에 다음 작업자의 컨텍스트에 전달되지 않음.
- 어드바이저는 코드를 보지 못한 상태에서 타겟 카드 제목만 보고, 자신이 무엇을 리뷰해야 하는지 착각하여 일반적인 모범 사례를 지어내는 환각 리뷰를 작성함.

### C. 잘못된 에이전트 할당 및 어드바이저 역할 위반 (Zero-Base 프롬프트)
- `zeroConfigService.js` 내부 프롬프트에서 초기 기획 카드를 `dev_advisor`로 고정 배정하도록 하드코딩되어 있음. (이로 인해 어드바이저가 1차 실무에 투입됨)
- 이후 AI가 `<next_sprint>`로 자율 생성할 때 담당자 배정 제약이 약해 QA를 임의로 배정하는 문제가 발생함.
- `executor.js` 내 하드코딩 폴백 맵(`_HARDCODED_FALLBACK`)에 `dev_qa`의 디폴트 모델이 `Opus`로 잘못 연결되어 있었음.

---

## 3. 해결 방안 (제안)
> ⚠️ 아래 조치는 CEO의 명시적 승인(컨펌) 후 진행 예정입니다.

1. **XML 파서 수정 (빈 텍스트 방지)**: `executor.js`에서 태그를 삭제하지 않고 `[Review Request ➔ 누구]` 형태의 마크다운 텍스트로 변환하여 본문에 남겨 유저가 볼 수 있게 처리.
2. **디스크 파일 자동 주입 (환각 차단)**: 작업자가 `<file_operations>`로 생성/수정한 파일의 원문 코드를 파싱하여, 다음 사람에게 넘어가는 핑퐁 메시지(`review_request.message`)에 강제로 덧붙임. (리뷰어에게 실제 코드를 주입)
3. **역할 할당 정상화**: 
   - 제로베이스 스캐폴딩 프롬프트에서 1차 실무(PRD 작성)를 `dev_fullstack` 또는 `mkt_planner`로 변경.
   - 프롬프트에 "dev_advisor는 절대 실무를 하지 않고 마지막에 리뷰만 한다"는 규칙 명시.
4. **폴백 모델 테이블 정정**: `dev_qa`가 `Opus`가 아닌 가벼운 `Gemini 3 Flash`를 사용하도록 정정.
