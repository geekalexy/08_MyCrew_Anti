# [07] Phase 20: 온보딩 기획 고도화 (B2B SaaS)

**작성:** Luca | **Phase:** 20 | **Prime 참조 리뷰:** 7차

---

## 목표
단순한 초기 설정 화면을 넘어, B2B 고객이 **"나를 위한 플랫폼"**이라고 느낄 수 있는 팀 중심의 브랜디드 온보딩 경험을 제공한다.

---

## 핵심 설계

### 1. 팀명(Workspace Name) 기반 고유 URL 브랜딩
- STEP 1에서 팀명 입력 → 추후 `mycrew.ai/{teamName}` URL 발급 예정을 안내.
- 입력 시 실시간으로 URL 미리보기 (`→ mycrew.ai/socian`) 제공.
- 팀명은 `workspace_name` 설정키로 DB 저장.

### 2. 이중 지출 방지 구독 모드 분기
**기획 배경:** 사용자가 Gemini Advanced 등을 개인적으로 이미 구독 중인 경우, API 키 비용을 추가로 부담시키는 것은 모순.
```
[구독 인증 모드]  - 계정 이메일 입력 → Test 검증
[API Key 모드]    - sk-... / AIzaSy... 직접 입력 → Test 검증
```
- 초기 진입 시 두 선택지를 큰 카드로 제시, 선택 후 상세 입력창 슬라이드 다운.

### 3. 플랫폼 수익 모델 명확화
- MyCrew는 **모델 구독료를 대리 수취하지 않음** (이중 지출 방지).
- 대신 **워크스페이스 플랫폼 이용료**만 과금하는 구조로 포지셔닝.
