# [기획] MyCrew 라이브 스플릿 프리뷰 (Live Split Preview) PRD

**문서 작성일**: 2026-05-06
**상태**: 기획 완료 (개발 대기)
**목표**: 에이전트(개발팀)가 작성한 코드를 아리 엔진(서버)을 통해 즉각적으로 렌더링하고, 대시보드 내 모달에서 Split View(분할 화면) 형태로 실시간 확인(Preview)할 수 있는 UI/UX 환경 구축.

---

## 1. 아키텍처 및 백엔드 설계 (Iframe + Static Server)

**1.1. 설계 방향**
- 복잡한 도커/빌드 컨테이너 없이 **현재 아리 엔진(Node.js/Express)을 정적 웹 서버(Static Server)로 활용**하여 극강의 심플함 추구.
- 에이전트가 `<file_operations>` 태그로 물리적 디스크(`outputs`)에 파일을 생성하면, 이를 그대로 브라우저로 서빙.

**1.2. 백엔드 라우팅 (server.js)**
- **추가 라우트**: `/preview/:projectId`
- **로직**: 해당 프로젝트의 `outputs` 폴더를 `express.static` 미들웨어로 연결.

---

## 2. 프론트엔드 UI/UX 설계 (TaskDetailModal)

**2.1. 트리거 (버튼 배치)**
- **위치**: TaskDetailModal 우측 상단 혹은 마크다운으로 렌더링된 코드 블록 우측 상단.
- **UI**: `[👀 PREVIEW]` 버튼 배치. 클릭 시 모달 확장 및 Split View 진입.

**2.2. 모달 확장 (Expanded Split View)**
- 모달 크기가 화면의 90% 이상으로 확장됨.
- **좌측 (Left Pane)**: 기존 Task 상세 내용 및 코멘트 영역 (코드 리뷰/지시 목적).
- **우측 (Right Pane)**: `<iframe>`을 활용해 프로젝트 결과물이 렌더링되는 프리뷰 영역. (`src="http://localhost:4007/preview/{해당프로젝트ID}/outputs/index.html"`)

**2.3. 리사이저블 드래그 (Draggable Resizer)**
- 좌우 패널 사이에 마우스 드래그가 가능한 **경계선 핸들(Resizer)** 배치.
- 좌우 너비 비율(기본 50:50)을 유동적으로 조절 가능.
- **이슈 방지**: 드래그 중 마우스가 Iframe 안으로 들어가 이벤트가 끊기는 현상을 막기 위해, 드래그 시 Iframe에 `pointer-events: none` 적용.

**2.4. 추가 편의 기능 (Iframe 상단 바)**
- **[새로고침]**: 에이전트가 코드를 덮어썼을 때 클릭 한 번으로 즉시 최신화.
- **[새 창 열기]**: 프리뷰를 브라우저의 별도 탭으로 띄우는 기능 지원.

---

## 3. 에이전트 프롬프트 조정 (Vibe Coding 스타일)

**3.1. 지시사항 추가**
개발팀(dev_*) 에이전트들의 프롬프트에 다음 내용을 추가하여, 뷰어에서 즉각 실행 가능한 형태의 코드를 작성하도록 유도.
> "프론트엔드 산출물은 반드시 `outputs/index.html`을 진입점으로 생성할 것. 별도의 빌드(npm install 등) 과정 없이 즉시 브라우저에서 렌더링 가능한 바닐라 환경(또는 CDN 활용)으로 작성하라."
