# Phase 32: 텔레그램 미니 앱(TMA) 기반 모바일 승인/지휘 센터 기획서

## 1. 기획 배경 및 역할 재정의 (Background & Role Redefinition)
* **문제점 (Pain Point):** 현재 텔레그램 봇을 통한 텍스트 기반 소통은 빠르고 접근성이 좋으나, **'단일 채팅방(Flat Chat)'의 한계**로 인해 여러 프로젝트의 맥락(Context)이 섞이고 모호해지는 치명적인 단점이 존재함.
* **텔레그램 채팅방의 역할 축소 (알람 센터):** TMA 도입 이후, 기존의 텔레그램 텍스트 채팅방은 직접 지시를 내리는 곳이 아닌 **"단순 알림 수신함(Push Notification Center)"**으로 역할이 축소됨. (예: "새로운 리뷰 대기 건이 있습니다", "치명적인 오류 발생" 등의 단방향 알람만 수신)
* **미니 앱의 역할 격상 (모바일 지휘소):** 실제 프로젝트 선택, 태스크 생성, 타임라인 확인, 피드백 등 **모든 양방향 인터랙션은 전적으로 텔레그램 미니 앱(TMA) 내부에서 수행**함.
* **핵심 가치:** 텔레그램의 '즉각성'은 유지하면서, 웹앱의 '구조화된 UI(프로젝트 선택, 리뷰 덱, 특정 카드 지정)'를 결합하여 모바일 환경에서 완벽한 **'CEO 전용 지휘 통제실(Command Center)'**을 구축함.

---

## 2. 핵심 기능 및 UX/UI 시나리오

### 2.1 진입 및 프로젝트 분기 (Project Selector)
* 텔레그램 봇이 보내온 알람 하단의 `[MyCrew 열기]` 버튼을 누르거나, 메뉴 버튼을 통해 미니 앱이 팝업됨.
* 첫 화면에서 현재 권한이 있는 **[프로젝트 목록]**이 카드 형태로 노출됨.
* 유저가 프로젝트를 선택하여 입장하는 순간, 해당 미니 앱 내의 모든 채팅과 타임라인 액션은 **해당 프로젝트 스코프(Context Scope)**로 완벽히 격리됨. (텔레그램 소통 시 발생하던 맥락 혼재 문제 100% 해소)

### 2.2 승인 대기열 덱 (Review Deck)
* 데스크탑의 거대한 칸반 보드를 모바일에 억지로 구겨 넣는 대신, CEO(결정권자)에게 가장 중요한 **'Review(승인 대기)' 상태의 카드만 스와이프 형태의 덱(Deck) 또는 리스트로 압축 제공**.
* 각 리뷰 카드에는 AI가 요약한 결과물과 첨부파일이 표시되며, 하단에 `[승인 (Done)]` / `[반려 및 재작업 (In Progress)]` 버튼 직관적 배치.

### 2.3 타임라인 및 태그 지시 (Contextual Chat & Timeline)
* **타임라인 모드:** 미니 앱 하단 탭을 통해 해당 프로젝트의 실시간 진행 과정(Activity Log)과 결과물을 텔레그램 채팅처럼 스트리밍으로 수신.
* **넘버링 태그 지시:** 타임라인에 올라온 특정 이벤트나 태스크 카드에는 `#번호`가 부여되어 있음. 유저는 해당 번호를 탭(Tap)하거나 인용하여 "이 부분 색상 톤 다운해줘", "이 문구는 이렇게 수정해"라고 **맥락이 유지된 추가 지시**를 내릴 수 있음.
* **즉석 태스크 생성:** 미니 앱 내의 채팅창에 텍스트나 음성으로 지시를 내리면, 현재 활성화된 프로젝트의 칸반 보드에 즉시 태스크로 자동 생성 및 할당됨.

## 3. 기술 아키텍처 및 구현 방안

1. **Frontend (TMA Client):** 
   * React (Vite) + Telegram Web Apps API (`window.Telegram.WebApp`) 사용.
   * 텔레그램 테마 색상을 자동으로 상속받아 이질감 없는 다크/라이트 모드 UI 구현.
2. **Backend (API & Socket):** 
   * 미니 앱이 열릴 때 텔레그램이 제공하는 `initData` 해시를 검증하여 사용자의 세션을 인증.
   * 데스크탑과 동일하게 Socket.io를 연결하되, 진입 시 선택한 `projectId` 룸에만 조인(`socket.emit('joinProject', projectId)`).
3. **Database 연동:** 
   * 미니 앱에서 `GET /api/tasks?project=A&status=REVIEW` API를 호출하여 리뷰 카드만 로드.

## 4. 기대 효과 (Business & User Impact)
1. **모바일 사용성 극대화:** 밖에서 이동 중일 때 무거운 데스크탑 대시보드를 열지 않고도, 텔레그램 안에서 손가락 터치 몇 번으로 프로젝트의 병목(Review)을 해소할 수 있음.
2. **명확한 지시 구조:** 단일 텔레그램 채팅방에서 "아까 그 디자인..." 이라고 말할 필요 없이, 해당 프로젝트 보드 안에서 **특정 카드나 로그를 직접 참조(태그)**하여 지시하므로 AI와 휴먼 에이전트 모두에게 오해의 소지를 없앰.
3. **강력한 셀링 포인트:** "텔레그램 메신저 하나로 멀티 프로젝트를 관리하고, AI 요원들을 지휘하는 경험"은 기존 B2B SaaS(슬랙, 노션)의 한계를 뛰어넘는 마이크루만의 독보적인 강점이 됨.
