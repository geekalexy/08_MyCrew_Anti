### C0-A. 지시 & 할당 (4개)

> 아리에게 받은 자연어 요청을 전문 프롬프트로 정제하고 크루에게 병렬 분배

| # | 스킬명 | 설명 | 입력 | 출력 |
|---|--------|------|------|------|
| **0-1** | 전문 프롬프트 생성 | 자연어 요청 → 실행 가능한 정밀 프롬프트 재작성 | 자연어 요청 | 전문 프롬프트 |
| **0-2** | 태스크 병렬 분해 | 하나의 목표 → 크루별 독립 태스크로 분리·동시 할당 | 목표 + 프롬프트 | 병렬 태스크 목록 |
| **0-3** | 크루 역할 매핑 | 태스크 특성에 맞는 크루 에이전트 선택·할당 | 태스크 목록 | 크루별 할당표 |
| **0-4** | 스케줄 감독 | 크루가 계획한 발행 스케줄 평가·승인·조정 | 스케줄 초안 | 승인 or 수정 지시 |

---

### C0-B. 리뷰 & 보고 (3개)

> 크루 결과물의 1차 관문 — 통과 시 CEO 이관, 불통과 시 재지시

| # | 스킬명 | 설명 | 입력 | 출력 |
|---|--------|------|------|------|
| **0-5** | 1차 결과물 비평 리뷰 | 크루 산출물을 비평적 시각으로 평가 → PASS/FAIL 판정 | 결과물 + 기준 | PASS/FAIL + 피드백 |
| **0-6** | 재지시 (FAIL 시) | 불합격 시 정확한 수정 지시 + 크루 재할당 | FAIL 사유 | 수정 지시 프롬프트 |
| **0-7** | CEO 보고 이관 | 통과 결과물 → CEO Review 열 이동 + 보고서 생성 | 최종 결과물 | 보고 카드 + 이관 |

**리뷰 흐름:**
```
크루 완료 → [0-5] 1차 비평 리뷰
              ├── PASS → [0-7] CEO 보고 이관
              └── FAIL → [0-6] 재지시 → 재작업 → 재리뷰
```

| **0-11** | 우선순위 조정 | CEO 긴급 요청 시 기존 태스크 우선순위 재조정 | CEO 지시 | 업데이트된 우선순위 |

### C0-E. 품질 준수 감독 (2개)

> 크루가 "올바른 방식으로" 일하는지 두 축으로 감독

| # | 스킬명 | 감독 대상 | 주기 | 미준수 시 |
|---|--------|---------|------|---------|
| **0-12** | 스킬 활용 자율성 감독 | 명시 없이도 스킬을 자율 적용하는가 | 태스크 완료 시 수시 | 스킬 보강 피드백 |
| **0-13** | 컨텍스트/템플릿 이행 감독 | 태스크 전 필수 파일(컨텍스트/템플릿)을 반드시 읽는가 | **일간 + 주간** | 경고 → 재작업 → 규칙 강화 |

**0-13 이행 감독 사이클:**
```
[일간] 완료 태스크 중 컨텍스트/템플릿 미참조 건 확인
  이행 ✅ → 정상 / 미이행 ❌ → 경고 + 재작업 지시

[주간] 이행률 집계
  이행률 ≥ 80% → 정상
  이행률 < 80% → 원인 분석
    └→ 프롬프트 강화 / SOP 업데이트 / 크루 재교육
```
