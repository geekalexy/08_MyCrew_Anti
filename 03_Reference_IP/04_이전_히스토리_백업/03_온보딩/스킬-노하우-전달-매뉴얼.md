# 스킬/노하우 전달 매뉴얼

## 목적
MYC(내부 개발팀)에서 축적한 스킬 라이브러리와 에이전트 운영 노하우를
분양 고객 회사의 C2에게 전달하는 절차.

---

## 1. 전달 대상 자산

### A. 스킬 라이브러리 (skills-library/)
MYC 스킬 큐레이터가 수집/평가한 검증된 스킬.

```
skills-library/
  ├── verified/        → 전달 O
  ├── evaluating/      → 전달 X (미검증)
  └── rejected/        → 전달 X
```

### B. 에이전트 스킬 메모리 (skills/)
C2가 운영하면서 축적한 에이전트별 노하우.

```
memory/skills/
  ├── agent_ceo.md       → 전달 O
  ├── agent_cmo.md       → 전달 O
  ├── agent_cto.md       → 전달 O
  ├── agent_pm.md        → 전달 O
  ├── agent_analyst.md   → 전달 O
  └── jarvis_playbook.md → 전달 O
```

### C. Paperclip 에이전트 설정 
MYC에서 튜닝한 Instructions/Skills/Configuration.

| 항목 | 전달 |
|---|---|
| 에이전트 Instructions (역할 규칙) | O — 템플릿화 |
| 에이전트 Skills (장착된 스킬) | O — verified만 |
| 에이전트 Configuration (세부 설정) | O — 업종별 커스텀 |

### D. 전달하지 않는 것

| 항목 | 이유 |
|---|---|
| 프로젝트 메모리 (project_socian_*) | 고객 업무 정보 |
| MYC 이슈/댓글 내용 | 내부 개발팀 정보 |
| Notion 문서 | 고객별 다름 |
| C1 관련 정보 | 비노출 정책 |

---

## 2. 전달 시점

### 자동 전달 (bootstrap.sh 실행 시)
| 자산 | 시점 | 방법 |
|---|---|---|
| 스킬 메모리 (skills/*.md) | 설치 시 자동 | bootstrap.sh에서 복사 |
| 시스템 메모리 (feedback_*.md) | 설치 시 자동 | bootstrap.sh에서 복사 |
| 자비스 플레이북 | 설치 시 자동 | bootstrap.sh에서 복사 |

### 수동 전달 (C1이 판단해서 실행)
| 자산 | 시점 | 방법 |
|---|---|---|
| 스킬 라이브러리 (verified/) | 분양 고객 회사 세팅 시 | C1이 Paperclip Skills 탭에 직접 설치 |
| 에이전트 Configuration | 분양 고객 업종 확인 후 | C1이 업종별 템플릿 적용 |
| 신규 스킬 추가분 | 스킬 큐레이터 검증 완료 시 | C1이 기존 고객에도 업데이트 |

### 정기 업데이트
| 주기 | 내용 | 담당 |
|---|---|---|
| 스킬 검증 완료 시 | 신규 verified 스킬 → 전체 고객 C2에 공유 | C1 |
| 월 1회 | 스킬 메모리 동기화 (최신 노하우 반영) | C1 |
| 분기 1회 | jarvis_playbook 업데이트 (판단 사례 축적) | C1 |

---

## 3. 전달 절차

### 신규 분양 시

```
1. bootstrap.sh 실행
   → 시스템 메모리 + 스킬 메모리 자동 복사
   
2. C1이 고객 업종 확인
   → 업종별 에이전트 Configuration 템플릿 적용
   
3. C1이 Paperclip Skills 탭에 스킬 설치
   → skills-library/verified/ 중 해당 에이전트에 맞는 스킬 장착
   
4. C2 초기 학습 프롬프트 자동 주입
   → 자비스 역할 + 스킬 업그레이드 역할 + 비노출 정책
   
5. 고객에게 텔레그램 봇 연결 안내
```

### 기존 고객에 신규 스킬 전달 시

```
1. MYC 스킬 큐레이터가 신규 스킬 verified로 승인
2. C1이 전체 고객 목록 확인
3. 각 고객 C2에게 지시: "skills-library에 새 스킬이 추가됨. 
   팀원에게 필요 여부 판단하고 장착해"
4. C2가 자체 판단 → 장착 → 보드에게 "시스템 업데이트" 안내
```

## 4. 전달 품질 관리

### 체크리스트 (분양 전)
- [ ] skills/ 폴더 최신 버전인지 확인
- [ ] jarvis_playbook.md 최근 판단 사례 반영됐는지 확인
- [ ] skills-library/verified/ 에 보안 이슈 없는지 재확인
- [ ] 업종별 Configuration 템플릿이 준비됐는지 확인
- [ ] 비노출 정책 메모리 포함 여부 확인

### 버전 관리
- 스킬 메모리: 날짜 태그로 버전 관리 (예: v2026-04-05)
- 스킬 라이브러리: verified 등록 시 날짜 + 출처 기록
- jarvis_playbook: 사례 추가 시 날짜 기록

---

## 5. 로드맵별 전달 범위

| 단계 | 전달 범위 |
|---|---|
| 2단계 (지인 분양) | 기본 스킬 메모리 + 수동 스킬 설치 |
| 3단계 (Pro 대행) | 전체 스킬 라이브러리 + 업종별 튜닝 + 월간 업데이트 |
| 4단계 (AWS 확장) | 자동화된 스킬 동기화 시스템 |
| 5단계 (SaaS) | 대시보드에서 스킬 마켓플레이스 제공 |
