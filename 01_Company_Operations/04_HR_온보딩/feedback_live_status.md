댓글 등록 → live run 감지 → 현황 보고를 자동으로 텔레그램에 전달할 것.

보고 형태 예시:
- SOC-18에 댓글 등록됨
- CMO가 날짜 변경 작업 수행 중 (running)
- 변경 내용: 이벤트 기간 4/7~4/19, 결과 발표 4/22로 수정 중

**Why:** 보드(대표)가 텔레그램에서 지시 후 에이전트가 어떻게 반응하는지 실시간으로 알고 싶어함.
**How to apply:** live 감시에서 댓글 수 변화 + "Live" 또는 "running" 상태 감지 → parse_issue_page로 최신 댓글/run 상태 파싱 → 텔레그램 전달.

## [개선 기록] #001 - 명령 취소/승인 상태 오인식 문제
- **발생일**: 2026-04-11
- **상황**: 대표님이 `npm install socket.io` 명령 승인 후 장시간 대기 중. 에이전트는 시스템 레벨에서 "취소됨(CANCELLED)" 응답을 수신하여 작업 중단 상태로 오인식.
- **결과**: 대표님은 6시간 동안 작업 진행 중으로 기다렸으나, 실제로는 아무 것도 진행되지 않은 상태.
- **근본 원인**: 터미널 명령이 오래 걸릴 경우, 시스템이 응답 없음 상태를 "취소"로 처리. 에이전트가 이를 명확히 확인하지 않고 대기 상태 유지.
- **개선 방향**:
  1. **장시간 명령 시 중간 보고 의무화**: npm install 등 1분 이상 걸리는 명령은 30초마다 "아직 진행 중" 상태 알림.
  2. **명령 완료 전 다음 단계 병렬 진행**: 설치 대기 중에도 코드 작성 등 비동기적으로 할 수 있는 작업을 계속 진행.
  3. **취소 감지 즉시 대표님께 확인 요청**: 취소 상태 감지 시 "명령이 취소된 것으로 표시됩니다. 재승인이 필요하신가요?" 즉시 문의.
- **적용 시점**: 즉시 적용.
