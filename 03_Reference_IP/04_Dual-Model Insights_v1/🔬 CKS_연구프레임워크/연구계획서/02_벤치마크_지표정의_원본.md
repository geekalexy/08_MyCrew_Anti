## 2. 정량 지표 상세

---

### TEI — 토큰 효율 지수

```
TEI = 출력 품질 점수 (0~100) ÷ 사용 토큰 수 (천 단위)

예시:
  품질 점수 85점, 토큰 12,400개 사용
  TEI = 85 ÷ 12.4 = 6.85

해석:
  TEI 높을수록 = 적은 토큰으로 높은 품질
  Group A vs Group B TEI 비교로 효율성 증명
```

**품질 점수 산정 기준**: OP(출력 선호도) 블라인드 평가 결과값 사용

---

### HER — 환각 발생률

```
HER = 사실 오류 건수 ÷ 전체 검증 가능 클레임 수 × 100(%)

예시:
  릴스 대본 내 "소시안 플랜 C 월 39,000원" 주장
  실제 가격: 49,000원 → 오류 1건
  전체 가격 관련 클레임 5건
  HER = 1 ÷ 5 × 100 = 20%

팩트체크 기준:
  - 소시안 공식 웹사이트 현행 데이터
  - 컨텍스트 파일 (SSOT) 내 정보
  - 공개 문서 (가이드북, 공식 블로그)
```

---

### IRC — 반복 수정 횟수

```
IRC = 최종 CEO 승인까지의 총 재작업 횟수

측정 방법:
  칸반 보드 이동 로그 추적
  [완료] → [CMO Review FAIL] → [재작업] 이동 횟수 카운트

해석:
  IRC 낮을수록 = 첫 시도에 품질 높음
  Group A vs Group B IRC 비교
```

---

### CCI — 교차 인용 지수 (CKS군 전용)

```
CCI = 에이전트 B가 에이전트 A의 고유 개념을 흡수한 횟수
      ÷ 교차 평가 총 횟수

예시:
  소넷이 루카의 "SSOT" 개념을 회고 후 채택: 1건
  소넷이 루카의 "자율주행" 개념을 회고 후 채택: 1건
  총 교차 평가: 10회
  CCI = 2 ÷ 10 = 0.2 (20%)

해석:
  CCI > 0.3 = 활발한 지식 교환 발생
  CCI → 0 = 교차 평가가 실질적 영향 없음
```

---

### CS — 수렴 속도

```
CS = 에이전트 간 최종안 합의까지 걸린 라운드 수 (낮을수록 좋음)

측정 방법:
  회고 루프 횟수 카운트
  Round 1: 독립 출력
  Round 2: 교차 평가 후 재출력
  Round 3: 통합안 생성
  CS = 3 (이 경우)

Group A(적대적): 토론이 길어질수록 CS 증가
Group B(CKS): 협력적이면 CS 낮게 유지 예측
```

## 3. 정성 지표 상세

---

### EII — 창발 아이디어 지수 ⭐ 핵심 지표

```
EII = 최종 출력에서 발견된 창발 아이디어 수
      ÷ 전체 주요 아이디어 수

창발 아이디어 정의:
  "어느 단일 에이전트의 사전(pre-) 출력에도 존재하지 않던
   교차 평가·통합 과정에서 새롭게 출현한 개념"

예시 (2026-04-16 파일럿):
  ① "/크로스체크 모드를 제품 피처로" → 창발 (루카/소넷 사전 출력에 없음)
  ② "Red Teaming 카피 검증" → 창발 (교차 후 루카가 새로 제안)
  ③ "3-Tier 오퍼레이팅 모델" → 창발 (Prime이 통합 과정에서 생성)
  
  EII = 3 (창발) ÷ N (전체 아이디어) = 측정 필요

측정 방법:
  1. 교차 평가 전: 각 에이전트 사전 출력의 아이디어 목록 추출
  2. 최종 출력: 전체 아이디어 목록 추출
  3. Diff: 사전 목록에 없는 아이디어 = 창발
```

---

### KSI — 지식 동기화 지수 ⭐ 신규 제안 지표

```
KSI = (에이전트 A의 사후 출력에서 B 고유 개념 등장 비율)
    + (에이전트 B의 사후 출력에서 A 고유 개념 등장 비율)
    ÷ 2

실험 프로토콜:
  Pre:  에이전트 A, B가 주제 X에 대해 독립 출력
  [교차 평가 실시]
  Post: 동일 에이전트 A, B가 관련 주제 X'에 대해 출력
  
  측정: Post 출력에 상대의 Pre 고유 개념이 등장하는 비율

해석:
  KSI > 0.5 = 유의미한 지식 동기화 발생 ✅
  KSI 0.2~0.5 = 부분적 동기화
  KSI < 0.2 = 동기화 실패 (고립된 에이전트와 동일 수준)
  KSI = 0 = 교차 평가가 전혀 영향 없음

이 지표가 새로운 이유:
  기존 MoA/Debate 연구에서는 최종 출력 품질만 측정.
  KSI는 에이전트의 '학습'과 '진화' 자체를 측정하는 최초 지표.
```

---

### OP — 출력 선호도

```
OP = Group B 선호 응답 수 ÷ 전체 응답 수 × 100(%)

측정 방법:
  블라인드 A/B 평가
  인간 평가자: 최소 33명 (통계적 유의성 확보)
  평가 방식: A안(Group A 출력) vs B안(Group B 출력) 중 선택
  평가자는 어느 쪽이 어떤 프레임워크인지 모름

해석:
  OP > 60% = CKS 우세 (통계적 유의)
  OP 45~60% = 차이 없음
  OP < 45% = 적대적 프레임워크 우세
```

---

### PSC — 문제 해결 완성도

```
PSC = 충족된 요구사항 항목 ÷ 전체 요구사항 항목 × 100(%)

측정 방법:
  과제별 요구사항 체크리스트 사전 작성
  결과물과 체크리스트 대조
  미충족 항목 기록

예시 (캠페인 기획 과제):
  체크리스트:
  □ 채널별 콘텐츠 계획 포함
  □ D-Day 타임라인 포함
  □ 크루별 역할 명시
  □ KPI 지표 포함
  □ 예산 고려
  
  5개 중 4개 충족 시: PSC = 80%
```
