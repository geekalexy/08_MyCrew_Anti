## 1. 💼 비즈니스 모델 연계: 듀얼 티어(Dual-Tier) 배포 전략

마이크루의 아키텍처는 백엔드(Node), 프론트엔드(Vite), 우회 터널(MCP)이 철저하게 분리된 MSA(Microservices Architecture) 형태입니다. 이를 활용해 비용 방어와 유료 전환을 극대화하는 **Two-Track 전략**을 채택합니다.

### 🟢 Tier 1: 100% 로컬 프리미엄 체험판 (Free/Freemium)
- **배포 형태**: 사용자가 데스크톱 앱(또는 통합 패키지 스크립트)을 ダウン로드 받아 자신의 PC에 직접 구동.
- **비용 구조 (서버비 $0)**: 
  - 사용자의 PC 자원(CPU/RAM)을 중앙 서버 대신 100% 활용.
  - **BYOK (Bring Your Own Key)**: 구동 시 앱 내에 사용자 본인의 Gemini/Claude API 키를 등록하게 하여 AI 호출 토큰 요금을 마이크루 본사에 부담시키지 않음.
- **타겟 효과**: 인프라 적자 없이 전 세계 수만 명의 유저가 강력한 7인 AI 요원을 무료로 체험하며 '락인(Lock-in)' 효과를 체감.

### 💎 Tier 2: 클라우드 싱크 (Premium Cloud Sync)
- **배포 형태**: 로컬에서 체험 후, "모바일 활용" 및 "팀 단위 협업"이 필요한 고객군을 대상으로 구독료(예: 월 $30~$50)를 받고 중앙 클라우드 서버 접근 권한(OAuth) 부여.
- **제공 가치**:
  1. **멀티디바이스 동기화**: 카페에서 스마트폰으로 접속해도 칸반 보드 및 대본 상태 실시간 확인.
  2. **무제한 토큰 풀(Pool)**: 유저 개별 API 키 체제에서 벗어나, 엔터프라이즈 환경의 중앙 AI 토큰 자유 활용 기능 제공 (마진 확보).
  3. **팀 워크스페이스**: 직원들과 에이전트 작업 현황 공유 및 멀티플레이 부여.

## 2. 🛡️ 봇 탐지 회피 전략: NotebookLM 로컬 터널 (BYOMCP)

가장 무겁고 차단 위험이 높은 "NotebookLM (구글) 브라우저 자동화" 처리는 **클라우드에 통합하지 않고 유저의 PC 연산력을 착취(오프로딩)**하여 해결합니다.

### [문제점: 중앙 집중형 스크래핑의 한계]
- 클라우드 서버(AWS 등)에서 수십 명의 유저를 대신해 Puppeteer(헤드리스 브라우저)를 띄울 경우, **막대한 RAM 고정 비용($100+)**과 **레지덴셜 프록시 요금**이 부과됨.
- 구글의 강력한 Bot Detection에 의해 클라우드 IP가 즉시 영구 밴(Ban) 될 확률이 99%.

### [해결책: BYOC(Bring Your Own Client) 터널링 방식]
1. 유저의 PC(로컬호스트)에 마이크루 시스템 설치 당시에 **NotebookLM 전용 MCP 우회버전**이 함께 설치됨. (포트 3000 전담)
2. 클라우드에 떠 있는 **아리 엔진(Ari Engine)**은 무거운 브라우저를 띄우는 대신, 유저의 로컬 터널 주소(`http://127.0.0.1:3000`)로 가벼운 **API JSON 지시서**만 하달.
3. 유저 본인의 일반 가정용 인터넷 망과 개별 PC 사양을 사용해 백그라운드 크롬 창을 띄워 구글 인증(자동 팝업) 및 파일 업로드를 수행.
4. **결과:** 구글은 정상적인 일반인 사용자로 인지하여 차단율 0%. 마이크루 본사의 서버 스펙은 100유저 동시 접속 시에도 최소형(CPU 2Core / 2GB RAM)으로 거뜬함.

## 3. 💿 통합 설치 시퀀스 (Integration Sequence)

현재 구현된 `start_mycrew.command` 스크립트를 통해 로컬 유저도 1-Click 설치 및 백그라운드 구동 경험을 제공합니다.

```bash
# 구동 3단계 (Script Architecture)
1. 아리 엔진 부팅 (Port 4000) - AI 에이전트 두뇌 및 메인 API
2. MCP 우회 터널 부팅 (Port 3000) - 의존성 자동 체크(NPM Install) 및 구글 로그인 유도 대기
3. 웹 대시보드 부팅 (Port 5173) - UI 프론트엔드 연결
```

- 사용자는 여러 개의 터미널을 띄우거나 설정을 만질 필요 없이, 실행 파일 클릭 한 번으로 위 3가지 마이크로 서비스가 백그라운드에서 상호 연결되어 즉각적인 AI 활용이 가능하도록 모듈화 완료.
