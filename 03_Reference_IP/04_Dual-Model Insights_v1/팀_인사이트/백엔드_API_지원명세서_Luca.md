## 1. 히스토리 로직: '명령 위상(Topology)' 구조화
**📍 프론트엔드 요구사항**: `지시자 ➝ 수행자` 화살표 흐름 UI 렌더링.
**⚙️ 백엔드 대응 작업**:
*   기존에는 칸반 티켓의 `history`나 `comments`가 단순 작성자 관점의 텍스트로 저장되었습니다.
*   **API Response DTO 변경**: `GET /api/tasks/{id}/history` 호출 시, 각 히스토리 객체에 `source(발신)`, `target(수신)`, `action_type(명령/응답/수정)` 필드를 명시적으로 분리하여 JSON 형태로 내려주어야 합니다.
    *   *예시 데이터*: `{ "step": 3, "source": {"id":"user-1", "name":"CEO"}, "target": {"id":"agent-pico", "name":"PICO"}, "type": "ORDER", "content": "여기 수정해" }`

---

## 2. 아티팩트(결과물) 프리뷰 식별자 플래그 부여
**📍 프론트엔드 요구사항**: 미디어가 포함된 결과물일 시 명시적인 `[프리뷰 버튼]` 활성화.
**⚙️ 백엔드 대응 작업**:
*   LLM 에이전트가 텍스트 산출물을 내놓을 때, 프론트엔드가 정규식으로 이미지를 깨작깨작 찾게 만들면 렌더링 부하가 심해집니다.
*   **Artifact Meta 필드 신설**: 백엔드의 파서(Parser) 노드에서 마크다운 리턴 값 내부에 미디어(Image, Video)가 있는지 사전 판별합니다.
*   있을 경우 DB에 저장 시 `has_artifact: true`, `artifact_type: "image"` 플래그를 추가하고, API 리스폰스의 최상위 Depth에 `artifact_url` 배열을 건네주어 프론트엔드가 즉시 버튼을 그릴 수 있게 돕습니다.

---

## 3. 인라인 에디팅 전용 "가벼운 스웜(Short-lived Swarm)" API 신설
**📍 프론트엔드 요구사항**: 아티팩트 결과물 창에서 채팅으로 "배경색 어둡게 해줘" 입력 시 즉각 수정.
**⚙️ 백엔드 대응 작업 (🔥 최중요 과제)**:
*   기존 시스템은 수정 요청이 들어오면 전체 칸반 Phase 워크플로우를 처음부터 끝까지 다 돌리는 무거운 구조였습니다.
*   **신규 라우터 개발**: `POST /api/agents/inline-edit`
*   유저가 부분 수정을 지시하면, 오케스트레이터(ARI)를 거치지 않고 **해당 포맷팅을 담당하는 에이전트(LUMI)만 단독으로 핑(Ping)을 때려** 부분 수정값만 즉각 리턴받는 가벼운 백엔드 호출 우회로(Bypass)를 열어주어야 합니다. 토큰 낭비와 대기 시간을 획기적으로 줄입니다.

---

## 4. 네트워크 페이로드(Payload) 다이어트
**📍 프론트엔드 요구사항**: 칸반 보드 전단 `TaskCard`의 렌더링 메타데이터 최소화.
**⚙️ 백엔드 대응 작업**:
*   현재 `GET /api/tasks` (칸반 리스트 호출 API) 시 카드 본문(`content`)이나 무거운 스키마가 전부 날아가고 있습니다.
*   카드에는 `Title`, `Assignee`, `Priority` 3개만 필요하므로, 이 목록 API 에서는 본문을 제외한 **초경량 DTO 모델**을 적용하여 SQL 쿼리 시간과 브라우저 네트워크 부하를 동시에 줄이는 최적화 작업을 병행합니다.

---

## 5. 종합 요약: API 엔드포인트 변경 명세표

프론트엔드 작업 지원을 위해 백엔드 파트가 이번 스프린트에서 수행해야 할 4개의 API 작업 요약입니다.

| API 엔드포인트 | 현재 상태 (As-Is) | 스펙 변경 작업 (To-Be) |
|---|---|---|
| `GET /api/tasks` | `content` 전체 반환 (무거움) | **경량 DTO 3필드 전환** (네트워크 페이로드 다이어트) |
| `GET /api/tasks/:id` | ❌ 미구현 | **단건 상세 조회 라우터 신설** 필요 |
| `GET /api/tasks/:id/comments` | `author` / `content` 2필드 구조 | **source ➝ target 위상 구조로 변환** |
| `POST /api/agents/inline-edit` | ❌ 미구현 | **LUMI 단독 핑 가벼운 bypass 라우터 신설** |
