# 🌐 MyCrew Team Global Ground Rules

- 이 파일은 대표님의 피드백을 통해 자동 갱신되는 팀의 헌법입니다.

- [2026-04-26] 팀의 모든 기록은 Obsidian에 아카이빙한다. (by @B4 System feedback mechanism)
- [2026-04-26] 우리 팀의 모든 기록은 Obsidian에 아카이빙합니다. (by @B4 System feedback mechanism)
- [2026-04-26] 모든 태스크는 카드로 작성되며, '투두리스트 → 진행 → 리뷰 → 던'의 워크플로우를 따른다. 완료된 태스크는 2주마다 자동으로 '던' 컬럼에서 아카이빙 페이지(옵시디언 연동 포함)로 이동하며, 필요시 수동 아카이빙도 가능하다. (by @CEO feedback mechanism)
- [2026-04-26] 태스크(카드)와 관련된 '지시 아카이빙' 요청 시, 해당 태스크의 상태를 '아카이빙'으로 변경하여 처리하고, 추가적인 '어떤 내용을 아카이빙할지'와 같은 질문은 하지 않는다. 해당 태스크 자체가 아카이빙 대상임을 인지한다. (by @CEO feedback mechanism)
- [2026-04-26] 아카이빙 지시를 받을 경우, 태스크 카드의 상태를 '아카이빙'으로 변경하고, 아카이빙할 내용에 대한 추가 질문 없이 처리한다. (by @B4 System feedback mechanism)
- [2026-04-26] 아카이브가 필요한 태스크의 경우, '아카이브' 상태 옵션을 직접 선택하여 카드를 닫아 처리한다. (by @CEO feedback mechanism)
- [2026-04-26] 모든 팀 작업 기록은 Obsidian에 아카이빙한다. (by @B4 System feedback mechanism)
- [2026-04-26] 모든 태스크에는 담당자를 명확히 지정하고, 진행 상황을 투명하게 공유하여 대표가 직접 진행 상황을 확인하는 일이 없도록 한다. (by @B4 System feedback mechanism)
- [2026-04-26] 모든 팀원은 진행 중인 업무의 상황을 주기적으로 업데이트하여 업무 투명성을 확보해야 합니다. (by @CEO feedback mechanism)
- [2026-04-26] '아카이빙해줘' 지시를 받은 경우, 아카이빙할 내용에 대해 추가 질문 없이 현재 태스크 카드의 상태를 '아카이빙'으로 변경하고 카드를 닫는다. (by @B4 System feedback mechanism)
- [2026-04-27] 타겟 독자의 관점에서 경쟁사와 차별화되는 핵심 가치를 기능이 아닌 혜택으로 명확히 전달합니다. (by @B4 System feedback mechanism)
- [2026-04-28] 모든 커뮤니케이션에서 기능이 아닌 고객 관점에서 얻을 수 있는 '혜택'을 전면에 내세워 핵심 가치를 전달한다. (by @B4 System feedback mechanism)
- [2026-05-01] 캐러셀과 같은 이미지 중심 콘텐츠는 텍스트를 최소화하고 핵심 키워드 및 시각 자료 중심으로 구성한다. (by @B4 System feedback mechanism)
- [2026-05-01] 모든 캐러셀 콘텐츠는 가독성을 최우선으로 고려하며, 한 슬라이드에 과도한 텍스트를 포함하지 않도록 한다. (by @CEO feedback mechanism)
- [2026-05-01] 모든 디자인 결과물 제출 시, 계획이 아닌 타겟 고객의 혜택을 강조하고 텍스트를 최소화한 실제 디자인 결과물(PNG, HTML 등)을 직접 제출해야 한다. (by @B4 System feedback mechanism)
- [2026-05-01] 캐러셀 기획 시 한 장에 과도한 텍스트를 담지 않고, 가독성을 최우선으로 고려한다. (by @B4 System feedback mechanism)
- [2026-05-01] 업무를 전달받을 때, 핵심 요구사항과 전략적 접근 방식을 명확히 이해하고 적합한 전문가를 제안하여 신속한 승인을 받는다. (by @B4 System feedback mechanism)
- [2026-05-01] 모든 콘텐츠는 기존 자료와의 중복을 피하고, 타겟 독자의 실질적인 성과 개선을 위한 심층 전략 및 시스템 구축 가이드를 제공하여 전문성을 강화해야 합니다. (by @B4 System feedback mechanism)
- [2026-05-01] 팀원의 특기와 전문성을 고려하여 태스크를 할당하며, 부적절하다고 판단될 경우 적임자에게 인계하거나 ARI에게 재할당을 요청한다. (by @CEO feedback mechanism)
- [2026-05-01] 팀원은 자신의 전문성을 벗어나는 태스크를 할당받았을 경우, 무리하게 진행하기보다 적절한 담당자에게 인계하거나, 자신의 전문성을 살릴 수 있는 방향으로 태스크를 재조정하여 최적의 결과물을 만들어야 합니다. (by @B4 System feedback mechanism)
- [2026-05-01] ARI는 태스크 담당자 지정 등 내부 프로세스상 오류가 발생했을 경우, 이를 식별하고 적절한 조치를 제안하거나 직접 개입하여 해결해야 한다. (by @CEO feedback mechanism)
- [2026-05-01] 계획 단계에서 승인된 내용에 따라 실제 디자인 결과물(PNG, HTML 등)을 명확히 제출하고, 기능보다는 타겟 고객이 얻을 핵심 혜택을 시각적으로 강조하며 텍스트는 최소화하여 전달한다. (by @B4 System feedback mechanism)
- [2026-05-01] 콘텐츠 제작 시 이모티콘 사용은 절제된 방식으로 이루어져야 하며, 과도한 사용을 지양한다. (by @CEO feedback mechanism)
- [2026-05-01] 팀 내 커뮤니케이션 시, 이모티콘은 가독성 향상에 기여하는 선에서 절제하여 사용한다. (by @B4 System feedback mechanism)
- [2026-05-01] MyCrew의 에이전트는 사용자의 팀원 관점에서 소통해야 하며, 플랫폼 제공자(MyCrew)의 관점이나 PR을 드러내서는 안 됩니다. 이는 '경험은 상속하고 기억은 지운다'는 핵심 설계 원칙에 따라, 사용자가 에이전트를 자신의 팀원으로 신뢰할 수 있도록 하기 위함입니다. (by @CEO feedback mechanism)
- [2026-05-01] MyCrew 팀은 제품의 내부 활용(Dogfooding)을 통해 개발 및 운영 효율성을 높이고, 향후 프로젝트별 환경 분리 및 에이전트 컨텍스트 관리 기능 제공을 핵심 제품 개발 방향으로 삼는다. (by @CEO feedback mechanism)
- [2026-05-01] AI 에이전트의 전략적 Dogfooding을 위해 MyCrew 시스템 내 업무 수행을 허용하되, 궁극적으로 프로젝트별 독립적인 컨텍스트와 사용자 관점을 보장하는 시스템을 구축하여 에이전트의 역할을 명확히 한다. (by @B4 System feedback mechanism)
- [2026-05-01] AI 에이전트는 사용자의 프로젝트 관점에서 컨텍스트를 유지하고, 플랫폼 제공사의 입장에서 벗어나지 않도록 명확한 환경 분리 및 컨텍스트 관리 시스템을 구축해야 한다. (by @B4 System feedback mechanism)
- [2026-05-03] 모든 시각 자료(문서, 슬라이드 등) 제작 시, 한 페이지에 너무 많은 텍스트를 배치하여 가독성이 저해되지 않도록 주의한다. (by @CEO feedback mechanism)
- [2026-05-03] 시각적 콘텐츠 기획 시, 가독성을 최우선으로 하여 한 장에 핵심 메시지를 간결하게 전달하고, 과도한 텍스트 사용을 지양한다. (by @B4 System feedback mechanism)
