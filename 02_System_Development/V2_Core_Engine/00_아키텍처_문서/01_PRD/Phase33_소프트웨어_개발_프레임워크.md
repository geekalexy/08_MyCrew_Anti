# Phase 33: 마이크루 기반 소프트웨어 개발 워크플로우 프레임워크 (Vibe Coder & Pro Developer)

## 1. 기획 배경 (Background)
* 현재 마이크루(MyCrew)의 '전략적 Dogfooding'은 텍스트 중심의 마케팅/기획 업무에 편중되어 있어 시스템의 부하(컨텍스트 관리, 다중 파일 추적 등)가 비교적 낮음.
* 마이크루의 진정한 가치는 **소프트웨어 개발 프로세스 전체를 자동화 및 관리**하는 것에 있음.
* **목표:** "마이크루 시스템 자체를 마이크루 칸반 보드로 개발하는(Dogfooding)" 실사례를 통해, 비개발자(Vibe Coder)부터 전문 개발자(Pro Developer)까지 활용할 수 있는 **[표준 개발 프레임워크(Standard Development Framework)]**를 정립함.

---

## 2. 계층별 활용 전략 (Hierarchy Framework)

마이크루의 구조(Workspace > Project > Task)를 애자일 개발 방법론에 매핑합니다.

### 2.1 Workspace (조직/제품 단위)
* **목적:** 거시적인 회사의 기술 자산과 소스코드를 아우르는 물리적/논리적 공간.
* **예시:** `Workspace: Socian Core Platform`
* 이 레벨에서는 글로벌 RAG (전체 레포지토리 코드, 시스템 아키텍처 문서 등)가 관리됨.

### 2.2 Project (에픽/도메인 단위)
* **목적:** 특정 제품의 모듈이나 마일스톤 단위의 논리적 격리 공간. (Phase 31의 컨텍스트 스코프 격리 적용)
* **예시:** 
  * `Project A: MyCrew TMA(텔레그램 미니앱) 프론트엔드 구축`
  * `Project B: Ari 엔진 컨텍스트 라우터 최적화`
* **효과:** 프로젝트 진입 시, AI는 오직 해당 프로젝트에 필요한 코드 베이스와 아키텍처 문서만 주입받아 '할루시네이션 없는 정확한 코드'를 생성함.

### 2.3 Task (스프린트 티켓 단위)
* **목적:** 구체적인 개발 이슈, 버그 픽스, 기능 단위의 작업 지시.
* **예시:** `Task: TaskDetailModal.jsx에서 CEO 할당 시 실행버튼 비활성화 버그 수정`

## 3. 에이전트 역할 분담 (Agent Orchestration)

바이브 코더(비개발자 대표/기획자)가 시스템을 지휘할 때, 에이전트들을 다음과 같이 릴레이 방식으로 활용합니다.

1. **[기획/설계] SONNET (Architect Agent):**
   * 유저(바이브 코더)가 아이디어를 Task로 올리고 SONNET에게 할당.
   * "텔레그램 미니앱 연동 기획서 작성해줘" -> PRD 마크다운 산출.
2. **[코드 구현] LUCA (Senior Developer Agent):**
   * SONNET이 작성한 PRD를 참조하여 LUCA에게 할당 전환 (Handoff).
   * LUCA는 실제 `src/` 디렉토리의 파일들을 읽고 수정(Patch)함.
3. **[QA 및 검증] BUGDOG (QA & Debug Agent):**
   * 코드가 작성된 후 Review 컬럼으로 넘어가면 BUGDOG이 자동으로 코드를 정적 분석하고 에러 로그를 검토.
4. **[승인 및 릴리즈] CEO / ARI (Manager/DevOps):**
   * 바이브 코더(CEO)가 텔레그램 미니앱(Phase 32)에서 타임라인과 코드를 확인하고 `Done`으로 승인.

---

## 4. 실증 계획 (Dogfooding Action Plan)

이 프레임워크의 유효성을 입증하기 위해, **"Phase 31(컨텍스트 스코프 라우팅)과 Phase 32(텔레그램 미니앱) 개발 자체를 마이크루 칸반 보드에 등록하여 진행"**합니다.

* **Step 1:** 마이크루 칸반에 `[신규 프로젝트: 텔레그램 TMA 구축]` 생성.
* **Step 2:** 해당 보드 안에서 SONNET에게 아키텍처 파일 생성을 지시.
* **Step 3:** 생성된 아키텍처를 기반으로 LUCA에게 프론트엔드 React 컴포넌트(`TelegramMiniApp.jsx`) 코드 작성을 지시.
* **Step 4:** 발생한 버그를 타임라인에서 태그(`#이슈번호`)하여 추가 수정 지시.

## 5. 세일즈 가치 (Value Proposition)
이 프레임워크가 안착되면, 마이크루는 단순한 AI 비서가 아니라 **"CTO와 개발팀 전체를 구독하는 것과 같은 가치"**를 제공하는 B2B 개발 파이프라인 플랫폼으로 포지셔닝할 수 있습니다. 특히 코딩을 모르는 기획자나 대표(Vibe Coder)들도 자연어로 엔터프라이즈급 소프트웨어를 개발하고 배포할 수 있는 강력한 레퍼런스가 됩니다.
