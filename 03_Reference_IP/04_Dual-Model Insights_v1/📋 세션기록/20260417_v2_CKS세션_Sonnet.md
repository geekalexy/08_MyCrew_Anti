## 2. 프라임 B팀 기획 리뷰 핵심

프라임이 루카와 소넷의 B팀 기획안을 동시 검토하여 통합 권장안 도출.

### 소넷 기획안 평가
- ✅ 탁월: 스트림 분업 + 트리거 3가지 + 크루 명단 6명
- ⚠️ 보완: 루카의 Red Teaming / Flash→Pro 비용 계단식 개념 없음

### 루카 기획안 평가
- ✅ 탁월: Red Teaming(브랜드변호사vs진상고객) 독창 발상
- ⚠️ 보완: 트리거 조건 없음, 크루 구체 배정 없음, UI 힌트 부족

### 프라임 통합 최종안 — 3-Tier 시스템
```
Tier 1: 스트림 분업 (일상 90%)   → Sonnet 조합 2
Tier 2: 크로스체크 (중요 9%)    → Sonnet 조합 3
Tier 3: Red Teaming (외부노출 1%) → Luca 발상
```

**프라임 최종 한 줄:**
> "듀얼 크루는 사치가 아니라 보험이다.
> 평소에는 파이프라인으로 비용을 줄이고,
> 중요할 때만 크로스체크로 리스크를 줄여라."

## 4. UX/UI 고민 사항 (대표님 원문)

> "온보딩은 온보딩에 최적화된 화면을 제공하고,
> 첫 번째 프로젝트는 팀빌딩으로 해서 사용자가 시간의 흐름,
> 외부 요인 등 요구사항에 따라 팀 운영 룰과 크루들이 하지 말아야 할
> 것들이 점점 명확해질 텐데 그때마다 이것을 불만으로 보면 안 될 것 같고,
> 오히려 팀빌딩 과정으로 가볍게 룰을 생각나는 대로 카드에 적어나가게 하고
> 변경할 룰은 투두로 옮기고 적용할 룰은 진행으로 등으로 하는게 어떨까...
> 이런 고민과 팀 페이지가 개편이 많이 되어야 할 것 같고
> 이때 생각 정리가 좀 되어서 듀얼 뷰 화면 기획을 하면 좋겠어."

### 소넷 해석

**핵심 개념: "룰 카드 칸반"**
```
룰 아이디어 → [카드에 적기]
    └→ 검토 후 [투두(To-Do)]
           └→ 적용 시 [진행(In Progress)]
                  └→ 안착 시 [완료(Done)]
                          └→ 시스템 프롬프트 자동 반영
```

**의미**: 팀 불만 = 팀빌딩 과정. 부정 피드백이 아니라 룰 진화의 재료.

## 5. 연구 목표 수정 — 대표님 발표 (원문)

**포인트 1 — 토큰 효율 문제**
> "내가 만든 CMUX + 개인비서 + 페이퍼클립의 결과물 퀄리티가 나쁘지 않았다.
> 단 과도한 토큰 소모 문제로 전격 사용을 중지했다."

**포인트 2 — 관찰**
> "프라임 포함 너희들 인사이트 도출 과정에서 팀 변화를 목격했다.
> AI 멀티에이전트에도 협력 문화를 접목할 수 있으며
> 이를 통해 더 진화할 수 있겠다는 영감을 받았다."

**포인트 3 — 목표 수정 (원문)**
> "AI 업계의 기존 다중 에이전트 프레임워크는 주로 적대적 생성이나
> 경쟁 기반의 토론 모델에 의존하여 환각을 줄이고 성능을 높였다.
> 본 연구는 에이전트 간의 '경쟁'이 아닌 교차 평가와 회고 프로세스를 통한
> '협력적 지식 동기화(Collaborative Knowledge Synchronization)'가
> 집단 지성 향상 및 창의적 문제 해결(새로운 아이디어 도출)에 미치는
> 긍정적 효과를 정량적/정성적으로 증명하는 것을 새로운 목표로 하고 싶다.
> 돈은 나중에 따라올 것."

## 6. CKS 실험 설계 — 핵심 합의 사항

### 연구 방법
- 논문 + 제품 동시 추진 ("둘 다 해야 증명된다")
- 비교 그룹(적대적 프레임워크) 필수
- 두 그룹은 **완전 격리** (같은 회사/프로젝트 내 공존 불가)

### 격리 원칙
```
Group A (대조군): 적대적 프레임워크 — 별도 격리 환경
Group B (실험군): CKS 프레임워크  — MyCrew 현 환경
동일 과제 투입 후 벤치마크 비교
```

### 측정 지표 확정
```
정량: TEI (토큰 효율), HER (환각률), IRC (수정 횟수), CCI (교차 인용), CS (수렴 속도)
정성: EII (창발 아이디어), KSI (지식 동기화), OP (선호도), PSC (완성도)
```

### KSI — 핵심 신규 지표
> 기존 연구에 없는 지표. 에이전트의 '학습과 진화' 자체를 측정.
```
KSI = (A 사후 출력의 B 개념 등장 비율 + B 사후 출력의 A 개념 등장 비율) ÷ 2
KSI > 0.5: 유의미한 지식 동기화 발생
```

## 7. MyCrew = 리빙 랩 구조

```
MyCrew 플랫폼
  ├── 격리 프로젝트 A: 적대적 팀 (대조군)
  ├── 격리 프로젝트 B: CKS 팀 (실험군)
  └── 벤치마크 대시보드: 실시간 지표 집계
      → 논문 데이터 + 제품 개선 데이터 동시 생성
```
