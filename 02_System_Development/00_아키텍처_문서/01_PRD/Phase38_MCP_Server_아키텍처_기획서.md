## 1. 개요 (Overview)

본 기획서는 MyCrew(마이크루)의 시스템 환경을 특정 IDE 종속이나 하드코딩된 단일 백엔드 구조에서 벗어나, **개방형 MCP(Model Context Protocol) 서버 아키텍처**로 격상시키는 시스템 고도화 방향을 정의합니다.

이를 통해 마이크루는 단순한 'AI 대시보드'를 넘어, 전 세계의 수많은 AI 도구(Claude Code, Cursor 등)와 연동되어 상태 관리와 오케스트레이션을 담당하는 **'범용 AI 운영체제(OS)'**로 도약합니다.

## 2. 사용자 상황별 시스템 환경 (User Segments & Constraints)

마이크루는 사용자 페르소나와 설치 환경에 따라 **투트랙(Two-Track) 하이브리드 구조**로 동작하며, 각 환경별 한계와 이를 극복하는 전략을 명확히 규명합니다.

### A. 저관여 유저 (마케터, 기획자 / SaaS 기반 API 직결 모드)
*   **환경 설명**: 안티그래비티나 로컬 터미널 환경을 구축하지 않고, 웹(SaaS) 기반으로 자신의 **Gemini API 키**만을 꽂아서 사용하는 유저.
*   **포지셔닝**: "API 하나로 굴러가는 나만의 독립적인 AI 기획/마케팅 대행사"
*   **기능적 한계(제약)**: 
    *   사용자의 로컬 파일 시스템 제어 및 물리적인 터미널 환경(빌드, 스크립트 실행) 접근이 차단됨.
    *   코드나 기획서를 작성할 수는 있으나, 이를 즉시 브라우저 외부에서 실행(Preview)하거나 로컬 DB를 구축하는 등의 물리적 개발 액션에 제약이 따름.
*   **MCP 고도화 방향**: 위험한 터미널 엑세스 대신, Notion, Slack, Google Drive 등 **Cloud API 기반의 MCP 도구**만 선별적으로 오픈하여 외부 서비스와의 연결성을 극대화함.

### B. 고관여 유저 (개발자, 하드코어 메이커 / 어댑터 확장 모드)
> 🔗 **연관 심층 분석서**: [Phase38_MCP_Migration_기능영향도_분석서](file:///Users/alex/Documents/08_MyCrew_Anti/02_System_Development/00_아키텍처_문서/01_PRD/Phase38_MCP_Migration_기능영향도_분석서.md) (현재 모든 기능의 MCP 전환 진단 리스트)
*   **환경 설명**: 안티그래비티, Claude Code 등 터미널(CLI)과 강력한 AI 개발 도구를 이미 갖춘 상태에서 이를 마이크루에 연동하는 유저. (현재 대표님 환경이자 마이크루의 핵심 타겟)
*   **포지셔닝**: "강력한 CLI/IDE 실행자들에게 방향과 맥락을 떠먹여 주는 완벽한 수석 프로젝트 매니저(PM)"
*   **기능적 한계 타파**:
    *   마이크루가 직접 코딩하지 않고, 무겁고 복잡한 태스크를 외부의 강력한 도구(Claude Code 등)로 위임(Baton Touch)함.
    *   로컬 권한 제약이 없으므로, 파일 읽기/쓰기, 서버 빌드, 테스트 구동까지 100% 무제한 자동화 가능.

## 3. 핵심 아키텍처 전환: "Ari 엔진의 MCP 서버화"

현재의 Ari 엔진(Node.js 서버) 자체를 터미널 실행 권한을 가진 **MCP 서버**로 전환했을 때의 시스템 차이점을 비교합니다.

### 3.1 도구(Tool) 확장 방식의 패러다임 시프트
*   **기존 (하드코딩 방식)**: 새로운 스킬(예: 인스타 분석, 파일 읽기)을 추가하려면 `executor.js`에 일일이 자바스크립트 래퍼(Wrapper) 코드를 짜고, JSON 스키마를 정의해야 함. 도구 추가에 개발 리소스가 큼.
*   **MCP 전환 후 (Plug-and-Play)**: 표준 MCP 규격을 따르므로, 오픈소스로 배포된 수백 개의 MCP 서버(Notion, GitHub, SQLite 등)를 블록 끼우듯 연결만 하면 Ari가 즉시 해당 도구를 학습하고 사용할 수 있음.

### 3.2 생태계 대통합 및 터미널 툴 연동 (Claude Code 등)
*   **기존 (닫힌 생태계)**: 마이크루 대시보드와 백엔드가 1:1로 묶여 있어, 사용자가 터미널에서 띄운 Claude Code가 마이크루의 상태(칸반 보드)를 조작할 수 없었음.
*   **MCP 전환 후 (생태계 통합)**: 
    *   마이크루 자체가 MCP 서버가 됨으로써, 터미널의 Claude Code가 `마이크루 MCP 서버`에 접속해 마이크루의 DB 리소스를 읽고 쓸 수 있음.
    *   예: 터미널에서 클로드 코드가 스크래핑 작업을 마치면, MCP 통신을 통해 마이크루의 칸반 보드 카드를 자동으로 "Done"으로 이동시키고 로그에 보고서를 남길 수 있음. 마이크루 대시보드가 CLI의 완벽한 GUI 모니터로 작동.

### 3.3 환각(Hallucination) 제로 데이터 스트림
*   **기존 (텍스트 파싱의 한계)**: 긴 데이터(인스타 JSON 결과 등)를 다룰 때, LLM이 텍스트를 읽고 추론하여 DB 입력 포맷을 제멋대로 바꾸거나 누락시키는 환각 현상 발생.
*   **MCP 전환 후 (Direct Resource Access)**: MCP의 `Resources` 개념을 통해, AI가 텍스트를 파싱하는 것이 아니라 마이크루 DB에 다이렉트로 안전하게 접속하여 정확한 데이터 스트림을 입출력함. 중간에 AI의 자의적 해석이 개입할 여지가 원천 차단됨.

## 4. 도입 시 장단점 결론

### 🟢 장점 (Pros)
1.  **환각 0%의 무한 확장성**: 도구를 억지로 파싱하지 않고 오픈 표준으로 붙이므로, 도구 확장에 따른 오류와 개발 부하가 사라집니다.
2.  **타사 최고 지성 툴(Claude Code 등)의 흡수**: 우리가 모든 도구를 만들 필요 없이, 이미 터미널 생태계를 장악한 뛰어난 AI 도구들을 마이크루의 "실행 팔다리"로 그대로 부려먹을 수 있습니다.
3.  **투트랙 전략 완성**: 저관여 유저에게는 독립적인 AI 대행사로, 고관여 유저에게는 AI 오케스트레이션 OS로 동시 판매 가능한 완벽한 아키텍처가 구축됩니다.

### 🔴 단점 및 해결 과제 (Cons)
1.  **지능형 스위칭 체계 필요**: MCP로 강력한 도구를 쥐여줘도 기본 탑재된 뇌(Flash)의 지능이 부족하면 도구를 제대로 못 씁니다. 복잡한 MCP 도구를 호출할 때는 동적으로 Sonnet/Pro 급으로 전환되는 **'지능 스위칭 체계'**가 선행되어야 합니다.
2.  **클라우드 보안**: SaaS 유저를 위해 터미널 MCP를 열어줄 경우 서버가 탈취될 수 있으므로, SaaS 모드에서는 철저하게 'Cloud API MCP'만 허용하는 권한 샌드박스 구현이 필요합니다.
