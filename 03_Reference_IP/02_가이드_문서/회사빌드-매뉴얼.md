## Phase 1: 업종/목적 파악 → 회사 설계

분양 전 고객과 사전 미팅(또는 설문)으로 아래를 파악합니다.

### 1-1. 기본 정보 수집

| 팀 규모(희망) | AI 6명 | _________ |

### 1-2. 업무 규칙 (권고 세팅)
AI 회사의 업무 방식은 일반 회사와 다릅니다. 사용자는 처음이라 어떻게 해야 할지 모르므로, 아래 검증된 규칙을 기본 세팅합니다.

```
[기본 세팅 — 모든 분양에 적용]
- 이슈 기반 업무 관리 (Paperclip에서 이슈 생성 → 배정 → 완료)
- 산출물은 CEO/PM 리뷰 후 보드에 제출 (크로스 피드백 필수)
- 모든 외부 게시/발송은 보드(대표) 승인 후 실행
- 업무 흐름: 기획 → 시안 → 리뷰 → 실행 → 보고
- AI 비서가 텔레그램으로 보드에 보고 (보고체, 한국어)
- 에이전트 간 상호 피드백으로 품질 관리

※ 고객이 업무 경험이 쌓인 후 본인 스타일로 조정 가능
```

## Phase 2: 프로젝트 구조화

### 2-1. 프로젝트 설계 원칙
- **업무 흐름 순서**로 번호 부여 (01_전략 → 02_기획 → 03_실행 → ...)
- **Paperclip 프로젝트 = 로컬 폴더** 1:1 매핑
- 공통 프로젝트: 온보딩, 레퍼런스, 보고서는 업종 무관 필수

### 2-2. 업종별 프로젝트 템플릿

#### 마케팅/광고 (소시안 실사용)
```

02_콘텐츠_기획/     — 콘텐츠 캘린더, 시안
03_발행_완료/       — 발행된 콘텐츠 아카이브
04_이미지_소스/     — 디자인 에셋
05_보고서/         — 주간/월간 보고
06_온보딩/         — 신규 에이전트 교육 자료
07_레퍼런스/       — API 문서, 가이드
08_이벤트_기획/     — 프로모션, 캠페인
09_광고주센터_기획/  — 광고 플랫폼 관리
```

#### 커머스/쇼핑몰
```

02_상품_기획/       — 신상품, MD 기획
03_마케팅/         — 프로모션, 광고

05_보고서/         — 매출/트래픽 보고

07_레퍼런스/       — 플랫폼 가이드
```

#### 서비스/SaaS
```

02_기능_기획/       — 기능 스펙, 와이어프레임
03_개발_관리/       — 개발 이슈 추적
04_마케팅/         — 랜딩페이지, 콘텐츠
05_보고서/         — KPI 보고
06_온보딩/         — 교육 자료
07_레퍼런스/       — API, 기술 문서
```

#### 컨설팅/에이전시
```

03_제안서/         — 제안서 템플릿, 아카이브
04_리서치/         — 산업 분석, 트렌드
05_보고서/         — 프로젝트 보고
06_온보딩/         — 교육 자료
07_레퍼런스/       — 방법론, 프레임워크
```

### 2-3. 폴더 생성 + Paperclip 프로젝트 동기화
```bash
# bootstrap.sh에서 자동 실행
PROJECT_DIRS=("01_전략" "02_콘텐츠_기획" "03_발행_완료" ...)
for dir in "${PROJECT_DIRS[@]}"; do
    mkdir -p "$WORKSPACE_DIR/$dir"
    # Paperclip API로 프로젝트 생성
    curl -X POST http://127.0.0.1:3100/api/companies/{companyId}/projects \
      -d '{"name":"'"$dir"'", "localFolder":"'"$WORKSPACE_DIR/$dir"'"}'
done
```

## Phase 3: 온보딩 자료 준비

### 3-1. 필수 온보딩 문서 (06_온보딩/ 에 저장)

| 파일명 | 내용 |
|--------|------|
| `회사소개.md` | 회사명, 업종, 미션, 타겟 고객, 핵심 제품/서비스 |
| `업무규칙.md` | 업무 프로세스, 승인 체계, 보고 규칙 |
| `프로젝트구조.md` | 폴더 인덱스, 각 프로젝트 설명 |
| `톤앤매너.md` | 브랜드 톤, 대외 커뮤니케이션 가이드라인 |
| `금지사항.md` | 하면 안 되는 것들 (비노출 정책 등) |

### 3-2. 온보딩 자료 작성 요령
- **구체적으로**: "좋은 콘텐츠를 만들어라" ✕ → "인스타 릴스 30초, 후킹 3초 내, CTA 포함" ○
- **예시 포함**: 잘된 예시와 안 된 예시 함께
- **업데이트 규칙**: 새로운 규칙 생기면 온보딩 폴더에 즉시 반영

## Phase 4: 대표 AI(CEO) 생성 → 채용 진행

### 4-1. CEO 에이전트 최초 생성
CEO는 가장 먼저 생성하는 에이전트입니다. 역할:
- 다른 에이전트 채용 지시
- 업무 배분 및 품질 관리
- 보드(대표)에게 보고

```
CEO 초기 지시:
"너는 {회사명}의 CEO AI다.
회사 목적: {핵심 목적}
온보딩 폴더(06_온보딩/)의 자료를 숙지하고,
아래 순서로 팀을 빌드해라:

1. 온보딩 자료 전체 학습
2. 필요 역할 파악 → 채용 제안서 작성 (보드 승인 후 진행)
3. 승인된 역할의 에이전트 생성
4. 각 에이전트에게 온보딩 자료 학습 지시
5. 초기 업무 배분"
```

### 4-2. 채용 순서 (권장)

| 순서 | 역할 | 이유 |
|------|------|------|
| 1 | CEO | 팀 빌드 총괄 |
| 2 | PM | 업무 구조화, 프로세스 관리 |
| 3 | 핵심 실무자 | 업종별 메인 역할 (마케터, 기획자 등) |
| 4 | 전문가 | 디자이너, 분석가 등 |
| 5 | 지원 | 스킬 큐레이터 등 |

### 4-3. 에이전트 생성 시 필수 설정
각 에이전트 생성 시:
- **역할(role)** 명확히 지정
- **온보딩 자료 학습** 지시
- **보고 라인** 설정 (→ CEO 또는 PM)
- **담당 프로젝트** 배정

## Phase 5: 스킬 강화

### 5-1. 초기 운영 → 관찰 (1~2주)
에이전트가 실제 업무를 수행하면서:
- 잘하는 것 / 못하는 것 파악
- 지시 이행률 체크
- 산출물 품질 평가

### 5-2. 스킬 장착 기준
| 상황 | 조치 |
|------|------|
| 같은 실수 반복 | 해당 영역 가이드라인 스킬 추가 |
| 산출물 품질 낮음 | 잘된 예시 + 체크리스트 스킬 추가 |
| 특정 도구 사용 미숙 | 도구 사용법 스킬 추가 |
| 창의성 부족 | 레퍼런스/영감 자료 스킬 추가 |

### 5-3. 스킬 강화 사이클
```
관찰 → 문제 파악 → 스킬 작성 → 장착 → 재관찰 → 튜닝
```

### 5-4. 크로스 피드백 프로세스
- 에이전트 작업 완료 → CEO/PM 리뷰 → 피드백 반영 → 보드 제출
- CEO 작업 → 관련 담당자 피드백 → 반영 후 보드 제출
- 이 과정에서 에이전트 간 학습 발생

## 부록: 소시안 실사례

### 팀 구성
| 역할 | 담당 업무 |
|------|----------|
| CEO | 전략 총괄, 에이전트 관리, 크로스이슈 |
| CMO | 마케팅 총괄, 콘텐츠 방향 |
| Content Marketer | 콘텐츠 기획/제작 |
| Performance Marketer | 광고 운영, 성과 분석 |
| Marketer 1, 2 | 실무 마케팅 |

### 프로젝트 ↔ 폴더 매핑
```
Paperclip 프로젝트     로컬 폴더
─────────────────     ──────────────────
01_전략              → 01_전략/
02_콘텐츠_기획        → 02_콘텐츠_기획/
03_발행_완료          → 03_발행_완료/
...                  → ...
```

### 운영 교훈
1. **에이전트 지시는 구체적으로** — 추상적 지시는 낮은 품질로 이어짐
2. **크로스 피드백 필수** — 단독 작업 후 바로 제출하면 누락 발생
3. **온보딩 자료가 품질을 결정** — 잘 만든 온보딩 = 잘 하는 에이전트
4. **폴더 규칙 초기 세팅이 중요** — 나중에 바꾸면 혼란
5. **스킬은 점진적으로** — 한번에 많이 넣으면 오히려 혼란, 관찰 후 하나씩
