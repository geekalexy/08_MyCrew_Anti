# [Phase 39] MyCrew 지식 그래프 & 아카이브 허브 (Graphify 기반)

## 1. 개요 (Overview)
기존 칸반 보드의 '완료' 카드들을 모아두던 수동적인 아카이브를 전면 개편하여, 프로젝트 전체를 **"거대한 지식 신경망(Knowledge Graph)"**으로 변환합니다.
핵심 엔진으로 **Graphify**를 도입하여, 생성된 코드의 추상 구문 트리(AST), 기획서, 회의록, 이미지 등 모든 다중 모달(Multi-modal) 산출물을 그래프의 '노드(Node)'로 관리합니다. 이를 통해 토큰 한계(Context Window Limit)와 환각(Hallucination) 문제를 근본적으로 해결하고, 초정밀 디버깅과 사실(Fact) 기반의 전문가 에이전트 구동을 가능하게 합니다.

---

## 2. 핵심 애플리케이션 시나리오 (Application Scenarios)

### 시나리오 A: 초고속/초정밀 AST 코드 디버깅 (Zero-Context Debugging)
* **상황**: 복잡하게 얽힌 마이크로서비스 구조에서 에러 발생.
* **기존 한계**: AI가 버그를 잡으려면 관련될 법한 파일 10~20개를 전부 텍스트로 읽어들여 10만 토큰 이상 소모 (비용 폭발 및 핵심 맥락 유실).
* **Graphify 혁신**: 에이전트(Luca)가 텍스트를 읽지 않고, MCP 도구를 통해 지식 그래프에 Cypher 하이퍼쿼리를 날립니다. 
  * `query: "shortest_path(TaskDetailModal, database.js)"`
  * 단 몇 초 만에 수백 토큰의 비용으로 **"Modal ➔ API 라우터 ➔ DB 매니저"** 간의 호출 궤적과 변수 의존성을 핀포인트로 찾아내 즉각 픽스합니다.

### 시나리오 B: Legalize-KR 기반 법무/특허/의료 전문가 시스템 (Fact-based AI)
* **상황**: MyCrew를 IT 코딩용이 아닌 '법무법인 리걸 어드바이저' 등 특정 도메인 용도로 활용.
* **데이터 인입**: `legalize-kr` 저장소(대한민국 법령을 Git 커밋으로 관리)를 Graphify 파이프라인에 주입.
* **Graphify 혁신**: 수십만 장의 법령 문서가 **[개정 연도] ➔ [관련 법령] ➔ [조항] ➔ [관련 판례]** 형태의 그래프로 구축됩니다.
* **결과**: "2023년 개정된 민법 제X조와 연관된 판례를 찾아줘"라는 요청에 LLM이 지어내는(환각) 것 없이, 정확한 그래프 노드만 추출하여 100% 팩트에 기반한 자문을 수행합니다.

### 시나리오 C: 스프린트 타임머신과 아카이브 (Sprint Time-Machine)
* **기능**: 칸반 보드의 `DONE` 카드를 일괄 아카이빙(Bulk Archive).
* **Graphify 연동**: 아카이브된 카드들은 단순한 리스트가 아니라, `[기획서 노드] ➔ [디자인 노드] ➔ [프론트/백엔드 코드 노드]`로 이어지는 릴레이 체인(Context Chain)으로 묶여 시각화됩니다.
* **재가동 (Re-branch)**: 과거의 특정 기획(노드)에서 가지를 쳐서 새로운 스프린트 파이프라인을 즉시 재가동할 수 있습니다.

### 시나리오 D: 무한 컨텍스트 체인 (Limitless Context Chain)
* **기존 한계**: 이전 작업 내용을 텍스트로 계속 이어붙이면 토큰 제한(Context Limit)에 부딪혀 AI가 핵심을 잃어버렸습니다.
* **Graphify 혁신**: 에이전트는 텍스트 대신 **"현재 위치한 그래프 좌표(Node ID)"**만 넘겨받습니다. 만 개의 파일이 얽혀 있어도 에이전트는 좌표를 중심으로 거미줄을 타듯 필요한 정보만 실시간 Fetch하므로 토큰 소모 없이 무한한 맥락을 유지합니다.

### 시나리오 E: 뭉텅이 자료 '폭식' 메커니즘 (Massive Ingestion)
* **상황**: 사용자가 폴더나 zip 파일에 정리되지 않은 수백 개의 PDF, 논문, 코드를 마구잡이로 투척.
* **Graphify 혁신**: 에이전트는 텍스트를 정독하며 허덕이는 대신, 백그라운드 워치독이 Tree-sitter와 LLM을 동원해 즉시 키워드 연관성 및 계층 구조(상위➔하위 개념) 뼈대를 세워버립니다.
* **결과**: 아무리 방대하고 지저분한 자료라도 단숨에 완벽한 지형도로 치환되어 AI의 뇌에 완벽히 흡수(폭식)됩니다.

### 시나리오 F: 초격차 역질의 티키타카 (Fact-based Reverse-Questioning)
* **상황**: 기획자가 "저번 자료 바탕으로 환불 로직 좀 기획해봐"라고 막연하게 지시.
* **기존 한계**: AI가 두루뭉술하게 대충 뻔한 답변을 늘어놓음.
* **Graphify 혁신**: 지식망을 이미 꿰뚫고 있는 에이전트는 추론하지 않습니다. 즉시 그래프를 탐색하여 충돌 지점을 발견해냅니다.
* **결과**: *"대표님, 환불 로직(Node A)은 전자상거래법(Node B)과 연결되어 있으나, 기존 PG 모듈(Node C)과 충돌합니다. 예외 처리를 먼저 설계할까요?"* 라며, 사람의 생각 잔가지를 쳐주는 **압도적인 조언자(Advisor)** 역할을 수행합니다.

## 3. 개발 우선 순위 및 로드맵 (Development Roadmap)

### [Step 1] Graphify 코어 인프라 연동 (우선순위: 최상)
1. **환경 구축**: `graphifyy` 패키지 설치 및 Python 기반 MCP(Model Context Protocol) 서버 데몬 설정.
2. **배치 크롤러 파이프라인**: 
   - `graphify --update` 명령어를 래핑하여, 칸반의 카드가 `DONE` 처리되어 산출물(OUTPUT)이 나올 때마다 주기적/실시간으로 프로젝트 폴더를 스캔하여 `graph.json`과 `GRAPH_REPORT.md`를 갱신하는 백엔드 워치독 구축.

### [Step 2] AI 에이전트(Executor)에 하이퍼쿼리 스킬 장착
1. **MCP 도구 주입**: 
   - `executor.js`에서 AI가 구동될 때, 기존의 무식한 `grep_search` 툴 대신 Graphify MCP 서버의 `query_graph`, `get_node`, `shortest_path` 도구를 장착.
2. **시스템 프롬프트 개편**: 
   - "코드를 탐색할 때 무조건 파일을 읽기 전에 Graphify 도구를 사용해 지형을 먼저 파악하라"는 룰(Rule) 주입.

### [Step 3] 프론트엔드: 아카이브 & 지식 그래프 뷰어
1. **아카이브 페이지 신설**: 대시보드 GNB에 '지식베이스 허브' 탭 신설.
2. **일괄 아카이브**: 메인 보드의 `DONE` 카드를 아카이브로 밀어내는 버튼 추가 (보드 청결도 향상).
3. **Graph HTML 임베딩**: Graphify가 자동 생성하는 `graph.html`(시각화 뷰어)을 Iframe 등의 형태로 아카이브 상세 페이지에 띄워, 사용자가 프로젝트의 거시적 맥락을 한눈에 파악하게 제공.

### [Step 4] 도메인 확장 (추후 과제)
1. 법령(`legalize-kr`), 특허, 의료 논문 등 대규모 텍스트/마크다운 문서를 통째로 Graphify에 태워 거대한 '도메인 에이전트(Domain Agent)' 모델 개발.

## 4. 아키텍처 의사결정 요약 (기존 대비 변경점)
* **FTS5 (SQLite 전문 검색) 역할 축소**: 기존에는 FTS5를 메인 검색 엔진으로 기획했으나, 단순 텍스트 매칭의 한계로 인해 **검색 UI의 보조적 Fallback 역할**로 축소합니다.
* **Graphify로의 전면 전환**: 시스템의 진정한 '지식'과 '문맥'의 이해는 벡터 임베딩(Vector DB)이 아닌, **NetworkX & Leiden 군집화 기반의 지식 그래프(Graphify)**가 100% 담당합니다.
