# mycrew 프로젝트 폴더 구조

## 번호 체계
번호(01~08)는 **정렬용**이며 우선순위와 무관합니다.

| # | 폴더 | Paperclip 프로젝트 | 용도 |
|---|---|---|---|
| 01 | 전략 | Plan | 전략 문서, 리서치 산출물 |
| 02 | 브랜딩 | Branding | 브랜드 에셋, 가이드 |
| 03 | 온보딩 | Onboarding | 내부 업무 규칙, 메모리 체계 |
| 04 | 스킬 | 스킬 수집 | 재사용 가능한 업종별 노하우·템플릿 |
| 05 | 시스템 | 시스템 | bootstrap, 인프라, 시스템 스크립트 |
| 06 | 웹사이트 | 웹사이트 | mycrew.run 운영 문서 |
| 07 | 가이드 | 가이드 | 고객 전달 가능 매뉴얼 |
| 08 | 레퍼런스 | 레퍼런스 | 참고 자료, 프로토타입 코드 |

## 핵심 구분
- **스킬 폴더(04)** = 재사용 가능한 업무 노하우, 템플릿, 체크리스트
- **프로젝트 폴더(향후 09~ 추가)** = 특정 캠페인/프로젝트의 실행 산출물
- **온보딩(03)** = 내부 규칙 (내부용)
- **가이드(07)** = 고객 전달 가능 자료

## 크로스 프로젝트 참조 규칙
- 문서는 가장 관련성 높은 폴더에 **원본 1곳** 저장
- 다른 폴더에서 필요하면 `_참조.md` 파일로 원본 경로 기재
- 심볼릭 링크는 지양
