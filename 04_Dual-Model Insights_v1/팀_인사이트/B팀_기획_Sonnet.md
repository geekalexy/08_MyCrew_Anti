## 1. 실험이 증명한 핵심 원칙

오늘 실험(루카 vs 소넷)에서 확인된 것:
- 같은 역할, 다른 모델 → **겹치는 게 아니라** 서로 없는 걸 가지고 있음
- Gemini: 시스템 설계·거버넌스·감성 카피 강함
- Claude: 데이터 정밀도·채널 구조·변수 시스템 강함
- → 두 팀은 **"경쟁"이 아니라 "상류→하류 파이프라인"** 으로 설계해야 시너지

## 2. 크루 듀얼-모델 조합 3가지

### 조합 1 — "미러 크루" (완전 대칭형)

```
📌 구조: 동일 역할 × 2 (모델만 다름)

기획팀 A (Gemini)          기획팀 B (Claude)
  ├── 전략가 A                 ├── 전략가 B
  ├── 카피라이터 A              ├── 카피라이터 B
  └── 분석가 A                 └── 분석가 B
         │                            │
         └──────────┬─────────────────┘
                    ▼
              CMO (비교·병합·승인)
```

- **언제 쓰나**: 고객 향 중요 캠페인, 신규 이벤트 기획 첫 안
- **시너지**: 루카의 감성 후킹 + 소넷의 구조적 매뉴얼 → 자동 병합
- **단점**: 비용 2배, 속도 느림 → 중요한 건에만 사용

---

### 조합 2 — "스트림 분업형" ⭐ 기본 추천

```
📌 구조: 상류(Gemini) → 하류(Claude) 파이프라인

[A팀: Gemini 전략 스트림]
  아리 → CMO(Gemini) → 전략가(Gemini) → 카피라이터(Gemini)
                │
          지시서 + 뼈대 전달
                │
                ▼
[B팀: Claude 실행 스트림]
  콘텐츠 디렉터(Claude) → SEO(Claude) → 분석가(Claude)
                │
          결과물 → CMO Review
                │
                ▼
          CEO (대표님) 승인
```

- **언제 쓰나**: 일상 운영 (속도 + 품질 균형)
- **시너지**: 루카가 "뭘 만들지" 설계 → 소넷이 "어떻게 완성하지" 실행
- **장점**: 비용 효율적, 자연스러운 분업

---

### 조합 3 — "도전자 모드" (크로스체크, 프라임 제안 구현)

```
📌 구조: 평소엔 단일팀, 중요할 때만 크로스체크 발동

평소:
  아리 → 마케팅팀(Gemini) → CMO Review → CEO

중요 캠페인 감지 시 아리가 자동 분기:
  아리 ──→ A팀 (Gemini)  → A안
       └──→ B팀 (Claude) → B안
                    │
            CMO가 좌/우 비교
                    │
             병합안 → CEO
```

**트리거 조건 예시:**
- 예산 100만원 이상 캠페인
- 신규 채널 첫 진입
- CEO 반려 2회 이상 발생 시

## 3. 구체 크루 명단

| 크루명 | 모델 | 팀 | 핵심 스킬 |
|--------|------|----|----|
| **전략가 루카** | Gemini Pro | A팀 | `/태스크_병렬할당`, `/1차_실무비평` |
| **카피라이터 G** | Gemini Flash | A팀 | `/VOC_카피추출`, 감성 후킹 |
| **콘텐츠 디렉터 C** | Claude Sonnet | B팀 | 멀티채널 템플릿, 변수 시스템 |
| **SEO 에이전트** | Claude Sonnet | B팀 | 키워드, 해시태그, 출처 |
| **분석가** | Claude Sonnet | B팀 | KPI 집계, 성과 리포트 |
| **프라임** | Claude Opus | 심판 | 교차검증, 최종 의견서 |

> Gemini Flash = 속도 우선 (일상 카피)
> Gemini Pro = 품질 우선 (전략·설계)
> 비율 조절로 비용 컨트롤 가능

## 4. MyCrew UI 구현 포인트

```
① 크루 설정 화면 — "모델 백엔드" 선택 옵션
   → Gemini Flash / Pro / Claude Sonnet / Opus 선택

② 태스크 생성 — "크로스체크 모드" 토글
   → ON:  두 팀에 동시 할당 + 비교 뷰
   → OFF: 단일팀 (평상시)

③ 결과 비교 UI
   → 좌/우 분할 또는 탭 형태로 A안/B안 나란히
   → "이 부분 병합" 드래그로 최종안 조합
```

## 5. 운영 권장 방식

```
기본: 조합 2 (스트림 분업) — 일상 운영
      루카팀이 전략 → 소넷팀이 콘텐츠 실행

예외: 조합 3 (크로스체크) — 중요 캠페인
      동일 과제 → A안/B안 → CMO 비교 → 병합

판정: 프라임(Opus) — 분기별 or 전략 의사결정 시
      편향 없는 제3자 검증
```
