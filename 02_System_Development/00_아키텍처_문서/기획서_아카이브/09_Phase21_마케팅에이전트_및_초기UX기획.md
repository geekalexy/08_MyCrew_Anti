# Phase 21: 제로 픽션(Zero-Friction) 컨텍스트 수집 및 마케팅 에이전트 팀 UX 기획

## 1. 개요 (Overview)
기존의 복잡한 입력형 온보딩을 우회하여, SaaS 가입 직후 **가장 간편한 방법(웹사이트 URL 또는 스크린샷 1장)**만으로 AI가 기업의 맥락을 스스로 학습하고, 즉시 실무에 투입 가능한 **'마케팅 에이전트 팀'**을 구성하여 유저에게 "Wow Moment"를 선사하는 포스트 온보딩(Post-Signup) UX 기획입니다.

## 2. '컨텍스트 수집' 핵심 힌트 적용 전략
참고 문서(유튜브 대본)에서 얻은 가장 강력한 힌트는 **"웹사이트 스크린샷 1장으로 브랜드의 모든 맥락(디자인, 비즈니스, 톤앤매너)을 한 번에 마크다운(MD)으로 추출해내는 방식"**입니다.

### [기존 방식]
- 유저 타겟층은 누구인가요? 
- 브랜드 키 컬러는 무엇인가요?
- 직접 가이드라인을 작성해주세요. (이탈률 증가)

### [개선된 UX: Zero-Friction 방식]
1. 유저 가입 직후 초기 화면: **"회사의 웹사이트 주소를 입력하거나, 메인 페이지 스크린샷을 업로드해주세요."**
2. 백그라운드 처리 (아리 & 루카):
   - 비전 모델(Gemini 1.5 Pro)을 활용하여 웹사이트 UI/UX, 텍스트 카피, 아이콘 스타일 분석.
   - 테넌트 메모리(11_Ari_저장소) 내 `context/` 폴더에 즉시 3대 핵심 자산 자동 생성:
     - `brand_guideline.md` (톤앤매너, 핵심 가치, 키워드)
     - `design_style_guide.md` (타이포그래피, 컬러, 무드)
     - `business_context.md` (타겟 고객, 페인포인트, 솔루션)
3. 템플릿/SOP 초기화:
   - 수집된 맥락을 바탕으로 프로젝트 전체를 관장하는 업무 지침서(`claude.md` 또는 `MyCrew_SOP.md`) 자동 작성.

---

## 3. 마케팅 에이전트 팀 (Marketing Agent Team) 실무 적용안

유저의 컨텍스트 파악이 완료됨과 동시에, MyCrew 워크스페이스 내에 아래의 4단계 마케팅 특화 에이전트 파이프라인이 자동 활성화 및 배포됩니다.

### 🧩 팀 구성 (4-Step Pipeline)
1. **Research Agent**: `business_context.md`를 기반으로 시장 트렌드, 타겟 고객 반응 분석.
2. **Content Agent**: `brand_guideline.md`에 맞춘 톤앤매너로 뉴스레터, 소셜미디어 포스팅 텍스트(카피) 작성.
3. **Design Agent**: NanoBanana MCP(Gemini Image API) 등과 연동. `design_style_guide.md`를 준수하여 카드뉴스, 유튜브 썸네일, PPT 디자인 생성.
4. **Review Agent**: 최종 산출물이 초기 가이드라인과 SOP 규칙을 벗어나지 않았는지 상호 교차 검증 (크로스 체킹).

## 4. 사용자 경험(UX) 시나리오 (가입 후 첫 3분)

### 🔹 Step 1: "URL만 던져주세요" (T+0:00)
- 화면 중앙에 거대한 입력창 하나. "당신의 브랜드를 이해하기 위해 홈페이지 주소 하나면 충분합니다."
- 사용자가 `joshua.site` 입력 후 엔터.

### 🔹 Step 2: "AI가 브랜드를 스캔 중입니다..." (T+0:15)
- 터미널이나 대시보드 화면에서 에이전트들이 활동하는 모습 시각화.
- "로고 폰트 리버스 엔지니어링 중...", "브랜드 컬러 추출 중...", "타겟 고객군 모델링 중..." 메시지 실시간 노출.

### 🔹 Step 3: DNA 추출 완료 및 확정 (T+0:45)
- AI가 분석해낸 브랜드 키워드 3가지, 메인 컬러 팔레트, 브랜드 페르소나가 아름다운 UI 카드로 제시됨.
- 유저 액션: "완벽해요, 이대로 시작할까요?" (또는 가벼운 텍스트 수정 허용)

### 🔹 Step 4: 첫 번째 마케팅 캠페인 자동 시연 (T+1:30) 🎉 [WOW MOMENT]
- "마케팅 에이전트 팀이 구성되었습니다! 가입을 축하하는 기념으로, 고객님의 브랜드 가이드라인에 맞춰 내일 SNS에 올릴 카드뉴스와 뉴스레터를 이미 작성해 두었습니다."
- 화면 우측 패널에 **생성된 카드뉴스 5장(Design Agent)**과 **뉴스레터 본문(Content Agent)** 렌더링.
- **[예약 발행하기 (Buffer API 연동)]** 버튼을 바로 클릭할 수 있도록 유도.

## 5. 설계 시 고려사항 (Next Steps)
- **비용 최적화**: 가입 시 이미지/비전 모델을 한 번 크게 사용하지만(Gemini 1.5 Pro 스캔용), 이후 반복 작업은 Flash 기반 프롬프트 주입으로 처리하여 토큰 최적화.
- **자동화 인프라**: 소셜 예약(Buffer API) 등의 토큰은 사용자가 직접 세팅에서 기입할 수 있도록 온보딩 최후반부에 Security Vault 페이지 연결 필요.
- **템플릿 유연성**: 사용자가 원할 경우 언제든 자신이 쓰는 PPT 템플릿이나 기존 결과물을 Drag & Drop 하면, 에이전트가 이를 새로운 템플릿(md)으로 구조화하여 학습하는 경로 추가.
