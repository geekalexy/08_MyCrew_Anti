## 각 사이트의 연구 기여

### 사이트 1 — 소시안 (마케팅 SaaS)
- **태스크**: 캠페인 기획, 콘텐츠 제작, 온보딩 자동화
- **강점**: 파일럿 데이터 이미 존재 (2026-04-16 실험)
- **역할**: 파일럿 → Round 1 기준점

### 사이트 2 — AI/스타트업 뉴스미디어
- **태스크**: 기사 작성, 콘텐츠 기획, 에디토리얼 워크플로우
- **강점**: 콘텐츠 생성 도메인 — 소시안과 유사하나 목적 다름
- **역할**: 도메인 변형 검증 (마케팅 vs 저널리즘)

### 사이트 3 — 외주 개발사 ⭐
- **태스크**: 요구사항 도출(RE), 기획서 작성, 클라이언트 커뮤니케이션
- **강점**: 연구계획서 Phase 1 태스크와 **정확히 일치**
  > *"IT 비숙련 사용자를 위한 요구사항 도출 챗봇 시스템 기획"*
- **역할**: 연구 핵심 태스크의 실제 현장 검증

## 연구 설계상 의미

### 한계점 → 기여점 전환

```
기존 한계 (소넷 검토의견 Section ②):
  "단일 태스크 결과가 타 도메인에 적용되는지 별도 검증 필요"

3개 사이트 확보 후:
  소시안        → 마케팅 도메인
  뉴스미디어    → 미디어/에디토리얼 도메인
  외주 개발사   → IT 요구사항 도메인

결과: 3개 도메인 교차 검증 = 일반화 가능성 주장 가능
    한계점이 기여점으로 전환됨
```

### 논문 Contribution 추가 가능
> "본 연구는 단일 실험실 환경이 아닌, 이질적인 3개 실제 비즈니스 도메인
> (마케팅 SaaS / 미디어 / IT 서비스)에서 CKS 프레임워크를 검증함으로써
> 결과의 외적 타당성(External Validity)을 확보하였다."
