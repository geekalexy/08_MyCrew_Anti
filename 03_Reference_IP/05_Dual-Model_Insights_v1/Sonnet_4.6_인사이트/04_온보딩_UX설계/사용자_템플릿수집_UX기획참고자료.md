## 📌 목적

사용자가 템플릿 생성을 AI 에이전트에게 요청할 때,
에이전트가 최적의 결과를 도출할 수 있도록
**프롬프트 설계 기준과 개선 패턴**을 정리한 참고 문서.

---

## 1. 원본 프롬프트 문제점 진단 (실험 기반)

실제 실험에서 에이전트가 막혔거나 임의로 결정해야 했던 포인트:

| # | 문제 | 영향 |
|---|------|------|
| ① | **폴더 경로 오류** (잘못된 유저명) | 작업 전체 블로킹 |
| ② | **저장 위치 미지정** | 에이전트가 임의로 경로 선택 |
| ③ | **컨텍스트 파일 연결 없음** | 이미 만든 v3 파일 참조 안 됨 |
| ④ | **템플릿 수량 미지정** ("여러 개") | 에이전트가 임의로 결정 |
| ⑤ | **채널별 포맷 규격 없음** | 릴스 길이 등 임의 판단 |
| ⑥ | **이벤트 수치 전혀 없음** | 전부 `{{변수}}`로 처리 → 미완성 |
| ⑦ | **기존 자산 활용 지시 없음** | 폴더 뒤져서 추론만 함 |
| ⑧ | **출력 파일명 규칙 없음** | 에이전트가 임의로 결정 |

[CONTEXT] 컨텍스트 파일 경로:
  - {{workspace}}/{{팀폴더}}/컨텍스트/비즈니스_맥락3.md
  - {{workspace}}/{{팀폴더}}/컨텍스트/브랜드_가이드라인3.md
  위 두 파일을 반드시 먼저 읽고 작업할 것.

[REFERENCE_ASSETS] 기존 콘텐츠 참고 자산:
  - {{workspace}}/{{고객자료폴더}}/컨텐츠/
  위 폴더의 파일 구조를 파악하고, 기존 블로그 글체·캐러셀 구성·릴스 포맷을
  템플릿에 반영할 것.

[OUTPUT_DIR] 저장 경로:
  {{workspace}}/{{팀폴더}}/템플릿/

## 3. 핵심 개선 원칙 3가지

### 원칙 1: 컨텍스트 파일은 항상 명시 연결
```
❌ 나쁜 예: "소시안에 맞게 작성해줘"
✅ 좋은 예: "비즈니스_맥락3.md 파일을 반드시 먼저 읽고 작업할 것"
```
→ 브랜드 보이스, 가격, 해시태그를 자동 반영 가능

### 원칙 2: 출력 파일 구조는 사전 정의
```
❌ 나쁜 예: "채널별 템플릿 여러 개 작성해줘"
✅ 좋은 예: "파일명/포함 내용/형식/수량을 각각 명시"
```
→ 에이전트의 임의 판단 제거, 일관성 확보

### 원칙 3: 미확정 값은 변수로 명시
```
❌ 나쁜 예: (이벤트 수치 없이 진행)
✅ 좋은 예: "선착순 N명, 금액 {{금액}}원 — 미확정 시 변수 처리"
```
→ 실용적 미완성 상태 허용, 나중에 채워 넣기 용이

## 4. 폴더 경로 실수 방지 체크리스트

온보딩 시 에이전트에게 전달하는 경로 정보 표준화:

```
✅ workspace_root: /Users/alex/Documents/08_MyCrew_Anti/
✅ 팀폴더 패턴: XX_{{팀명}}{{고객이니셜}/
✅ 컨텍스트 경로: {{팀폴더}}/컨텍스트/
✅ 템플릿 경로: {{팀폴더}}/템플릿/
✅ 고객자료 경로: 06_{{고객명}}자료/
```

> 💡 모든 경로는 온보딩 초반 폴더 생성 단계에서
> README.md에 자동 기록해두면 이후 참조 경로 오류 방지 가능.

## 5. 사용자가 제공해야 할 최소 정보 체크리스트

에이전트가 템플릿을 제대로 만들기 위해 사용자에게 받아야 할 정보:

### 필수 (없으면 임의 판단)
- [ ] 캠페인/이벤트명
- [ ] 이벤트 기간 (시작일 ~ 마감일)
- [ ] 혜택 내용 (구체적 수치 포함)
- [ ] 참여 방법 (댓글/링크/폼 등)
- [ ] 홍보 채널 목록

### 권장 (있으면 품질 향상)
- [ ] 기존 콘텐츠 파일/폴더 경로
- [ ] 컨텍스트 파일 경로
- [ ] 채널별 포맷 규격 (크기, 길이)
- [ ] 원하는 템플릿 수량

### 선택 (없어도 추론 가능)
- [ ] 레퍼런스 이미지/영상
- [ ] 톤앤매너 특이사항
- [ ] 경쟁사 참고 자료
