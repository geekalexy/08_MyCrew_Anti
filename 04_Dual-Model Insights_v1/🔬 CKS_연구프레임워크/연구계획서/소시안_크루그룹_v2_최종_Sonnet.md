# 소시안 크루 그룹 최종 확정안 v2
## CKS 실험 — Group A (적대적) × Group B (비적대적) 최종 크루 + 스킬 리스트

> 작성자: 소넷 (Claude Sonnet 4.6)
> 작성일: 2026-04-17
> v1 대비 변경: 스킬 과다 우려 철회 + 워크플로우 라우팅 구조 반영 + 스킬 최종 확정
> 핵심 전제: 스킬은 "창고", 워크플로우 템플릿은 "지게차" — 호출당 1~2개만 활성화

---

## 🏗️ 구조 원칙

```
① 스킬 과다 = 부작용 없음
   → 워크플로우 템플릿이 라우팅 → 트리거마다 해당 스킬 1~2개만 활성화

② 워크플로우 벗어나는 요청
   → 스킬 多 = 임기응변 가능 (더 유리)
   → 스킬 無 = 동일하게 실패 (더 불리)

③ 진짜 리스크
   → ARI(0-2)의 라우팅 정밀도 — 스킬 수 문제 아님

④ TEI 측정 정규화 필수
   → 단위 품질당 토큰 비용 = Total_Tokens / (RFS × Task_count)

⑤ Team B Phase 조율
   → Phase Coordinator (Python Router, 토큰 0) 필요
```
