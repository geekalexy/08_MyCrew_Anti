# MyCrew 칸반 카드 및 상세 모달 UI/UX 전면 개편안

> **작성자**: Luca (CTO / System Architect)
> **최종 갱신일**: 2026-04-17
> **도메인**: 대시보드 - 프로젝트 칸반 UI 컴포넌트 개선
> **배경**: 현재 노출되는 카드 뷰 및 상세 팝업의 복잡도를 낮추고, 텍스트 가독성을 노션(Notion) 수준으로 끌어올리며, 결과물(아티팩트) 다이렉트 수정 경험을 극대화하기 위함.

---

## 1. 뷰 보드 칸반 카드 (Kanban Board Card)
보드 상에서 눈에 띄는 불필요한 시각적 노이즈를 제거하여 피로도를 낮춥니다.

* **입력 폼 제한**: 카드 전면에서는 복잡한 정보 노출을 최대한 생략하고 **`[타이틀]`, `[담당자(할당)]`, `[우선순위]`** 단 3가지만 기입 및 노출되도록 심플하게 제한.
* **불필요한 아이콘 제거**: 카드 우측 상단, 하단 등에 떠 있던 **'확장 아이콘'을 제거**하여 카드의 미니멀리즘 유지.

---

## 2. 카드 상세 팝업 모달 (Card Detail Modal)
카드를 클릭했을 때 뜨는 팝업 화면을 '노션(Notion)'의 쾌적한 룩앤필로 개편합니다.

* **레이아웃 사이즈**: 가로 폭을 **노션의 팝업 모달과 완전히 동일한 사이즈**로 넓혀 시원한 개방감 확보.
* **확장 아이콘 재배치**: 보드 카드에서 없앤 '전체화면 확장 아이콘(↗)'을 팝업 모달의 **좌측 상단**으로 재배치하여 명확한 동선 제공.
* **내용 영역 개방 (Borderless)**: 텍스트가 들어가는 본문 영역에 갇힌 느낌을 주는 박스(Box) 테두리나 배경 음영 처리를 과감히 삭제하고, 여백 구조를 크게 확보하여 탁 트인 캔버스 뷰 제공.
* **가독성(Line-height) 개선**: 현재 AI 크루가 출력한 결과물이 행 구분 없이 빽빽하게 뭉쳐 나오는 문제를 해결. 문단(Paragraph) 간의 `margin-bottom`을 넓히고 `line-height`를 최소 `1.6` 이상으로 적용하여 가독성을 대폭 개선.
* **지시-수행 흐름 UI (댓글/로직)**: 대화나 히스토리 구성 요소에서 `[지시자 ➝ 할당받은 자]` 화살표 아이콘을 추가하여, 누가 누구에게 일을 시키고 리포트했는지 위상(Topology) 흐름을 한눈에 인지하도록 개선.
* **[프리뷰(아티팩트)] 버튼 추가**: AI 크루의 최종 산출물이 이미지나 미디어일 경우, 본문 어딘가에 파묻히지 않게 상단이나 우측에 명시적인 `[프리뷰 보기]` 엑스트라 버튼 배치.

---

## 3. 확장 페이지 뷰 (Full-Page Expanded View)
팝업 좌상단의 확장 아이콘을 눌러 전체화면으로 넘어갈 때의 뷰 처리.

* **정보 통합 배치**: 확장 페이지에서는 카드의 상세 메타데이터(날짜, 상태, 뱃지)부터 본문 내용까지 '모든 정보'를 시원하게 배치.
* **마크다운 WYSIWYG 블라인딩**: 노션처럼 입력 렌더러를 개선하여, 해시태그(`#`)나 별표(`**`) 같은 마크다운 기호가 눈에 띄지 않도록 원본 텍스트에 적용된 서식만 렌더링(Raw 블라인딩 처리).

---

## 4. 프리뷰(아티팩트) 모드 & 인라인 에디팅 (UX 킬러 기능)
아티팩트 버튼을 클릭했을 때 발동하는 특수 목적 뷰.

* **집중 모드 발동**: 상세 구성 요소(작업자, 일정, 태그) 등은 모두 블라인드 처리(Hide)되고, 오직 우아한 **`[제목]`**과 **`[이미지/결과물]`**만 중앙에 풀스크린 뷰로 단독 렌더링됨.
* **클릭 투 에딧 (Click-to-Edit) / AI 채팅 오버레이**: 
  * 결과물 아래의 텍스트 컴포넌트를 유저가 클릭하면 바로 직접 수정(Inline Edit)이 가능한 포커스 모드로 전환.
  * 혹은 텍스트를 클릭 시 우측에서 플로팅 채팅창(혹은 인라인 채팅 팝업)이 팝업되며, *"이 부분 배경색 좀 다크 모드로 바꿔줘"* 라고 텍스트 입력만 하면 AI가 백그라운드에서 해당 객체를 도려내어 바로 수정하는 마법 같은 경험 제공.
