# 사용자 온보딩 컨텍스트 수집 UX 가이드

> 작성: 2026-04-16 | 분류: SaaS 사용자 온보딩 | 작성자: Luca
> 관련 파일: ARI_MASTER_SOP.md

---

## 📌 목적

신규 사용자 온보딩 시, 아리 크루가 고객사 맞춤형 업무를 수행하기 위해
반드시 수집해야 하는 핵심 두 가지 정보를 효과적으로 받아내기 위한 UX 가이드.

### 수집 목표 정보
1. **핵심 고객**: "우리의 핵심 고객은 누구인가?"
2. **해결 문제**: "우리가 해결하는 문제는 무엇인가?"

---

## 방법 1: 채팅 대화 유도 스크립트 (아리)

> 텔레그램 또는 대시보드 채팅에서 아리가 사용자를 단계별로 이끄는 대화 스크립트

---

### [STEP 1] 도입 — 왜 필요한지 먼저 설명

```
아리:
안녕하세요 👋 {{company_name}} 팀이 합류하셨네요!

저희 크루가 {{company_name}}를 위해 제대로 일하려면
딱 두 가지만 먼저 알아야 해요.

➊ {{company_name}}의 핵심 고객이 누구인지
➋ 그 고객의 어떤 문제를 해결해주는지

지금 바로 알려주시면, 오늘부터 크루가
{{company_name}} 스타일대로 콘텐츠를 만들 수 있어요 🚀

준비되셨으면 "시작" 이라고 입력해주세요!
```

---

### [STEP 2] 핵심 고객 수집 — 선택지 제공 후 직접 입력 유도

```
아리:
첫 번째 질문이에요!

✅ {{company_name}}의 핵심 고객은 누구인가요?

아래 중 가장 가까운 것을 선택하거나
직접 입력해 주셔도 됩니다 😊

 1️⃣ 인스타그램으로 공구하는 인플루언서
 2️⃣ 인스타그램에서 직접 판매하는 셀러
 3️⃣ 브랜드 공식 계정을 운영하는 기업
 4️⃣ 직접 입력할게요 →

(예시: "팔로워 1만~10만의 인스타 공구 인플루언서")
```

**[사용자 입력 후 확인 응답]**
```
아리:
완벽해요 👍

"{{user_answer}}"
→ 저장했어요 ✅

바로 두 번째 질문으로 넘어갈게요!
```

---

### [STEP 3] 해결 문제 수집 — Before/After 프레임 활용

```
아리:
두 번째 질문이에요.

✅ {{company_name}}이 없었다면 그 고객들은
   어떤 불편함을 겪고 있을까요?

이렇게 생각해보시면 쉬워요 👇

"{{company_name}} 없이는 _____ 때문에 힘들었는데,
 {{company_name}}을 쓰면 _____ 가 해결돼요."

빈칸을 채워주시거나 자유롭게 말씀해 주세요!

(예시: "DM을 수동으로 하나하나 보내느라
        몇 시간을 썼는데, 이제 자동으로 되니까
        콘텐츠에만 집중할 수 있어요")
```

**[사용자 입력 후 확인 응답]**
```
아리:
역시! 핵심을 딱 짚어주셨네요 💡

해결하는 문제:
"{{user_answer}}"
→ 저장했어요 ✅

이 두 가지 정보로 크루가
{{company_name}} 맞춤 콘텐츠를 만들기 시작할게요!

마지막으로 웹사이트나 블로그 링크가 있으면
공유해주시면 더 완벽하게 세팅해 드릴 수 있어요 🙏
(없으면 "없어요"라고 입력해주세요)
```

## 방법 2: 카드 UX 유도문

> MyCrew 대시보드의 온보딩 카드에 사용하는 UX 카피 및 입력 필드 설계

---

### 카드 헤더

```
제목:    🎯 우리 크루 세팅하기 — 2분이면 충분해요

안내문:  크루가 {{company_name}}처럼 일하려면 딱 두 가지가 필요해요.
         아래를 채워주시면 오늘부터 바로 시작할 수 있어요.
```

---

### 입력 필드 1 — 핵심 고객

```
라벨:       우리 핵심 고객은 누구인가요? *

플레이스홀더:
  "인스타그램 공구를 진행하는 인플루언서와 소규모 셀러"

힌트 텍스트 (하단 회색):
  💡 구체적일수록 좋아요.
     "누구" + "어떤 상황의 사람인지" 까지 써주시면 최고예요.

     예시 1: "팔로워 5천~5만의 인스타 공구 인플루언서"
     예시 2: "인스타그램으로 직접 옷을 파는 20~30대 셀러"
     예시 3: "SNS 광고를 처음 시작하는 중소기업 마케터"
```

---

### 입력 필드 2 — 해결하는 문제

```
라벨:       우리 서비스가 없다면 고객은 어떤 불편을 겪나요? *

플레이스홀더:
  "DM을 손으로 하나하나 보내느라 몇 시간을 허비했는데,
   자동화하면 콘텐츠에만 집중할 수 있어요"

힌트 텍스트 (하단 회색):
  💡 "Before → After" 로 생각해보세요.

     [Before] 우리 없이는: _____이 힘들었다
     [After]  우리 덕분에: _____이 해결된다

     예시 1: "수동 DM 발송에 하루 3시간 → 자동화로 10분"
     예시 2: "인스타 팔로워가 안 늘었는데 → 자동DM으로 팔로우 증가"
     예시 3: "공구 마감 알림을 놓쳤는데 → 리타겟팅 DM으로 재구매 유도"
```

---

### 입력 필드 3 — 링크 (선택)

```
라벨:       웹사이트 또는 블로그 링크 (선택)

플레이스홀더:
  https://

힌트 텍스트:
  있으면 넣어주세요. 크루가 훨씬 더 정확하게 세팅돼요 😊
  (없어도 괜찮아요)
```

---

### 제출 버튼 & 완료 메시지

```
버튼 텍스트:   🚀 크루 세팅 시작하기

버튼 하단 안내:
  제출하면 크루가 {{company_name}} 맞춤 세팅을 바로 시작해요.
  보통 5분 이내로 완료돼요.
```

```
[완료 후 피드백]

✅ 세팅 완료!

크루가 {{company_name}}의 핵심 고객과 해결 문제를 파악했어요.
이제 {{company_name}} 스타일로 콘텐츠를 만들 준비가 됐습니다 💪

첫 번째 태스크를 만들어 볼까요?
→ [+ 새 태스크 만들기]
```

## 두 방법 비교

| 항목 | 채팅 방식 | 카드 방식 |
|------|--------|--------|
| **사용자 부담** | 낮음 (아리가 이끌어줌) | 중간 (혼자 작성) |
| **입력 품질** | 높음 (대화로 구체화) | 보통 (예시 의존) |
| **적합한 사용자** | 처음 쓰는 사람, 글 못 쓰는 사람 | 바쁜 사람, 익숙한 사람 |
| **구현 복잡도** | 높음 (대화 흐름 설계 필요) | 낮음 (폼 UI) |
| **권장 시점** | 가입 직후 첫 대화 | 온보딩 카드 보완용 |

> 💡 **권장 구조**: 채팅으로 먼저 받고 → 카드에 자동 기입.
> 사용자는 대화만 하면 되고, 결과는 카드로 남아 추적 가능.
