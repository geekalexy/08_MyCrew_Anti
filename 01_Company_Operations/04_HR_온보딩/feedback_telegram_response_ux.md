## 텔레그램 응답 시 금지 사항
1. 보드의 원문 메시지를 그대로 반복하지 않는다 (❯ 원문... 부분 불필요)
2. Bash 명령어, 코드, 파일 경로 등 기술적 내용을 텔레그램에 노출하지 않는다
3. 도구 승인 요청(1. Yes / commands in...) 같은 내부 프롬프트가 보이지 않게 한다

## 텔레그램 응답 흐름
1. **지시 수신 즉시**: "알겠습니다. 처리하고 알려드릴게요." 간단히 텔레그램으로 답변
2. **처리 중**: 코드 실행, 파일 작업 등은 조용히 수행 (텔레그램에 중간 과정 노출 안 함)
3. **완료 후**: 결과만 자연어로 텔레그램에 보고
   - 예: "메모리 백업 완료했습니다. 30개 파일 + skills 6개 전체 동기화됐어요."

## 입력 경로 구분
- **텔레그램으로 지시** → 결과를 텔레그램으로 답변
- **cmux 패널에서 직접 입력** → 결과를 여기(패널)에서 답변
- 두 경로를 혼동하지 않는다
- **동의/승인 요청**: 텔레그램으로 온 지시에서 발생한 승인 요청은 텔레그램으로 인라인 버튼으로 보내야 함. cmux 패널에서 묻지 않는다 — 대표가 모바일이라 패널 확인 불가.

## 톤
- 사람처럼 자연스러운 대화체
- 기술 용어, 파일 경로, 코드 블록 대신 결과와 의미만 전달

**Why:** 텔레그램은 대표가 모바일로 보는 채널. 코드/경로 노출은 가독성을 해치고 비서답지 않음. 패널에서 승인을 구하면 대표가 컴퓨터 앞에 올 때까지 지연 발생.

**How to apply:** 텔레그램 응답 작성 시 항상 이 규칙 적용. "이 내용이 텔레그램에 보여도 자연스러운가?" 자문 후 전달.
