# 고능력자 AI 직원 채용 전략

## 핵심 콘셉트

Paperclip의 에이전트는 기본 설정(Instructions + Skills + Configuration)으로 동작한다.
mycrew의 차별화는 **C2(자비스)가 이 설정을 운영 경험 기반으로 지속 업그레이드**하여
더 능력 있는 AI 직원을 만들어내는 것이다.

```
Paperclip 기본:  에이전트 = 기본 Instructions + 기본 Skills
mycrew 차별화:   에이전트 = 기본 + C2가 축적한 Skills + 튜닝된 Configuration
                          ↑ 운영할수록 더 똑똑해짐
```

---

## 1. Paperclip 에이전트 설정 구조

각 에이전트(CEO, PM, CTO 등)는 다음 탭으로 구성됨:

| 탭 | 역할 | C2가 할 수 있는 것 |
|---|---|---|
| **Instructions** | 에이전트의 역할, 행동 규칙 | 운영 경험 기반으로 규칙 추가/수정 |
| **Skills** | 에이전트가 사용할 수 있는 기술 | 커스텀 스킬 추가 (API 연동, 도구 등) |
| **Configuration** | 세부 설정 (어댑터, 파라미터) | 정확한 값으로 수정/최적화 |

---

## 2. C2의 에이전트 업그레이드 프로세스

### 2-1. Skills 업그레이드

```
[관찰] C2가 에이전트 업무 결과를 모니터링
  ↓
[판단] "분석가가 시장조사 할 때 실시간 데이터가 필요하다"
  ↓
[실행] 분석가 > Skills 탭에 웹 검색 스킬 추가
  ↓
[결과] 다음 heartbeat부터 분석가가 실시간 데이터로 조사
```

스킬 추가 예시:

| 에이전트 | 추가 스킬 | 효과 |
|---|---|---|
| CEO | 팀 성과 분석 스킬 | 주간 리포트 자동 생성 |
| CTO | 코드 리뷰 스킬 | PR 자동 리뷰 |
| PM | 사용자 피드백 분석 스킬 | NPS 자동 집계 |
| 분석가 | 웹 리서치 스킬 | 실시간 경쟁사 모니터링 |
| CMO | SNS 분석 스킬 | 채널별 성과 트래킹 |

### 2-2. Configuration 튜닝

```
[관찰] 사용자가 Configuration에 대략적으로 기입
  ↓
[판단] C2가 운영 데이터 기반으로 최적값 파악
  ↓
[실행] Configuration 값 정밀 수정
  ↓
[결과] 에이전트 응답 품질/속도 향상
```

튜닝 예시:

| 설정 | 사용자 입력 (대략적) | C2 튜닝 (정밀) |
|---|---|---|
| 역할 설명 | "마케팅 담당" | "B2B SaaS 마케팅 전문, 콘텐츠+퍼포먼스 병행, 주 타겟: 중소기업 대표" |
| 응답 스타일 | "한국어로" | "한국어 보고체, 데이터 기반, 표/차트 활용, 결론 먼저" |
| 우선순위 | 미설정 | "P0: 매출 직결 업무, P1: 리드 생성, P2: 브랜딩" |

### 2-3. Instructions 고도화

```
[관찰] CEO가 팀 구성 시 매번 비슷한 실수를 반복
  ↓
[판단] "채용 시 역할 중복 체크 규칙이 필요하다"
  ↓
[실행] CEO Instructions에 "팀 구성 시 기존 멤버와 역할 중복 확인 후 채용" 규칙 추가
  ↓
[결과] 다음 채용부터 중복 없이 최적 구성
```

## 3. 자동 성장 사이클

```
운영 1주차: 기본 에이전트 (Paperclip 기본값)
    ↓ C2 모니터링 + 스킬 메모리 축적
운영 1개월: 튜닝된 에이전트 (업종 맞춤 설정)
    ↓ C2가 Skills/Configuration 업그레이드
운영 3개월: 고능력 에이전트 (노하우 반영된 전문가)
    ↓ 스킬 메모리가 분양 고객에게도 전달
분양 고객: 처음부터 3개월차 수준의 에이전트로 시작
```

## 4. 경쟁 차별화 포지셔닝

| | Paperclip 단독 | mycrew (Paperclip + C2) |
|---|---|---|
| 에이전트 설정 | 사용자가 직접 | C2가 자동 최적화 |
| 스킬 추가 | 사용자가 직접 | C2가 필요 시 자동 추가 |
| 성장 | 수동 | 운영할수록 자동 성장 |
| 분양 시 | 기본 설정 | 축적된 노하우 포함 |
| 비유 | 빈 사무실 임대 | 교육된 팀 파견 |

### 마케팅 메시지

> "mycrew의 AI 직원은 사용할수록 똑똑해집니다.
> 당신이 일하는 동안 자비스가 팀원들의 스킬을 업그레이드합니다."

## 5. 외부 스킬 소싱 전략

C2가 자체 경험으로 스킬을 축적하는 것 외에, **외부에서 검증된 스킬을 수집하여 팀원에게 장착**하는 전략.

### 소싱 채널

| 채널 | 설명 | 담당 |
|---|---|---|
| **Skillskit.sh** | 커뮤니티 스킬 마켓플레이스 | C1 + MYC 팀원 |
| **Claude 공식 스킬** | Anthropic 제공 검증된 스킬 | C1 + MYC 팀원 |
| **GitHub 오픈소스** | MCP 서버, 플러그인 등 | C1 + MYC 팀원 |
| **자체 개발** | 운영 중 필요한 커스텀 스킬 | C1 + MYC 팀원 |

### 소싱 프로세스

```
[수집] CTO + C1이 외부 스킬 채널을 주기적으로 탐색
  ↓
[평가] 우리 에이전트에 유용한지 판단
  - 보안 검토 (악성 코드, 데이터 유출 리스크)
  - 호환성 확인 (Paperclip 어댑터와 호환)
  - 성능 테스트 (sandbox 환경에서 시험)
  ↓
[보관] 스킬 라이브러리에 저장
  - mycrew/skills-library/ 폴더에 카테고리별 보관
  - 평가 결과 + 사용 가이드 문서화
  ↓
[장착] C2가 적절한 시점에 에이전트에 스킬 추가
  - 업무 패턴 분석 → "이 에이전트에 이 스킬이 필요하다" 판단
  - Paperclip Skills 탭에서 추가
  ↓
[검증] heartbeat 실행 후 성능 향상 확인
```

### 역할 분담

| 역할 | 담당 업무 |
|---|---|
| **C1 + MYC 팀원** | 외부 스킬 탐색, 평가, 검증, 스킬 라이브러리에 등록 |
| **C2 (각 회사)** | 라이브러리에서 자기 회사 팀원에게 필요한 스킬 선택 → 장착 → 효과 모니터링 |

> C2는 스킬을 수집/평가하지 않는다. 검증된 라이브러리에서 "쇼핑"만 한다.
> 분양된 모든 C2가 같은 스킬 라이브러리를 공유한다.

### 스킬 라이브러리 구조

```
mycrew/skills-library/
  ├── verified/           — 검증 완료, 즉시 사용 가능
  │   ├── web-research.md
  │   ├── data-analysis.md
  │   └── ...
  ├── evaluating/         — 평가 중
  │   ├── candidate-skill-1.md
  │   └── ...
  └── rejected/           — 부적합 판정 (사유 기록)
      └── ...
```

### 분양 가치

- 분양 고객은 mycrew가 축적한 **검증된 스킬 라이브러리**를 그대로 받음
- 고객이 직접 스킬을 찾고 평가할 필요 없음
- "교육된 팀 + 장비까지 갖춘 팀"을 제공하는 셈

## 6. 구현 로드맵

| 단계 | 내용 | 시점 |
|---|---|---|
| 1단계 | C2가 스킬 메모리에 노하우 축적 | 지금 |
| 2단계 | C1+MYC 팀원이 외부 스킬 소싱 시작 (skillskit.sh, Claude 공식) | 지금 |
| 3단계 | C2가 Paperclip Skills/Configuration 직접 수정 | Notion API 연동 후 |
| 4단계 | 자동 성장 사이클 구축 (모니터링→판단→업그레이드) | 3단계 안정화 후 |
| 5단계 | 분양 시 스킬 메모리 + 스킬 라이브러리 + 튜닝 설정 패키지 전달 | 분양 시작 시 |

## 7. C2 학습 내용 (검토 후 추가)

```
C2에게 추가 학습할 내용:
1. Paperclip 에이전트 Skills 탭에서 스킬 추가/수정 방법
2. Configuration 탭에서 설정 튜닝 방법
3. 운영 중 에이전트 성능 관찰 → 업그레이드 판단 기준
4. 스킬 메모리(skills/*.md)와 Paperclip Skills 탭 동기화
5. 스킬 라이브러리(skills-library/verified/)에서 팀원에게 필요한 스킬 선택
6. 선택한 스킬을 Paperclip Skills 탭에서 에이전트에 장착
7. 장착 후 효과 모니터링 → 스킬 메모리에 결과 기록
8. 새 스킬 장착 시 보드에게 "시스템 업데이트" 안내
```
