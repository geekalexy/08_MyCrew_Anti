## 1. 기획 배경 및 제약 조건 (Constraints)

현재 Antigravity IDE를 통해 브릿지 형태로 제공받는 **오픈라우터 계열 LLM(Claude 4.6 시리즈, GPT 등)은 하루 2시간의 사용 시간 제한(Quota)**이 존재합니다. 

따라서 기존처럼 모든 칸반 파이프라인(장시간 자율 주행)을 Claude에게 맡기면 업무 시간 도중 쿼터가 만료되는 치명적인 병목이 발생합니다. 본 기획서는 앞서 정의한 **3대 상위 워크플로우(Architect / Dev / Review 모드)**에 맞춰, **"시간이 오래 걸리는 중노동은 Gemini 3.1 Pro에게, 짧고 치명적인 정밀 타격은 Claude에게"** 배정하는 스킬 조합 및 모델 라우팅 아키텍처를 정의합니다.

## 2. 모드별 워크플로우와 스킬 & 모델 배정표

### 📐 [모드 1] 기획/설계 모드 (Architect Mode)
*목적: 사용자의 텍스트를 분석하여 V1.0 PRD를 도출하고 개발 스코프를 확정하는 짧은 단기 세션.*

*   **배정 모델**: **Claude Opus 4.6 (Thinking)**
    *   **선정 사유**: 기획은 전체 프로젝트의 성패를 좌우하지만, 한 번 기획을 뽑고 나면 대화가 길게 이어지지 않습니다(통상 10~20분 소요). 따라서 가장 똑똑한 최고 지성 모델(Opus)의 2시간 쿼터를 이곳에 아낌없이 투자하여 최고의 아키텍처 초안을 얻어냅니다.
*   **할당되는 MCP 스킬 (도구)**:
    1.  `analyze_scope`: Sequential Thinking을 활용해 사용자 요구사항의 모순을 찾고 우선순위를 쪼갭니다.
    2.  `split_roadmap`: 분석 결과를 바탕으로 `v1.0_MVP_PRD.txt`, `v1.1_PRD.txt` 물리적 파일을 생성합니다.
*   **💡 벤치마킹 MCP 차용 포인트**:
    *   **Sequential Thinking**: 요구사항을 바로 문서로 쓰지 않고, 다각도로 리스크를 검토하는 "사고 과정(Thought)"을 강제하기 위해 차용됩니다.
    *   **Claude Task Master**: 모든 작업의 기준을 단일 진실 공급원인 `PRD.txt`로 강제하는 설계 사상을 차용합니다.

### 💻 [모드 2] 개발/실행 모드 (Dev Mode)
*목적: 확정된 PRD를 바탕으로 수십 개의 태스크 카드를 만들고, 순차적으로 자율 코딩을 진행하는 장기 세션.*

*   **배정 모델 A (장시간 자율 주행 전담)**: **Gemini 3.1 Pro (High)**
    *   **선정 사유**: 전체 파이프라인을 따라 파일 수십 개를 만들고 수정하는 작업은 몇 시간이 걸릴 수 있습니다. 쿼터 제한이 없는 Gemini 3.1 Pro(High)를 메인 작업자로 배정하여 멈춤 없는 자율 주행을 보장합니다.
*   **배정 모델 B (정밀 타격 전담 / 스나이퍼)**: **Claude Sonnet 4.6 (Thinking)**
    *   **선정 사유**: Gemini가 풀기 힘든 매우 복잡한 UI 애니메이션이나 난해한 알고리즘 카드가 등장할 때만 사용자가 직접 `/execute [지시사항]`으로 Sonnet을 단기 소환(10분 이내)하여 고도화된 코드를 뽑아냅니다.
*   **할당되는 MCP 스킬 (도구)**:
    1.  `run_tasks`: Shrimp Task Manager 방식의 의존성 기반 자율 코딩 스킬.
    2.  `trace_bug`: 단순 텍스트 추론이 아닌, 에러 발생 시 **Graphify에 Cypher 쿼리를 날려 함수/의존성 그래프를 정확히 역추적(Trace-back)**한 뒤, Sequential Thinking으로 논리적 해결책을 도출하는 초고효율 디버깅 스킬. (토큰 소모 급감 및 정확도 상승)
    3.  `git_commit`: 작업 완료 시 AutoGit을 통해 버전을 안전하게 저장하는 스킬.
*   **💡 벤치마킹 MCP 차용 포인트**:
    *   **Shrimp Task Manager**: 기획서를 원자 단위(Atomic)의 태스크로 쪼개고, 의존성에 따라 서브 에이전트를 자율적으로 돌리는 '실행 파이프라인' 전체를 차용합니다.
    *   **Sequential Thinking**: 단순 코딩이 아닌, 에러가 터졌을 때 버그의 원인을 역추적(Debugging)하는 특수 모드로 차용됩니다.
    *   **Claude Task Master**: 개발 모드 진입 시 기획용 스킬(Tool)을 모조리 해제하고 코딩용 스킬만 로드하는 "토큰 최적화(Selective Loading)" 아키텍처를 차용합니다.

### 🕵️‍♂️ [모드 3] 리뷰 및 인프라 모드 (Review / QA Mode)
*목적: 작성된 코드 전체를 스캔하여 지식망을 업데이트하고, 잠재적 버그를 리포팅하는 백그라운드 세션.*

*   **배정 모델**: **Gemini 3.1 Pro (Low) 또는 Gemini 2.5 Pro**
    *   **선정 사유**: 프로젝트 전체 코드를 읽고(엄청난 토큰 소모) 노드와 엣지를 추출해 Graphify로 묶는 작업은 매우 무거운 백그라운드 작업입니다. 토큰 비용이 저렴하고 제한이 없는 모델을 배정합니다.
*   **할당되는 MCP 스킬 (도구)**:
    1.  `extract_graph`: 코드가 완성될 때마다 메타데이터와 함수 관계를 추출해 `Graph.json`을 업데이트합니다.
    2.  `audit_code`: PRD 대비 누락된 예외 처리나 보안 취약점을 검토합니다.
*   **💡 벤치마킹 MCP 차용 포인트**:
    *   **Claude Task Master**: 역할에 따라 Main Model(Claude)과 Research Model(Perplexity)을 완벽히 분리하여 API를 쏘는 '다중 모델 라우팅' 사상을 차용하여, 백그라운드 리뷰 작업을 비용이 싼 Gemini에게 100% 위임합니다.

## 3. 핵심 아키텍처: 시간 제한 방어 및 핫스왑 (Quota Defender)

Claude의 "일 2시간" 제약을 시스템 레벨에서 방어하기 위해 다음 로직을 마이크루 백엔드에 탑재합니다.

1.  **쿼터 모니터링 (Quota Monitoring)**:
    *   Antigravity 어댑터에서 반환되는 토큰 사용량과 세션 시간을 지속적으로 누적 계산합니다.
2.  **강제 핫스왑 (Fallback Hotswap)**:
    *   만약 사용자가 Sonnet을 켜둔 상태로 방치하거나 너무 긴 자율 주행을 지시해서 Claude의 남은 시간이 **15분 미만**으로 떨어지면, 시스템이 경고 알림("Claude 쿼터가 부족하여 Gemini 3.1 Pro로 실행 모델을 교체합니다")을 띄우고 자동으로 **Gemini 3.1 Pro**로 실행 엔진을 스위칭합니다.
3.  **명시적 도구 차단 (Tool Selective Loading)**:
    *   모드별로 꼭 필요한 도구만 로드합니다. 기획 모드(Opus)에서는 무거운 `AutoGit`이나 `graphify_extract` 도구를 로드하지 않아 토큰과 처리 시간을 극한으로 절약합니다. (Claude Task Master 벤치마킹 적용)
