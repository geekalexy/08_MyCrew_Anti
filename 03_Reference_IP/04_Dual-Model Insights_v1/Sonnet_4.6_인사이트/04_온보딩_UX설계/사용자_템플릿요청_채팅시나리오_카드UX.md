# 템플릿 자료 수집 요청 — 설득 중심 재설계

> 작성: 2026-04-16 (개선판) | 분류: 온보딩 UX 기획
> 전제:
>   1. 사용자는 이 작업이 귀찮고, 중요성을 낮게 인식한다
>   2. 컨텍스트+템플릿 자료가 크루 결과물 만족도의 90%를 차지한다
>   3. 사용자가 자주 접근하고 싱크해야 한다는 것을 주지시켜야 한다

---

# 🧠 설계 전략

## 기존 방식의 문제
```
기존: "자료 좀 주세요" → 귀찮음 → 안 줌 → 크루 결과물 별로 → 서비스 이탈
```

## 개선 방향
```
새 방식: "지금 5분이 앞으로 3개월을 결정해요"
  → 왜 중요한지 먼저 각인
  → 귀찮음보다 손해가 더 크다는 걸 느끼게 함
  → 최대한 쉽게 → 성공 경험 제공 → 습관화 유도
```

## 핵심 메시지 (전 채널 동일하게 유지)
> **"크루의 퀄리티 90%는 컨텍스트와 템플릿에서 나와요."**
> "AI가 아무리 똑똑해도, 우리 회사를 모르면 남의 회사 글을 쓰는 거예요."

# 💡 핵심 설계 원칙 (개선판)

| 원칙 | 기존 문제 | 개선 방법 |
|------|---------|---------|
| **설득 먼저** | 자료 줘→ 귀찮음 | "90% 결정" 각인 먼저 |
| **손해 시각화** | 중요성 몰라서 안 줌 | "안 주면 이렇게 돼요" 명시 |
