## 1. 🔍 배경 및 소놀봇(Sonolbot) 아키텍처 분석

대표님께서 제안해주신 소놀봇(Sonolbot)의 코덱스 ↔ 텔레그램 통신 구조를 벤치마킹하여, MyCrew의 모바일 통신 채널 확장을 검토합니다.

### 1.1. 소놀봇 핵심 아키텍처 요약
*   **구조**: `[텔레그램(모바일)] ↔ [소놀봇 데몬] ↔ [Codex/Claude CLI] ↔ [독립 샌드박스]`
*   **비용 무효화 (OAuth 세션 활용)**: API Key(종량제 과금)를 쓰지 않고 터미널에서 CLI 도구로 1회 로그인(OAuth)한 세션을 하이재킹. 소놀봇 데몬이 백그라운드에서 이 CLI 도구에 명령어를 주입하여 대답을 받아오므로, 월 구독료만으로 무제한에 가깝게 사용 가능.
*   **독립된 샌드박스**: AI가 코드를 짜고 직접 실행까지 하므로, 메인 시스템을 보호하기 위해 WSL(Windows Subsystem for Linux)이나 격리된 폴더 환경에서만 명령이 돌도록 구축.
*   **모바일 다이렉트 제어**: 텔레그램 봇 API를 폴링(Polling) 방식으로 모니터링하다가 메시지가 오면 곧바로 CLI에 던지고 텔레그램으로 직답. 중간 결재자나 UI 없이 모델과 사용자가 직통으로 연결됨.

## 2. 💡 MyCrew 시스템과의 접점 및 시사점

우리가 구축한 **AntiGravity 파일 브릿지(Step 1)**는 소놀봇의 철학과 100% 일치합니다.

*   **비용 무효화 (OAuth)**: 이미 Antigravity IDE의 세션을 빌려 `claude-opus-4-6` 등 최고사양 모델을 추가 API 비용 없이 끌어다 쓰고 있음.
*   **샌드박스(격리) 통신**: `.bridge/requests/`와 `.bridge/responses/` 폴더를 통한 파일 기반 비동기 통신으로 완벽한 격리 달성 완료.

**💡 개선 기회 (Gap)**
소놀봇은 "모바일 ↔ 모델"이 직통이지만, 우리는 현재 "텔레그램 ↔ Ari(비서) ↔ 칸반 보드 ↔ 최고사양 모델" 순서로 결재/라우팅 라인을 거칩니다. 따라서 모바일에서 외부 이동 중 즉각적이고 심층적인 딥워크를 지시하기 위해 **다이렉트 채널** 도입이 필요합니다.

## 3. 🚀 제안: 안티그래비티 ↔ 고사양 모델 텔레그램 전용 채널 개설안

중간 비서(Ari)를 거치지 않고 최고사양 모델(Claude Opus 4.6 등)과 독대하여 심층 작업을 지시할 수 있는 텔레그램 다이렉트 통신 파이프라인 개설 방안입니다.

### 방안 A: 텔레그램 봇 완전 분리 (투트랙 운영)
*   **MyCrew_Ari_Bot**: 일상 지시, 일정 관리, 간단한 질문 (즉각 응답, Gemini Pro)
*   **MyCrew_Prime_Bot (신규)**: 딥워크, 코드/아키텍처 설계, 심층 리서치 전용 (응답에 10~30초 소요되나 고품질 답변)
*   **장점**: 용도가 물리적으로 완벽히 분리되어 사용이 직관적.

### 방안 B: 단일 봇 + 전용 명령어 라우팅 (CTO 루카 추천 🌟)
*   **운영 방식**: 기존 봇을 그대로 쓰되, 텔레그램 채팅창에 `/prime [지시내용]` 또는 `/opus [지시내용]` 형태로 다이렉트 명령어를 추가.
*   **기술 흐름**: 해당 명령어가 감지되면 `server.js`의 텔레그램 라우터가 Ari를 생략하고 즉시 `.bridge/requests/`로 파일을 찔러넣어 Antigravity 어댑터를 직접 호출.
*   **장점**: 봇을 여러 개 관리할 필요가 없고, 인프라 추가 없이 `server.js` 분기문 하나만 추가하면 오늘 당장이라도 구현 가능.
