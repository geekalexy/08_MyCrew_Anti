# C1/C2 메모리 전달 및 고객 설치 가이드

## 1. 메모리 구조

```
~/.claude/projects/-Users-{사용자명}/memory/
  ├── MEMORY.md              (인덱스)
  ├── user_role.md           (고객별 다름)
  ├── project_chatops.md     (공통 - 시스템 구조)
  ├── feedback_cmux.md       (공통 - cmux 주의사항)
  ├── project_socian_team.md (고객 회사별 다름)
  └── ...
```

## 2. 메모리 분류

| 메모리 | 전달 방식 | 내용 |
|---|---|---|
| **공통 (템플릿)** | 파일 복사 | cmux 사용법, 브릿지 구조, 주의사항 |
| **고객별 (커스텀)** | 스크립트가 자동 생성 | 회사명, 에이전트 구성, 이슈 목록 |
| **C2 초기 학습** | 첫 실행 시 프롬프트 자동 주입 | 역할, cmux 명령어, Paperclip 조작법 |

## 3. 스크립트에 포함되는 메모리 세팅

```bash
# 설치 스크립트 중 memory 세팅 부분
MEMORY_DIR="$HOME/.claude/projects/-Users-$(whoami)/memory"
mkdir -p "$MEMORY_DIR"

# 1. 공통 템플릿 복사 (시스템 지식)
cp templates/project_chatops.md "$MEMORY_DIR/"
cp templates/feedback_cmux.md "$MEMORY_DIR/"

# 2. 고객 정보로 커스텀 생성
cat > "$MEMORY_DIR/user_role.md" << EOF
---
name: 사용자 프로필
type: user
---
- 회사명: ${COMPANY_NAME}
- 역할: 대표
EOF

# 3. 인덱스 생성
cat > "$MEMORY_DIR/MEMORY.md" << EOF
- [ChatOps 시스템](project_chatops.md) — 시스템 구조
- [Cmux 주의](feedback_cmux.md) — cmux 패널 안에서 실행
- [사용자](user_role.md) — 대표 프로필
EOF
```

## 4. C2 초기 학습 프롬프트 주입

스크립트가 C2 패널 생성 후 자동으로 학습 프롬프트를 전송:

```bash
# C2에 초기 학습 주입
cmux send --surface $C2_SURFACE "$(cat templates/c2_onboarding.txt)"
cmux send-key --surface $C2_SURFACE Enter
```

### c2_onboarding.txt 내용 (템플릿)

```
너는 cmux 터미널 안에서 실행되고 있는 Claude Code야. 다음을 기억해:

## 너의 환경
- cmux 패널에서 실행 중
- cmux CLI를 Bash 도구로 직접 사용 가능

## cmux 브라우저 명령어 (Paperclip 조작용)
- cmux browser snapshot --surface {BROWSER_SURFACE} → DOM 스냅샷
- cmux browser snapshot --surface {BROWSER_SURFACE} --compact → 간결 스냅샷
- cmux browser get --surface {BROWSER_SURFACE} text "main" → 페이지 텍스트
- cmux browser click --surface {BROWSER_SURFACE} <ref> → 요소 클릭
- cmux browser type --surface {BROWSER_SURFACE} <ref> "텍스트" → 타이핑
- cmux browser fill --surface {BROWSER_SURFACE} <ref> "텍스트" → 필드 채우기

## 패널 구성
- {C1_SURFACE} = C1 (대표 직접 소통)
- {C2_SURFACE} = C2 (너, 텔레그램 비서)
- {BRIDGE_SURFACE} = 텔레그램 브릿지
- {PAPERCLIP_TERM} = Paperclip 터미널
- {BROWSER_SURFACE} = Paperclip 브라우저

## 역할
- 대표가 텔레그램에서 메시지 보내면 너에게 전달됨
- 대표에게 보고하는 톤으로 한국어 응답
- Paperclip 브라우저를 cmux 명령으로 직접 조작 가능
```

## 5. 포함되는 것 vs 안 되는 것

### 스크립트에 포함
- cmux 패널 레이아웃
- Bridge 스크립트 (cmux_telegram_bridge.py)
- Paperclip 설치 및 실행
- Python 의존성 설치
- 공통 memory 파일
- C2 초기 학습 프롬프트

### 포함 안 되는 것

| 항목 | 이유 | 해결 방법 |
|---|---|---|
| C1의 실시간 맥락 | 세션 종료 시 사라짐 | memory 폴더에 핵심만 저장 |
| C2의 대화 히스토리 | 세션 종료 시 사라짐 | `Desktop/claude-3/c2/` 로그 참조 |
| Paperclip 회사 데이터 | Paperclip 자체 DB | 새로 onboard 필요 |
