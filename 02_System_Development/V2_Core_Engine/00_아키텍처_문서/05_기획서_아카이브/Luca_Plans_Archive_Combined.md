# 📑 Luca's Implementation Plans Archive (01-08)

프라임의 리뷰 이력에 맞춰 루카가 그동안 제안하고 실행했던 주요 기획/설계 문서들을 복구하여 아카이브했습니다.

---

## [01] Phase 17: 스킬 생태계 및 런타임 주입 아키텍처
- **목표:** 전사 에이전트의 능력을 모듈화하여 실시간으로 장착/해제 가능한 라이브러리 구축
- **핵심 설계:** 
  - `skills/` 디렉토리 기반의 독립적 스킬 파일 구조
  - 에이전트별 `skills.json` 설정을 통한 동적 로딩 로직

## [02] Phase 17: 방어 로직 및 시스템 안전 가드레일
- **목표:** 에이전트의 예외 상황 발생 시 시스템 전체의 안정성을 보장하는 Fallback 메커니즘
- **핵심 설계:**
  - `SafeExecutor` 도입: 모든 엔진 호출을 래핑하여 타임아웃 및 에러 핸들링
  - 무한 루프 방지를 위한 Recursive Limit 설정

## [03] Phase 17-4: 스킬 관리 UI 및 실시간 상태 동기화
- **목표:** 사용자가 대시보드에서 에이전트의 스킬을 눈으로 보고 토글할 수 있는 UX 제공
- **핵심 설계:**
  - `AgentDetailView` 내 스킬 라이브러리 탭 추가
  - Socket.io를 이용한 엔진-UI 간 스킬 활성화 상태 실시간 동기화

## [04] Hermes: 외부 채널 커뮤니케이션 통합 전략
- **목표:** 텔레그램, 슬랙 등 외부 메신저와의 쌍방향 연동 인터페이스 표준화
- **핵심 설계:**
  - 메신저별 어댑터 패턴 적용
  - 통합 메시지 큐(`MessageQueue`)를 통한 순차적 처리 및 로그 기록

## [05] Seniority Cloning: 보안 스크러빙 및 지식 캡슐화
- **목표:** 시니어의 노하우는 복제하되, 민감한 개인/기업 정보는 노출되지 않는 "Anonymized Seniority" 구현
- **핵심 설계:**
  - `scrubbing.js` 모듈을 통한 정규식 기반 데이터 비식별화
  - SKILL.md 파일 생성 시 자동 필터링 파이프라인 구축

## [06] 온보딩: SaaS 보안 아키텍처 및 KeyProvider
- **목표:** 로컬 개발 환경에서 SaaS 서비스로 전환하기 위한 동적 비밀정보 관리 체계
- **핵심 설계:**
  - `KeyProvider` 3단계 브릿지 (Memory -> DB -> .env)
  - `/api/secrets` 전용 엔드포인트 구축으로 보안성 강화

## [07] Phase 20: 온보딩 기획 고도화 (B2B SaaS)
- **목표:** B2B 고객을 위한 팀 단위 워크스페이스 구축 및 구독 기반 비즈니스 로직 반영
- **핵심 설계:**
  - 팀명(Workspace Name) 기반의 고유 URL 발급 프로세스 기획
  - 모델 구독 인증(API 비용 절감) vs 개별 키 입력 모드 분기 로직

## [08] Phase 20: 제품화 최종 구현 및 튜토리얼 자동화
- **목표:** 가입 직후 즉시 업무 투입이 가능한 수준의 자동화된 유저 경험(Onboarding Flow) 완성
- **핵심 설계:**
  - `TeamActivator`를 통한 팀 목표별 에이전트 스킬 세팅 자동화
  - `TutorialManager`를 이용한 아리의 가이드 미션(Kanban Card) 즉시 생성
