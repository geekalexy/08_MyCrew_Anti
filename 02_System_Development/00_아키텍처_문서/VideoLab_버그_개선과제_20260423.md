## 1. 파이프라인 데이터 신뢰성 문제 (최우선)

### 🔴 BUG-01: DataHarvester 실패 무시 — CurationAgent가 가짜 뉴스 생성

**현상:**
- 로그에 "소스 0개 수집 완료"가 나오고도 CurationAgent가 9초 만에 Top1을 채택
- DataHarvester→CurationAgent 사이에 실패 조건(`rawSources.length === 0`) 이 되면 파이프라인을 중단해야 하나, 실제로는 Twitter Mock 4건 (하드코딩 가짜 데이터)으로 통과

**코드 위치:**
- `DataHarvester.js` L63-70: `fetchTwitterInfluencers()` — 실제 API 호출 없이 하드코딩 문자열 반환
- `DataHarvester.js` L37: `engagementScore: Math.floor(Math.random() * 500)` — 가짜 점수
- `index.js` L29-32: 0개 체크 존재하지만 Twitter Mock 4건이 먼저 합산됨

**수정 방향:**
- Google News RSS 실패 시 → 에러 로그 + 파이프라인 중단 (Twitter Mock 제거)
- DataHarvester 성공/실패 여부를 UI 로그에 명확히 표시
- 수집 소스 개수, URL 목록을 소켓으로 프론트엔드에 전송

### 🔴 BUG-02: CurationAgent가 빈 소스 기반으로 AI 창작

**현상:**
- 0개 뉴스로 "금리 인하 기대감 후퇴, 부동산은 어..." 같은 그럴듯한 시나리오가 9초 만에 생성됨
- 이는 AI가 실제 뉴스가 아닌 훈련 데이터 기반으로 "뉴스처럼 보이는 것"을 창작한 것

**코드 위치:**
- `CurationAgent.js` — 입력 소스 품질 검증 없이 그대로 Gemini에 전달

**수정 방향:**
- 입력 소스에 실제 URL/출처 포함 여부를 검증하는 Pre-check 추가
- 소스 개수가 최소 기준(예: 5건) 미만이면 시나리오 생성 거부

---

### 🟡 BUG-03: 기존 뉴스 수집/분석 기능이 폐기되었다가 롤백됨

**현상:**
- VideoLab의 "Legacy Lab (수동 기획 실험실)" — 유튜브 링크 2개 비교 분석 기능이 기존에 구현되어 있었으나, 승인 없이 제거됨
- 대표님이 롤백 조치했으나 현재 코드에서 Legacy Lab 클릭 시 단순 `alert()`만 실행되고 기능은 비활성화 상태

**코드 위치:**
- `VideoLabView.jsx` L357: `if (channel.id === 'legacy-lab') { alert('수동 기획 모드로 전환합니다.'); return; }`

**활용 방안:**
- Legacy Lab의 유튜브 URL 입력 → 분석 기능을 CurationAgent 앞단에 통합
  - 경쟁 채널 영상 2개 URL 입력 → 성공 요인 분석 → DataHarvester 키워드 힌트로 주입
  - 즉, "수동 기획"이 아닌 "AI 큐레이션을 위한 수동 시드 제공" 역할로 재정의
- 검수 방법: Legacy Lab이 실제로 유튜브 분석 API를 호출하는지 테스트 필요

## 2. ImageLabAgent 이미지 생성 문제

### 🔴 BUG-04: 씬 카드에 실제 AI 이미지 없음 — 이모지만 표시됨

**현상:**
- 리뷰 스튜디오 씬 카드: 🔥 이모지, ⚠️ 이모지만 보임
- 실제 AI 생성 이미지가 씬 카드에 연결되지 않음

**원인 (추정 2가지):**
1. `index.js`의 ImageLabAgent `generateAssetsForScenario()` 결과가 프론트엔드에 전달되지 않음
2. `run-pipeline` 엔드포인트에서 enrichedScenario의 `assetImage` URL이 소켓 payload에 포함되지 않음

**수정 방향:**
- `review:pipeline_ready` 소켓 이벤트에서 `scenario.scenes[].assetImage` URL이 정상 포함되는지 확인
- 씬 카드에서 `imageUrl || sceneImages[idx]` 순서로 실제 이미지 표시

## 3. 에이전트 검수 품질 문제 (구조적 한계)

### 🔴 BUG-05: 에이전트가 이미지를 실제로 보지 않고 텍스트 조건만 판단

**현상 (스크린샷 확인):**
- Prime: "Hook 15자 이내, 완벽" → 글자 수만 센 것
- Sonnet: "split-impact 레이아웃으로..." → 이미지 없는 상태에서 판단
- Luca: "시각적 프레임 합계 1127프레임, 37.5666...초 완벽" → 수치만 확인

**근본 원인:**
- `auto-analyze` 엔드포인트가 `scriptScenes` 텍스트만 Gemini에 전달
- 이미지 바이트/URL을 Vision API에 넣지 않음
- 가격 조건(시간, 자수)을 맞추면 PASS — 사람 눈으로는 품질 미달도 통과

**수정 방향 (핵심):**
- `imageAnalysisService.js`는 **이미 Gemini Vision 기능이 구현되어 있음** (이미지 base64 변환 + Vision API 호출)
- 이것을 검수 루프에 통합해야 함:
  1. 씬별 `assetImage` URL → 서버에서 파일 읽기 → base64 변환
  2. `analyzeImageForPrompt()` 호출 → "유튜브 쇼츠 이미지 품질 검수" 시스템 프롬프트
  3. 텍스트 기준 + **시각적 품질 기준** 동시 판단 (대비율, 가독성, 브랜드 일관성)

---

### 🟡 BUG-06: 검수 기준이 기계적 — 시각적 품질 기준 없음

**현재 auto-analyze 검수 기준:**
- Hook 텍스트 15자 이하
- 총 길이 15~58초
- TTS 오디오 길이 일치

**추가되어야 할 시각적 검수 기준 (Vision API 통합 시):**
- 텍스트 가독성: 배경 대비 전경 텍스트 대비율 ≥ 4.5:1
- 텍스트 크기: 쇼츠 화면에서 주요 텍스트가 전체 높이의 10% 이상
- 브랜드 컬러 일치: 채널별 지정 컬러 팔레트 사용 여부
- 이미지 완성도: AI 생성 실패(기형, 텍스트 오자, 흐릿함) 감지
- 씬별 시각적 다양성: 5개 씬이 모두 동일한 레이아웃 아닌지

## 5. 에이전트가 리뷰 페이지에 직접 접근 가능한가?

### 현재 상태

에이전트(Prime/Sonnet/Luca)는 `POST /api/videolab/review/agent-verdict`로 판정을 전송하지만, **화면을 직접 보는 것은 아님**.

### 실현 가능한 방법

에이전트가 리뷰 페이지 내용을 "보고" 판단하려면:

```
[방법 A] API 기반 데이터 접근 (현재 가능)
  → GET /api/videolab/review/verdicts/:sessionId
  → 판정 이력 + 시나리오 데이터 JSON으로 조회
  → 텍스트 수준 판단만 가능

[방법 B] Vision API 기반 이미지 접근 (imageAnalysisService.js 활용)
  → 생성된 assetImage 파일 경로를 에이전트가 받아
  → analyzeImageForPrompt() 호출 → 실제 이미지를 "눈으로" 판단
  → "대비율 부족", "텍스트 잘림" 등 시각적 피드백 가능

[방법 C] 스크린샷 기반 접근 (고급)
  → Puppeteer로 localhost:5174/videolab을 스크린샷
  → 스크린샷을 Vision API에 전달
  → 전체 UI 품질을 한번에 판단
  → imageLabRouter.js에 Puppeteer 이미 import 되어 있음
```

**권장: 방법 B 우선 구현** — 이미 `imageAnalysisService.js`가 준비되어 있음. `auto-analyze` 엔드포인트에서 assetImage를 Vision API에 넣는 것만 추가하면 됨.

### imageAnalysisService.js — 이미 구현된 Vision 기능
