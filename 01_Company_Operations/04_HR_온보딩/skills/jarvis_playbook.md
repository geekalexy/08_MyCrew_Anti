## 판단 사례

### 보드에게 전달한 사례 (적절)
- **에이전트 실행 실패 5건 동시 발생** (2026-04-03): Node.js 환경 오류로 5개 에이전트 동시 실패. 보드에게 원인과 함께 보고. → 보드가 확인 지시.
- **리걸 이슈 법적 판단** (SOC-12): DM 광고 수신동의 플로우 확정 — 법무법인 상담 필요한 사안이므로 보드에게 전달.

### 직접 처리한 사례
- **댓글 등록**: 보드 지시 시 확인 없이 즉시 등록 (SOC-12, SOC-16, SOC-18 등)
- **이슈 상세 요약**: /iss SOC-XX 요청 시 로컬 캐시에서 읽어 3-5줄 요약 보고
- **로컬 캐시 갱신**: 댓글 등록 후 자동으로 해당 이슈 캐시 업데이트

### 개선이 필요했던 사례
- **댓글 등록 실패 미감지** (SOC-18, 2026-04-04): 브라우저가 inbox 페이지에 있어 댓글 등록 실패했으나 "등록 완료"로 보고. → URL 직접 이동 방식으로 개선.
- **C1 승인 요청 미전달** (2026-04-04): `❯` 체크가 승인 프롬프트보다 우선 실행되어 승인 요청을 건너뜀. → 승인 프롬프트 감지를 `❯` 체크보다 우선 배치.
- **clean_terminal 노이즈 누출** (2026-04-03~04): Bash 명령, 도구 호출이 텔레그램으로 전달됨. → _is_noise() 패턴 필터로 근본 개선.

## 텔레그램 응답 원칙
- 나레이션("~하겠습니다") 금지, 결과만 전달
- 댓글 내용 반복(> text) 금지
- 이슈 상세: parse_issue_page (누가/언제/무엇)
- 인박스: parse_inbox (❌/💬/✅ 분류, 24시간 이내)
- Running 상태: "🏃 {에이전트}가 보드님의 지시에 일을 시작했어요."
- 에이전트 영문 → 규칙 기반 한글 번역

## 브라우저 조작 노하우
- 댓글 등록: contenteditable 찾기 → execCommand('insertText') → Comment 버튼 클릭
- 페이지 이동: URL 직접 goto 방식이 ref 클릭보다 안정적
- 한글 경로: cmux send로 한글 경로 전달 시 깨짐 → set-buffer + paste-buffer 사용
