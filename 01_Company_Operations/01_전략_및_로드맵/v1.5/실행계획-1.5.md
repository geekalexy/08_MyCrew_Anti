# MyCrew v1.0 ~ v1.5 최적화 실행 로드맵 (The Agent OS Masterplan)

본 로드맵은 페이퍼클립 기반의 로컬 봇(1.0)에서, 지능이 자산화된 Antigravity 기반의 기업용 SaaS(1.5)로 나아가기 위한 가장 현실적이고 빠르며 리스크 없는 순차적 마일스톤을 정의합니다.

## User Review Required
> [!IMPORTANT]
> 본 로드맵은 시스템을 멈추지 않으면서(무중단) 아키텍처를 진화시키는 데 초점을 두었습니다. 각 단계별 우선순위 조정이나 추가되어야 할 업무가 있다면 피드백 부탁드립니다.

---

## 🚀 Phase 0: 가벼운 초기 인프라 검증 (Lightweight Setup)
하드포크(Hard-fork) 버전 도입 이전에, `paperclip.ing`의 공식 패키지를 활용하여 **가장 빠르고 가볍게 안티그래비티와 페이퍼클립 간의 API 통신 여부를 검증**합니다. 현재 PC에서 다음 과정을 수행합니다.

### 세부 태스크 (상세 가이드)
- [ ] **Paperclip 패키지 다운로드 및 설치**: `~/mycrew-sandbox` 등의 임시 작업 폴더를 생성하고 터미널에서 `npx paperclipai onboard --yes` 명령을 백그라운드로 실행. 
    * (이 명령어는 임베디드 Postgres DB와 로컬 서버를 자동으로 구성해 줍니다.)
- [ ] **로컬 API 포트 개방 확인**: 로컬호스트에 페이퍼클립 백엔드 포트(예: 3000 등)가 정상적으로 열렸는지 확인 및 서버 라이브 상태(Health check) 테스트.
- [ ] **Antigravity ↔ Paperclip API 통신 테스트**: 브릿지 서버 없이 안티그래비티 도구(run_command 또는 HTTP Request)를 통해 '테스트 목표(Goal) 생성'이나 '에이전트 호출' API를 찔러봄. 성공적인 JSON 응답이 오는지 파싱 검증.

## 🧠 Phase 1: 지능 자산화 (아리 SOP 완성 및 승인)
기존 컴퓨터에 있는 아리의 메모리와 경험(POC 결과 등)을 온전히 이식하여 '헌법'을 완성합니다. **이 문서화가 완전히 끝나고 대표님의 승인이 떨어져야만 본격적인 시스템 고도화 코딩 작업을 시작합니다.**

### 세부 태스크
- [ ] **[작업 대기] 아리의 뇌 추출 체계화**: 전달해주실 '확보된 메모리/레시피'를 바탕으로 `ARI_MASTER_SOP.md`를 작성. (추론을 배제한 명확한 도구 체인 레시피 구조화)
- [x] **[완료] Paperclip 엔진 포크(Fork)**: (외부 종속성 고정 완료됨)
- [x] **[완료] 로컬 맥락 보존 세팅**: (루카/아리에게 이미 적용됨)

## 🛠 Phase 2: 인프라의 역할 교체 (The Great Shift)
사용자에게 응답하던 클로드코드(Luca)를 엔진룸의 SRE(수리공)로 내리고, 브릿지의 소통 창구를 Webhook 기반으로 개편합니다.

### 세부 태스크
- [ ] **브릿지 서버 리팩토링 설계(Luca)**: 텔레그램 `/cmd` 입력을 Antigravity Webhook으로 전달하는 API Gateway 코드 작성.
- [ ] **Antigravity 도구함 규격화**: 페이퍼클립(에이전트 실행)을 Antigravity 내부에서 마음대로 찌를(HTTP Request) 수 있는 범용 도구(Tools)로 등록.
- [ ] **Luca의 SRE 데몬화**: Luca를 PM2 모니터링 시스템에 결합시켜, 마이크루 인프라 에러 시 자동 실행되어 디버깅 및 패치를 수행토록 전환.

---

## ☁️ Phase 3: Zero-Trust 모바일 관제 파이프라인
오프라인에서도 유실되지 않고, 방화벽(로컬망) 제약 없이 폰으로 AI 팀을 지휘할 수 있는 통신 기틀을 마련합니다.

### 세부 태스크
- [ ] **SQLite / Supabase 동기화**: 로컬 엔진(페이퍼클립)의 작업 로그를 클라우드 DB로 밀어 올리는(Outbound) 단방향 싱크 로직 구축.
- [ ] **리버스 터널망 구성**: Tailscale / Ngrok을 연동하여 Antigravity가 로컬 클라이언트에 안전하게 트리거를 날릴 수 있는 채널 확보.
- [ ] **Event Queue(명령 큐) 도입**: 유저가 폰으로 지시한 명령을 클라우드에 큐로 임시 적재한 후, 집의 Mac이 온라인이 될 때 즉시 소화하는 로직 개발.

## 🖥 Phase 4: 대시보드 추상화 및 Agent OS 완성 (상용화)
터미널의 '댓글(이슈)'과 복잡한 로그를 모두 숨기고, 깔끔한 조종석(Cockpit)으로 치환합니다.

### 세부 태스크
- [ ] **하이브리드 워크스페이스 구축**: Antigravity 내부에 Iframe / Widget 형태로 MyCrew 상태창(진행도, 산출물 다운로드 바) 임베딩.
- [ ] **지능적 중재 모듈(HIL)**: 에이전트 간 기술적 갈등 발생 시 예외(Exception) 로그를 띄우지 않고, CEO(사용자)에게 선택지 형태의 "A/B 승인 카드"를 모바일로 제시하는 UX 적용.
- [ ] **SaaS 분양 패키징**: `설치 스크립트(.sh)` 원클릭 배포 체계 마련 및 다중 사용자(멀티테넌트) 분양을 위한 테스트 환경 가동.

## Verification Plan
1. **Automated Tests**: 브릿지 서버 Webhook 라우팅 시 페이퍼클립 로컬 에이전트가 정상 반응하는지 API 테스트 수행.
2. **Manual Verification**: 오프라인 상태에서 모바일(텔레그램/Antigravity)로 명령 전송 후, Mac 구동 시 Event Queue가 정상적으로 명령을 내려받아 소화하는지 라이브 테스트.
