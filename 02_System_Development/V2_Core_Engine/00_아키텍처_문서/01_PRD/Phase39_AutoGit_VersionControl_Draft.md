# PRD: 마이크루 투명한 자동 깃(Invisible Auto-Git) 버전 관리 아키텍처 (Draft)

**문서 상태**: Draft (추후 Phase 39/40 도입 검토용)
**작성자**: 루카 (CTO / 아키텍처)

## 1. 개요 (Overview)
마이크루(MyCrew) 환경에서 코드의 변경 이력을 안전하게 관리하고 롤백(Rollback) 기능을 제공하기 위한 시스템 레벨의 버전 관리 아키텍처입니다. AI 에이전트가 직접 Git 명령어를 다루면서 발생하는 충돌(Merge Conflict)과 컨텍스트 오염을 방지하기 위해, 백엔드 인프라가 에이전트 모르게 조용히 버전 관리를 수행하는 **"투명한 자동 커밋(Invisible Auto-Git) 레이어"**를 도입합니다.

## 2. 아키텍처 계층 설계 (Architecture Design)

### 2.1. 인프라 계층: 투명한 자동 커밋 (Auto-Commit Layer)
에이전트는 단순히 `OUTPUT` 폴더 내의 물리적 파일을 읽고 쓰기만 합니다. 실제 버전 관리는 아리 엔진(Ari Engine)이 백그라운드에서 전담합니다.

*   **저장소 초기화 (Init)**: 
    *   `projectScaffolder.js`가 새로운 프로젝트 워크스페이스를 생성할 때, `07_OUTPUT` 폴더 내부에 자동으로 `git init`을 실행하여 로컬 저장소를 구성합니다.
*   **자동 커밋 트리거 (Trigger)**:
    *   에이전트가 단일 태스크를 완료하고 상태를 `DONE`으로 변경할 때마다 (또는 `executor.js`의 파이프라인 릴레이 스텝이 넘어갈 때마다) 백엔드가 자동으로 아래 명령을 실행합니다.
    *   `git add .`
    *   `git commit -m "Task #{Task_ID}: {Task_Title}"`
*   **기대 효과**: 에이전트는 프롬프트에 Git 사용법을 숙지할 필요가 없으며, 잘못된 명령어 사용으로 인한 Detached HEAD나 파이프라인 중단 사고를 원천 차단합니다.

### 2.2. UI/UX 계층: '타임머신 (Time Machine)' 탭
대시보드의 워크스페이스나 태스크 상세 모달 우측 스플릿 뷰 영역에 **[버전 히스토리]** 탭을 추가하여 시각적 타임라인을 제공합니다.

*   **히스토리 목록**: 
    *   백엔드에서 `git log`를 파싱하여, UI에는 "Task #15: 헤더 디자인 수정 (2시간 전)"처럼 읽기 편한 깔끔한 UI 컴포넌트로 렌더링합니다.
*   **시각적 프리뷰 (Read-Only Preview)**:
    *   특정 버전을 클릭하면 코드가 어떻게 변했는지 보여주는 **Diff View**를 띄우거나,
    *   임시 폴더(Temporary Dir)로 해당 커밋을 Check-out 하여 Iframe을 통해 과거 시점의 렌더링된 결과물을 띄워줍니다 (현재의 Live Preview 스플릿 뷰 재활용 가능).

### 2.3. 롤백(Rollback) 및 복구 파이프라인
프로젝트 상태가 망가졌을 때 원클릭으로 과거 코드로 되돌립니다.

*   **실행 흐름**:
    1. 사용자가 타임머신 탭에서 특정 과거 버전의 **[이 시점으로 복구]** 버튼을 클릭합니다.
    2. 백엔드 인프라가 `git restore` 또는 `git reset --hard`를 실행하여 `OUTPUT` 폴더의 물리적 파일을 과거 상태로 덮어씁니다.
    3. 복구 직후 "Rollback to Task #{Task_ID}" 라는 메시지로 자동 커밋을 하나 더 남겨 롤백 이력 자체도 추적 가능하게 만듭니다.
*   **에이전트 연동의 매끄러움**:
    *   물리적 파일이 통째로 과거로 돌아갔으므로, 사용자가 새 태스크로 "여기서부터 다시 버튼을 파란색으로 만들어"라고 지시하면, 에이전트는 (복구된) 현재 파일을 새로 읽고 작업을 이어나갑니다. 인지 부조화(Time-travel Discrepancy)가 발생하지 않습니다.

### 2.4. 컨텍스트 체이닝(Context Chaining)과의 시너지 연동
진행 중인 Phase 38(Context Chaining)의 기능과 결합할 수 있습니다.

*   **참조-커밋 매핑**: 사용자가 작성하는 `[#ID]` (예: `#5C6`) 코멘트가 생성될 당시의 Git 커밋 해시(Hash)를 DB에 매핑하여 저장합니다.
*   **완벽한 추적**: "이 코멘트를 나눌 당시의 소스코드"를 100% 정확하게 추적하고 시각화할 수 있어, AI와의 커뮤니케이션 및 디버깅 능력이 비약적으로 상승합니다.

## 3. 구현 검토 사항 (Implementation Notes)
*   **Node.js 연동**: `simple-git` 라이브러리 또는 `child_process.exec`를 활용한 비동기 Git 제어 모듈(`gitManager.js`) 구현 필요.
*   **보안 및 용량**: `.git` 폴더가 웹 서버에 의해 외부에 노출되지 않도록 정적 라우팅 예외 처리(`express.static` 제외) 필요.
*   **단일 유저 환경의 이점**: 충돌이나 복잡한 Branch/Merge 전략 없이, 무조건 Main 브랜치 하나에서 선형적(Linear) 커밋만 진행하므로 관리가 매우 단순함.
