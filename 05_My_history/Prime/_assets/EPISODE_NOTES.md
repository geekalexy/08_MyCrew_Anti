## EP-001: 인스타그램 분석 스킬 — AI 팀 협업의 교과서적 사례

**날짜**: 2026-05-06
**관련 아티클**: 2부 (조직편) — 2-3, 2-5 / 3부 (개발편) — 3-1

### 에피소드 요약

CEO가 아리(AI 비서)에게 인스타그램 크리에이터 계정 분석 능력을 부여하고 싶었다. 인스타그램 계정의 팔로워, 팔로잉, 게시물 수, 바이오, 최근 게시물 캡션을 수집해서 비즈니스 홍보에 활용하려는 목적.

### 실제 진행 과정 (AI 에이전트 팀 릴레이)

```
CEO (아이디어)
  → "인스타 계정 정보 수집하는 방법 기획해줘"
  
마케팅 리더 (mkt_lead / Gemini 3.1 Pro)
  → 워크플로우 5단계 설계 + 필요 도구/API 리서치
  → 기획서 산출
  
소넷 (Claude Sonnet 4.6)
  → 기획서 검토 → "활용 가능" 판정
  → 구현 의뢰 수락 → Puppeteer 기반 스크래핑 어댑터 개발
  → POC 검증 → 엔진 통합 → 스킬 등록 완료
```

### 핵심 구현 사항

| 항목 | 내용 |
|------|------|
| 수집 방식 | Puppeteer 브라우저 자동화 (별도 API 과금 없음) |
| 세션 관리 | 최초 1회 로그인 → 쿠키 영구 저장 |
| 단건 분석 | `instagramAnalyze` — 계정 1개 분석 |
| 배치 분석 | `instagramBatchAnalyze` — 최대 10개 동시 분석 |
| 안정성 전략 | CSS 셀렉터 대신 `og:description` 메타태그 활용 (SEO 태그라 변경 빈도 낮음) |

### 블로그 소재로 쓸 포인트

1. **CEO의 역할**: 코드를 안 짰다. "인스타 분석 기능 만들어줘"라는 의도만 전달
2. **릴레이 협업**: 마케팅 리더가 기획 → 소넷이 구현 → 아리에게 장착. 3명의 AI가 릴레이
3. **비용 제로**: 인스타 API 없이, Puppeteer(오픈소스)만으로 구현. 추가 과금 없음
4. **실패 에피소드 포함**: 처음 스킬 장착 시 실패 (skill-library에 폴더가 없어서 orphan 삭제됨) → 검증 로직 추가로 해결
5. **확장 설계**: 지금은 아리 혼자 수집+요약, 나중에 mkt_analyst에게 심층 분석 위임 구조 이미 설계

## EP-002: 루카의 구멍 — "빠르게 했는데 구멍이 많네" (웃프다 에피소드)

**날짜**: 2026-05-07
**관련 아티클**: 3부 (개발편) — 3-3 (디버깅), 3-5 (기술 부채) / 2부 — 2-4 (AI 팀원 충돌)

### 에피소드 요약

라이브 스플릿 프리뷰(Live Split Preview) 기능 개발. 에이전트가 만든 코드 결과물을 대시보드에서 즉시 미리보기할 수 있는 기능. 루카(Gemini, CTO 에이전트)가 백엔드를 담당했는데, 빠르게 완성했지만 구멍이 많았다.

### CEO가 발견한 루카의 실수들

| # | 실수 내용 | 영향 |
|---|---------|------|
| 1 | 기존 `07_OUTPUT` 폴더 경로를 제거 안 하고 새 `outputs` 폴더를 별도 생성 | 경로 이중화 — 에이전트들이 어디에 저장할지 혼동 |
| 2 | 새 프로젝트 하나에만 OUTPUT 폴더 생성, 나머지는 누락 | 다른 프로젝트에서 프리뷰 불가 |
| 3 | `inputs` 폴더는 아예 생성 로직 누락 | 에이전트가 참조할 입력 자료 경로 없음 |
| 4 | ariDaemon.js의 허용 경로(allowedPaths)에 `07_OUTPUT` 잔재 남아있음 | 보안 경로 불일치 |
| 5 | 도구 설명(description)에도 옛 경로 `outputs` 잔재 | 에이전트가 잘못된 경로로 파일 저장 시도 가능 |

### CEO가 직접 수정한 것들 (루카 수습)

- `ariDaemon.js` — allowedPaths `07_OUTPUT` → `OUTPUT` 교체 (2곳)
- `ariDaemon.js` — 도구 description 경로 `outputs` → `OUTPUT` 교체 (2곳)
- `projectScaffolder.js` — 신규 프로젝트 생성 시 `inputs`, `outputs` 폴더 생성 로직 추가
- `executor.js` — 에이전트 파일 저장 경로 교정

### CEO 감정 메모 (원문)

> "빠르게하는데 루카 참 구멍이 많네; 웃프다."
> "하...07_OUTPUT 안지우고 새로 프로젝트 하나에만 OUTPUT 만들고 INPUT은 안만든거 늦게 발견하고 수정하는데도 참 힘들었다."

### 블로그 소재로 쓸 포인트

1. **"웃프다" — 바이브 코더의 감정**: AI가 빠르게 만들어주면 감탄 → 구멍 발견하면 짜증 → 결국 내가 잡아야 함 → 웃기면서 슬프다
2. **AI의 "빠르지만 꼼꼼하지 않은" 특성**: 핵심 기능은 빠르게 구현하지만, 일관성·엣지 케이스·경로 정리 같은 마무리가 약함
3. **CEO가 QA 역할을 겸해야 한다**: 비개발자라도 "뭔가 빠졌는지" 감지하는 능력 필요
4. **수정도 쉽지 않다**: 어디를 고쳐야 하는지 찾는 것 자체가 비개발자에게 큰 도전
5. **루카-소넷 핸드오프**: 루카가 백엔드 완료 → 작업 지시서 작성 → 소넷에게 프론트엔드 위임. AI끼리 문서 기반 핸드오프
6. **🔑 CEO 핵심 인사이트 — "환상의 조합"**: 소넷은 꼼꼼한데 복잡한 작업에서 무한루프에 빠짐. 루카는 복잡하고 방대한 작업을 잘 다루는데 꼼꼼함이 떨어짐. **둘이 보완 관계** → 2-3편(적재적소 배치)의 핵심 소재
