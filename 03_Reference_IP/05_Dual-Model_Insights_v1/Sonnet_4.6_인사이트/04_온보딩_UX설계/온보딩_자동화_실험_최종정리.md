# 🧪 PART 1: 컨텍스트 수집 실험 (v1 → v3)

## 실험 개요

소시안(Socian) 고객사의 비즈니스 컨텍스트를
방법을 달리하며 3번 수집 → 품질 비교

---

## 실험 결과 비교표

| 항목 | v1 (스크린샷만) | v2 (+블로그) | v3 (4종 URL+가이드북) |
|------|--------------|------------|---------------------|
| 사업 정의 | ❌ 오분류 ("소셜 분석") | ✅ DM 자동화 | ✅ + 공식 슬로건 확인 |
| 폰트 | 추정 | 추정 | ✅ **SUIT** (CSS 확인) |
| 가격 플랜 | ❌ | Plan B 일부 | ✅ **4플랜 전체 + 할인율** |
| 기능 목록 | ❌ | 일부 | ✅ **8개 전체 + 가이드북 링크** |
| 타겟 페르소나 | 추정 | 공구셀러 확인 | ✅ **4가지 페르소나 구체화** |
| 해시태그 | ❌ | 9개 | ✅ 공식 확인 |
| 고객 채널 | ❌ | ❌ | ✅ **카카오톡 직접 링크** |

## 핵심 인사이트

> 💡 **Insight #1**: 스크린샷만으로는 사업 카테고리 자체를 오판할 수 있음
> → 스크린샷은 "디자인 보조"로만 사용, 사업 파악에 의존 금지

> 💡 **Insight #2**: 웹사이트 단독 분석은 사업 정의 오류 가능
> → 블로그/SNS 링크는 온보딩 필수 수집 항목

> 💡 **Insight #3**: 수집 소스가 많을수록 정확도 급상승
> → **웹사이트 + 가격페이지 + 문서(GitBook) + SNS** 4종 세트가 최소 표준

# 🏗️ PART 3: 온보딩 자동화 아키텍처 결론

## 현실적 최적 구조 (비용 제로)

```
[사용자] → 대시보드/텔레그램에 URL + 텍스트 입력
    ↓
[아리 서버] → Puppeteer로 URL 스크래핑 (정적 사이트)
    ↓ 스크래핑 완료 → JSON으로 저장
[어드민 알림] → 대표님에게 "처리 요청" 알림
    ↓ 대표님이 한 마디 ("처리해줘")
[루카 (Luca)] → 심층 분석 → MD 파일 3종 생성
    ↓
[아리 크루] → 컨텍스트 파일 읽어서 업무 수행
```

## 장기 목표 구조 (플랜별 AI 분리)

```
Free 플랜   → Gemini Flash (자동, 무료)
Pro 플랜    → Gemini Pro (자동, 무료)
Enterprise → Claude Sonnet API (유료, 최고 품질)
```

# 🎨 PART 4: 컨텍스트 + 템플릿 수집 UX 설계

## 수집할 핵심 정보 2가지

> **① 핵심 고객**: "우리의 핵심 고객은 누구인가?"
> **② 해결 문제**: "우리가 해결하는 문제는 무엇인가?"

## 채팅 수집 방식 (Before/After 프레임)

```
아리:
"[서비스명] 없이는 _____ 때문에 힘들었는데,
 [서비스명]을 쓰면 _____ 가 해결돼요."

빈칸을 채워주시면 됩니다!
```

## 온라인 자료 권장 수집 세트 (4종)

| 우선순위 | 자료 | 이유 |
|---------|------|------|
| ✅ 필수 | 웹사이트 URL | 서비스 정의, 슬로건 |
| ✅ 필수 | 가격 페이지 URL | 플랜, 혜택 |
| ⭐ 권장 | GitBook/Notion | 기능 전체 목록 |
| ⭐ 권장 | 블로그 or SNS | 톤, 해시태그, 마케팅 패턴 |

# 📋 PART 5: 템플릿 자료 수집 UX 설계

## 핵심 설계 원리

> **"크루의 결과 만족도 90%는 컨텍스트와 템플릿에서 나온다"**
> AI가 아무리 똑똑해도, 우리 회사를 모르면 남의 회사 글을 쓴다.

## 채팅 시나리오 3종

| 케이스 | 상황 | 아리 전략 |
|--------|------|---------|
| **A. 일반** | 신규 고객 | 중요성 각인 → URL → 파일 순 |
| **B. 거부** | "자료 없어요" | URL만으로도 가능함 안내 |
| **C. 재방문** | 기존 고객 | 이전 자료 활용 + 추가만 요청 |

## 카드 구성 4섹션

```
섹션 A: 온라인 채널 URL    ← 채팅에서 자동 기입
섹션 B: 기존 콘텐츠 파일   ← ⭐ 가장 중요
섹션 C: 브랜드 자산         ← 선택 (없으면 자동 추출)
섹션 D: 업데이트 주기 설정  ← 싱크 습관화
```

# 🗺️ PART 6: 온보딩 7일 워크플로우 설계

## 타임라인

```

───────────────────────────────────────────────────────────

   컨텍스트 수집 (Day 0~1)    템플릿 공유 유도 (Day 2~5)
```

## SECTION 1 핵심 흐름 (Day 0~1)

```
① 웰컴 위자드 (오버레이)
② 아리 채팅 → 핵심 고객 + 해결 문제 수집
③ URL 수집 → 자동 파싱
④ "컨텍스트 세팅 중" 카드 생성
⑤ 분석 완료 → 컨텍스트 파일 3종 자동 생성
```

## SECTION 2 핵심 흐름 (Day 2~5)

```
① Day 2: 아리 채팅 → "90% 중요성" 각인
② 기존 콘텐츠 파일 수집 (폴더 경로 or 첨부)
③ "기존 콘텐츠 자료 제출" 카드 (섹션 4개)
④ Day 3: 크루 분석 완료 → 스타일 보고서
⑤ Day 4: 첫 번째 템플릿 제공 → 피드백
⑥ Day 7: 세팅 완료 → 정식 운영 시작
```

# 🔑 PART 8: 핵심 결론 및 다음 액션

## 오늘 확정된 원칙들

1. **컨텍스트 + 템플릿 = 크루 품질의 90%**
2. **수집 소스 최소 표준**: 웹사이트 + 가격페이지 + 문서 + SNS
3. **채팅 = 안내 도구, 카드 = 공식 제출 창**
4. **자료는 싱크가 생명**: 2주마다 업데이트 알림 필수
5. **골든타임은 Day 0~2**: 이 구간에 집중 완료해야 함
