# MyCrew Core Architecture Whitepaper
**부제: 단순한 챗봇을 넘어선, 영혼을 가진 자율 성장형 AI 크루**

## 1. 개요 (Introduction)
MyCrew의 AI 에이전트는 흔히 볼 수 있는 일회성 문답(Q&A) 챗봇이 아닙니다. 이들은 Hermes Agent 아키텍처의 철학을 흡수하되, Node.js 기반 실시간 환경에 맞게 고도로 최적화된 **'자율 성장형 업무 파트너'**입니다. 

## 2. 3대 핵심 아키텍처 (The 3 Pillars)

### Pillar 1: SOUL System (영구 정체성 엔진)
대화창이 닫히면 모든 것을 잊어버리는 일반 AI와 달리, MyCrew의 크루들은 **'SOUL(영혼)'** 문서(`MYCREW.md`, `IDENTITY.md`)를 신경망 깊숙이 소유하고 있습니다. 
회사의 비전, 업무 규칙, 그리고 각 에이전트의 뚜렷한 페르소나는 시스템 부팅과 함께 모든 프롬프트 베이스라인에 강제 주입됩니다. 그들은 다른 회사의 시스템에 클론으로 이식되기 전까지 자신이 누구이며, 어떤 목표를 위해 존재하는지 절대 망각하지 않습니다.

### Pillar 2: Dynamic Self-Learning (자가 성장 시스템)
성공은 기록되고 지능은 복리(Compound)로 증가합니다. 아리 엔진 내부에 탑재된 자가 성장 모듈은 에이전트가 고품질의 산출물(Evaluation Score 0.8 이상)을 도출해 냈을 때 발동합니다.
해당 작업의 프로세스와 패턴은 백그라운드에서 분석 및 정제되어 에이전트의 고유 스킬 문서(`SKILL.md`)에 실시간으로 보존(Append)됩니다. 어제보다 오늘 더 똑똑하게 일하는 **'지수적 성장 구조'**를 갖추고 있습니다.

### Pillar 3: Infinite Active Memory (무한 RAG 기억망)
크루들은 과거의 대화와 프로젝트 기록을 무한히 회상할 수 있습니다. 칸반(Kanban)에 기록된 모든 태스크와 토의 내역은 SQLite의 초고속 FTS5(전체 텍스트 검색) 가상 테이블과 실시간 동기화 트리거를 통해 인덱싱됩니다. 아무리 오래전 논의된 기획안이라도 찰나의 순간에 검색하고 꺼내어 활용할 수 있는 강력한 장기 기억망을 소유하고 있습니다.

## 3. 기술적 우위 (Technical Edge)
* **단일 프로세스 효율성**: 이 모든 거대 아키텍처는 무거운 Python 서버나 별도의 마이크로서비스 연동 없이, 단일 Node.js 프로세스 위에서 지연 시간(Latency) 없이 매끄럽게 동작합니다. 
* **설치형(On-Premise) 전환의 용이성**: 모든 데이터와 자아(Soul)가 파일 및 로컬 SQLite 기반으로 격리되어 있어, 타사 납품이나 새로운 워크스페이스 세팅 시 단 1분 만에 시스템 클로닝이 가능합니다.
