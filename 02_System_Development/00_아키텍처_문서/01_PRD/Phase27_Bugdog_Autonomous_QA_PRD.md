# [Phase 27] Bugdog & 자율형 CS 리포팅 파이프라인 (PRD)

## 1. 기획 개요 (Overview)
*   **배경:** 상용화 이후 다수 텐넌트의 CS 관리를 위함이나, 그보다 앞서 **현재 MyCrew 자체 개발 환경(Dogfooding)에 즉시 도입**하여 Bugdog의 효용 가치를 증명하고 마이크루 개발 및 디버깅 효율을 극대화하기 위함.
*   **목표:** 버그 발생 시 시스템이 자율 점검(Bugdog)을 수행하고, 비서(Ari)가 스스로 판단하여 본사(HQ)로 CS 리포트를 자동 접수하는 **'Zero-Touch Proactive Support'** 환경 구축 및 당장 실무 적용.
*   **핵심 철학:** 제4의 벽을 수호하면서도, 자체 개발 과정부터 이 파이프라인을 적극 사용하여 "마이크루로 마이크루의 버그를 잡는" 자율화 사이클을 완성.

---

## 2. 핵심 아키텍처 및 워크플로우

### [Step 1] 무인 순찰견 'Bugdog' 가동
*   **작동 방식:** 새벽 시간대(예: AM 03:00) 또는 유휴 시간에 크론잡(Cron-job)으로 동작하는 백그라운드 E2E 테스터.
*   **역할:** 유튜브 업로드, 이미지 렌더링, 외부 API 통신 등 핵심 파이프라인을 가상으로 찔러보고(Health Check) 응답 지연이나 실패가 있는지 확인.
*   **출력:** 이상 감지 시, 개발팀만 읽을 수 있는 로우 데이터 형태의 `ErrorLog JSON` 생성 및 내부 이벤트 훅(Event Hook) 트리거.

### [Step 2] 아리(Ari)의 자율 진단 및 판단 (Autonomous Decision)
*   아리는 Bugdog이 던진 ErrorLog를 수신한 뒤, 에러의 성격과 심각도(Severity)를 스스로 분석합니다.
*   **[Low / Warning] 단순 지연 또는 일시적 네트워크 오류:** 
    *   본사에 보고하지 않고 자체 로그만 남기며, "새벽에 일시적인 지연이 있었으나 현재 정상입니다" 정도로 사용자에게 알림.
*   **[High / Critical] API 만료, 토큰 해지, 파이프라인 붕괴 등 치명적 오류:**
    *   **자율 판단 개입:** 사용자가 출근해서 `[본사로 전송]` 버튼을 누를 때까지 기다리지 않습니다. 아리가 즉시 버그 리포트를 작성하여 **본사(HQ)의 수집 서버로 자동 접수(Auto-Submit)** 합니다.

### [Step 3] UI 피드백 및 사용자 보고 (Front Desk UX)
*   **CS 리포트 대시보드 (Settings & Drawer 연동):** 
    *   CS 업무는 상시 노출될 필요가 없으므로, 메인 프로젝트 메뉴가 아닌 **설정(Settings) 페이지 내에 'CS' 탭**을 신설하여 배치.
    *   해당 탭 진입 시, **우측 패널(Timeline/Chatting 서랍)을 'CS 전용 버그 리포트 및 칸반 보드' 영역으로 전환**하여 공간 활용도를 극대화함.
*   **아리의 브리핑 (타임라인/채팅):**
    *   사용자가 아침에 대시보드를 열면 아리가 먼저 말을 건넵니다.
    *   *"대표님, 간밤에 Bugdog 정기 점검 중 이미지 렌더링 서버와 통신이 끊긴 것을 확인했습니다. 제가 즉시 본사(HQ) 개발팀에 긴급 복구(CS 리포트 #1042)를 요청해 두었으니, 복구가 완료될 때까지 카드뉴스 발행 업무는 잠시 보류해 주십시오."*
*   **진행 상태 연동:** 본사에서 에러를 픽스하고 상태를 `Resolved`로 바꾸면, 고객의 CS 리포트 메뉴에도 실시간으로 반영되며 아리가 "복구 완료되었습니다!"라고 보고함.

## 3. 필요 개발 스펙 (나중이 아닌 지금 즉시 도입)

1. **`Bugdog 데몬`**: Puppeteer 또는 단순 HTTP Fetch 기반의 마이크로 테스터 스크립트 작성 (`02_System_Development/04_인프라_및_도구/bugdog/`).
2. **`Ari 자율 판단 스킬`**: `skill-library/cs-reporter/` 스킬 신설. Error JSON을 분석해 본사(현재는 우리의 로컬 Dev HQ)로 리포트를 쏘는 기능.
3. **`프론트엔드 UI`**: 설정(Settings) 메뉴 내 CS 탭 구성 및 우측 Drawer를 CS 칸반 보드로 전환하는 로직(`CSKanbanDrawer.jsx`) 추가.

## 4. 기대 효과
*   **고객 (User):** 에러를 겪고 스트레스받기 전에 AI 비서가 알아서 대처하는 진정한 '자율 운영 체제'의 감동 경험.
*   **개발팀 (HQ):** 파편화된 고객의 불만 섞인 추상적 문의 대신, 아리가 정제해 준 **'완벽한 재현 경로와 에러 코드'가 담긴 리포트**를 받아 즉각적인 디버깅 가능.
