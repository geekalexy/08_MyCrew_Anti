## 1. 리뷰 목적
마이크루의 아키텍처를 기존 **파일 폴링(File-Polling) 기반에서 MCP(Model Context Protocol) 서버 구조로 전환**하는 거시적 전략(Phase 38)과, 브라우저 제어 권한을 가진 **크롬 익스텐션 에이전트 고도화**(Phase 38-1)에 대한 모든 기획서 작성이 완료되었습니다. 
본 리뷰 요청서를 통해 기획 방향의 무결성과 아키텍처 설계의 정합성을 최종 승인받고자 합니다.

## 3. 핵심 리뷰 요청 사항
1. **마이그레이션 전략 검토**: "UI/DB는 상속하되, 구형 파일 폴링 코어는 깃허브에 안정 버전으로 동결하고, 백엔드를 MCP 서버 코어로 심장 교체한다"는 전략이 개발 자원 배분 및 리스크 관리 측면에서 적절한지 승인 부탁드립니다.
2. **컨텍스트 오염 방지 정책**: 무분별한 DB 저장을 막기 위해 `/context` 커맨드의 결과물을 '1회성 휘발성 리소스'로 제한한 보안 조치 검토 부탁드립니다.
3. **목표 지향형(Goal-Driven) 전환**: 단순히 정해진 룰(Rule-based)대로 넘겨주는 릴레이가 아니라, 목표(`/Goal`)를 선언하면 외부의 MCP 클라이언트들이 알아서 전략을 세우는 차세대 자율 에이전트 모델로의 방향성에 동의하시는지 확인 부탁드립니다.
