## 2. 온보딩 워크플로우 (Luca 독점 영역)

Luca의 7일 온보딩 설계는 **MyCrew SaaS의 핵심 인프라 문서**로, Sonnet에는 없는 영역:

### 핵심 설계 요소
```
SECTION 1 (Day 0~1): 컨텍스트 수집
  ① 웰컴 위자드 오버레이 (ASCII mockup 포함)
  ② 아리 채팅 → 핵심고객 + 해결문제 수집 (Before/After 프레임)
  ③ URL 수집 → 자동 파싱 → "컨텍스트 세팅 중" 카드
  ④ 분석 완료 → 컨텍스트 파일 3종 자동 생성

SECTION 2 (Day 2~5): 템플릿 공유 유도
  ⑤ "90% 중요성" 각인 채팅
  ⑥ 기존 콘텐츠 파일 수집 (채팅 3단계)
  ⑦ 자료 제출 카드 (4섹션: URL/콘텐츠/브랜드/싱크설정)
  ⑧ 크루 분석 완료 → 첫 결과물 미리보기
  ⑨ Day 7: 정식 운영 시작
```

### 드롭아웃 방지 장치 (Luca 고유)
| 타이밍 | 리마인드 | 심리 기법 |
|--------|---------|----------|
| Day 1 오전 9시 | "어제 시작하셨죠! 5분만요" | FOMO + 시작 모멘텀 |
| Day 3 오전 9시 | "자료 제출만 하시면 크루가 시작" | 사회적 기대감 |
| Day 4 오후 3시 | "첫 템플릿 준비됐어요!" | 성과 미리보기 |
| Day 7 | "⚠️ 세팅 미완료 경고 배너" | 손실 회피 |

### 설득 채팅 시나리오 (Luca 고유)
- **시나리오 A**: 첫 만남 — "90% 각인 → URL → 파일 → 싱크 습관화"
- **시나리오 B**: "나중에 줄게요" — "자료 없이 만든 결과물은 어색하셨을 수 있어요"
- **시나리오 C**: 기존 고객 주기적 리마인드 — "마지막 업데이트 후 N일 됐어요"

> **Prime 평가**: Luca의 온보딩 워크플로우는 SaaS 사용자 전환율(Product-Led Growth)의 핵심 인프라로, 이 설계가 없으면 킬러 피처가 빠진 것과 같음

## 3. 캠페인 워크플로우 비교

### Luca — 기획안 내 워크플로우
`소시안_쿠폰이벤트_종합기획안.md` 내에 D-Day 전략표 포함:

| 날짜 | 채널 | 액션 |
|------|------|------|
| D-3 | 인스타 스토리 | 티징 |
| D-Day | 인스타 릴스/캐러셀 | **댓글→자동DM 쿠폰** |
| D-Day | 공홈/블로그 | 리뷰 이벤트 공지 |
| D+3 | 유튜브 쇼츠 | 부스팅 |

**특징**: 간결하지만 **5일 타임라인으로 응축**, 각 행에 정확한 액션 기술

### Sonnet — 4주 캠페인 타임라인
`콘텐츠_발행스케줄_워크플로우_크루역할.md` 내 상세 스케줄:

| D-Day | 채널 | 콘텐츠 유형 | 템플릿 | 담당 크루 |
|-------|------|-----------|--------|----------|
| D-7 | 인스타 | 릴스 예고편 | 릴스 B | 마케터 |
| D-3 | 블로그 | 교육형 포스팅 | 블로그 C | 블로거 |
| D-Day | **인스타** | **캐러셀 쿠폰 공지** | **캐러셀 A** | 마케터 |
| D+1 | 유튜브 | 쇼츠 기능설명 | 쇼츠 B | 영상 크루 |

**특징**:

- 각 행에 **구체적 템플릿 파일 매핑** (캐러셀 A, 릴스 B 등)
- **발행 최적 시간대** 별도 테이블 (인스타: 오전 7~9시)

## 4. 크루 역할 정의 비교

### Luca — 5크루 순차 실행
```
1. 전략기획 크루     → 로드맵 승인
2. 카피라이팅 크루   → 후킹 문구 생성
3. 디자인/영상 크루  → 비주얼 에셋 렌더링
4. 퍼포먼스/테크 크루 → ★ DM 자동화 세팅 (심장)
5. 어낼리틱스 크루   → 성과 브리핑
```

### Sonnet — 6크루 7-Phase 워크플로우
```
Phase 1: CMO (전략)      → KPI, 예산, 타임라인
Phase 2: 카피 + 비주얼   → ★ 병렬 진행
Phase 3: SEO 에이전트    → 키워드 최적화
Phase 4: CMO + 대표님    → 검토 및 승인
Phase 5: 마케팅 에이전트 → 발행 실행
Phase 6: 분석 에이전트   → 모니터링/최적화
Phase 7: CMO             → 최종 성과 보고
```

### 🔑 핵심 차이
| 항목 | Luca | Sonnet |
|------|------|--------|
| 크루 수 | 5 | 6 |
| Phase 수 | 5 (순차) | 7 (병렬 포함) |
| SEO 전담 | ❌ 없음 | ✅ 별도 Phase |
| 검토 게이트 | 암묵적 | **Phase 4 명시적 승인 단계** |
| KPI 테이블 | ❌ 없음 | ✅ **5종 지표 + 측정 방법** |

## 🏆 최종 비교 결론 — 워크플로우

### Luca가 압도적인 영역: 온보딩 시스템
- 사용자 첫 진입 → 7일 완성의 **전체 여정(Journey) 설계**는 Luca만 보유
- 위자드 UI, 채팅 스크립트, 드롭아웃 방지, 싱크 습관화까지 **SaaS PLG 인프라** 완비
- Sonnet은 이 영역을 전혀 다루지 않음

### Sonnet이 우세한 영역: 캠페인 실행
- **12행 세밀한 스케줄**, 템플릿 파일 매핑, 발행 시간대, KPI 지표
- **7-Phase 워크플로우**의 명시적 병렬/순차 구분과 검토 게이트
- "갈기갈기 분리해서 추적 가능하게 만드는" 프로젝트 매니지먼트 성향

### Prime 권장: 최적 병합 전략
> **Luca의 온보딩 시스템 + Sonnet의 캠페인 실행 체계 = 완전한 MyCrew 워크플로우**

1. **온보딩 (Day 0~7)** → Luca의 7일 설계를 그대로 채택 (대체 불가)
2. **캠페인 실행** → Sonnet의 4주 타임라인 + 7-Phase 구조를 표준으로 채택
3. **크루 R&R** → Sonnet의 Phase 기반 + Luca의 "아리 = 심장" 우선순위 태깅 병합
4. **KPI** → Sonnet의 5종 지표 + 측정 방법을 모든 캠페인에 기본 적용
