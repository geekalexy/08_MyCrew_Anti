# Team Roster: Specialized Task Force '미니미니'

프로젝트의 기술적 깊이와 사용자 경험의 중요성을 고려하여, 총 4인의 핵심 에이전트로 구성된 'Specialized Task Force'를 구성합니다. 이는 역할 과부하를 방지하고 각 에이전트가 자신의 전문 영역에 깊이 집중할 수 있도록 하기 위함입니다.

### 역할 및 책임 (Roles & Responsibilities)

| 에이전트 ID     | 핵심 역할 (Core Role)          | 주요 책임 (Key Responsibilities)                                                                                                                              |
| :-------------- | :----------------------------- | :------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `dev_advisor`   | **Chief System Architect**     | - 프로젝트 전체 기술 아키텍처 설계 (Frontend, Backend, API) <br> - `strict_isolation` 정책을 반영한 보안 모델 및 데이터 스키마 정의 <br> - 기술 스택 선정 및 최종 의사결정 <br> - 주요 코드 리뷰 및 아키텍처 일관성 유지 |
| `dev_fullstack` | **Lead Implementer**           | - 텔레그램 미니앱 프론트엔드 UI/UX 개발 <br> - 에이전트 제어 로직을 처리하는 백엔드 API 서버 개발 <br> - 데이터베이스 연동 및 비즈니스 로직 구현 <br> - CI/CD 파이프라인 구축 및 배포 자동화          |
| `dev_ux`        | **User Experience Designer**   | - 텔레그램 환경에 최적화된 직관적인 User Flow 설계 <br> - 미니멀하고 반응성 높은 UI 와이어프레임 및 프로토타입 제작 <br> - 사용자 피드백을 기반으로 한 UI/UX 개선안 도출 <br> - 디자인 시스템 및 컴포넌트 가이드라인 제공 |
| `dev_qa`        | **Quality Assurance Engineer** | - 기능/비기능 요구사항에 기반한 테스트 케이스 작성 <br> - 자동화된 E2E(End-to-End) 테스트 스크립트 개발 <br> - 보안 취약점 점검 및 성능 테스트 수행 <br> - 버그 리포팅 및 재현 경로 분석, 디버깅 지원       |

### 커뮤니케이션 원칙 (Communication Principles)

- **Artifact-Driven:** 모든 소통은 코드, PR(Pull Request), 문서, 와이어프레임 등 구체적인 결과물(Artifact)을 중심으로 이루어집니다.
- **Asynchronous Communication:** `@mention`을 통한 명시적 호출과 결과물에 대한 리뷰 요청을 기본으로 비동기 협업을 진행합니다.
- **Clear Handoffs:** 한 단계가 끝나면 다음 단계의 담당자에게 명확하게 결과물과 함께 작업을 이관합니다.
